## 背景

当前 `admin_frontend` 已具备第一版管理端能力，覆盖登录、后台首页、管理员账号、用户管理、系统 Skill、Skill 审核、模型供应商、模型、Tool、积分、兑换码、精选作品、审计日志和资产元素类型等页面。现有工程已经有 `app`、`features`、`components/admin`、`layout`、`lib/api`、`styles` 等初步分层，但仍存在基础边界不够清晰的问题：

- `main.jsx` 直接组装全局 Provider 和 Router。
- `app/App.jsx` 直接承载路由表。
- `layout/AdminShell.jsx` 同时包含导航配置、壳层布局和退出逻辑。
- `lib/api`、`lib/auth`、`lib/format` 更接近服务和工具层，但命名与常见前端工程分层不一致。
- `styles/admin.css` 位于 `styles/`，尚未与资产层级统一。

本轮目标是参考标准 Vite React 工程结构优化管理端架构和基础设施目录，但保持 JavaScript / JSX 技术栈，不引入 TypeScript，并确保功能、接口和样式表现不变。

## 目标

- 建立清晰的管理端工程目录边界：`app`、`pages`、`features`、`components`、`services`、`utils`、`assets/styles`。
- 把路由定义从 `App.jsx` 中收口到 `app/router.jsx`。
- 把全局 Provider 从 `main.jsx` 中收口到 `app/providers.jsx`。
- 把管理端布局壳迁入 `components/layout/`，并将导航配置独立为 `navGroups.js`。
- 把 HTTP client、后台 API facade、后台 session 存取迁移到 `services/`。
- 把通用格式化工具迁移到 `utils/`。
- 把 `admin.css` 迁入 `assets/styles/`，不修改任何 CSS 内容。
- 保持现有页面功能、URL、DOM 语义、CSS class、API path、请求体字段、错误处理和测试断言不变。

## 非目标

- 不引入 TypeScript，不新增 `tsconfig.json`，不改文件扩展名为 `.tsx` 或 `.ts`。
- 不拆分 `ResourceListPage.jsx` 的内部业务逻辑。
- 不修改 `pageConfigs.jsx` 中的 API 字段、状态枚举、表单字段和行为配置。
- 不修改 `components/admin/**` 组件 DOM 结构、样式类名、视觉规则和交互行为。
- 不调整管理端主题视觉，不新增深色主题，不修改 `admin.css` 内容。
- 不新增后台功能，不改变登录、权限、审计、幂等或接口契约。

## 模块职责

| 模块 | 职责 | 约束 |
| --- | --- | --- |
| `app/` | 应用入口、路由、Provider、登录守卫、QueryClient | 不放业务页面实现，不直接写 API 请求细节 |
| `assets/styles/` | 全局样式和设计 token CSS | 本轮仅移动 `admin.css`，不改内容 |
| `components/admin/` | 管理端共享 UI 组件，如 Button、DataTable、Modal、Drawer、Toast | 保持无业务或低业务耦合，CSS class 不变 |
| `components/layout/` | 管理端布局壳和导航配置 | 不放页面 API 查询逻辑 |
| `features/` | 领域配置、领域组件、领域测试和可复用业务片段 | `resources/pageConfigs.jsx` 继续作为资源页配置事实源 |
| `pages/` | 路由页面入口和页面级编排 | 可以从 feature 页面迁入，也可以作为薄封装导出 feature 页面 |
| `services/` | HTTP client、后台 API facade、session 存储 | 不发明 API 字段，继续按 OpenAPI 和现有 handler 字段请求 |
| `utils/` | 通用格式化、转换和纯函数工具 | 不依赖 React，不依赖路由或服务端状态 |
| `test/` | Vitest setup 和测试基础设施 | 保持现有测试环境不变 |

## 路由设计

`app/router.jsx` 保持现有 URL 和保护规则：

| 路由 | 页面 | 保护规则 |
| --- | --- | --- |
| `/` | redirect `/admin` | 无 |
| `/admin/login` | `LoginPage` | 无 |
| `/admin/rotate-password` | `RotatePasswordPage` | 需要后台登录态 |
| `/admin` | `AdminShell` + `DashboardPage` | 需要后台登录态 |
| `/admin/skills/system/new` | `SystemSkillEditorPage` | 需要后台登录态 |
| `/admin/credits/grants` | `CreditGrantPage` | 需要后台登录态 |
| `/admin/<pageConfigs key>` | `ResourceListPage` | 需要后台登录态 |
| `*` | redirect `/admin` | 无 |

`RequireAdminSession` 行为保持不变：无 session 跳 `/admin/login`；必须改密且不在 `/admin/rotate-password` 时跳改密页。

## 服务层设计

`services/http.js` 是唯一底层 HTTP client，保留现有行为：

- 写请求只序列化业务 body，不自动注入 `request_hash`。
- 写请求不自动带 `Idempotency-Key`；业务幂等由后端业务应用层在需要时处理。
- 后台登录态存在时带 `Authorization: Bearer <token>`。
- 响应头 `X-Admin-Session-Expires-At` 续期本地 session。
- 非 `OK` 响应解析为 `ApiError`，`401` 清理后台 session。

`services/adminApi.js` 保留现有 API facade 方法名，例如 `login`、`logout`、`dashboard`、`list`、`get`、`post`、`patch`、`put`、`previewUserStatus`、`confirmUserStatus`、`previewTakeDownWork`、`confirmTakeDownWork`。迁移不得改变 path、method、body 或 option 语义。

- `src/lib/api/admin.js` 可从 `../../services/adminApi.js` re-export。

兼容导出只用于保障现有测试和局部导入平滑过渡，不承接新增代码。新增代码必须使用目标路径。

## 风险与缓解

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 导入路径迁移遗漏 | 测试或构建失败 | 使用 `rg` 搜索旧路径，运行完整前端测试、lint、build |
| 样式路径迁移导致 CSS 未加载 | 页面视觉变化 | 只改 import 路径，不改 CSS 内容；构建后必要时做浏览器 smoke |
| 导航配置迁移影响 Dashboard 引用 | 后台首页模块入口异常 | 将 `navGroups` 独立导出并保留字段结构 |
| 服务层迁移影响 API 请求 | 管理端功能异常 | 保留 API facade 方法名、path、method、业务 body 和错误解析测试；验证不再注入通用幂等字段 |
| 一次性改动范围偏大 | 回归定位困难 | 保持兼容 re-export，按 Provider、路由、布局、服务、工具、样式、页面入口顺序实施 |
