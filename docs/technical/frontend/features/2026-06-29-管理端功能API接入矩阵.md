## 本次接入补齐项

- 通用资源页支持自定义筛选、表单动作、`PATCH`/`PUT`、JSON 字段、JSON 字符串字段、数组字段、复选框和动作成功提示。
- OpenAPI 同步管理端实际 DTO：管理员创建、用户关键词、系统 Skill 草稿/测试结果、模型供应商、模型价格快照、Tool 策略/计价/白名单、积分发放过期时间、精选作品筛选。
- 业务服务补齐用户关键词搜索、模型价格快照写入和公开作品管理端筛选。
- 高风险操作继续通过确认弹窗、预览 token、JSON body 原因和后端业务防重进入审计链路。

## 2026-06-29 Agent 真实运行链路 E2E 归档

关联脚本：`scripts/e2e-agent-runtime-config.sh`
关联报告：`tests/reports/admin-full-browser-rpc-db-2026-06-29.md`

本节补齐“管理端配置是否真的被用户侧 Agent 使用”的验收口径，不再把 HTTP 200 或页面 smoke 当成闭环。

已固化链路：

| 层级 | 固化断言 |
| --- | --- |
| 管理端写入 | 后台登录后创建真实模型供应商、真实 image 模型、设置 image/global 默认模型、创建系统 Skill 和 `output_elements` |
| Business DB | 断言 `model_providers/models/default_models/skills/skill_versions/skill_output_element_schemas/skill_test_cases` 均落到本轮唯一数据，Skill 已发布，测试用例数量为 3 |
| Agent RPC 消费 | Agent 通过 Business RPC 获取可路由 Skill、发布版 Skill Spec 的 `output_elements` 和默认 image 模型快照 |
| 用户提示词运行 | 普通用户创建 Agent session/run，prompt 中包含唯一 `route_hints.keyword`，必须触发 `agent.skill.selected(matched_reason=route_hint:keyword)` |
| Agent DB | 断言 `agent_runs.skill_selection` 保存本轮 Skill + `output_elements`，`agent_runs.model_selection_snapshot` 保存本轮模型、供应商运行引用、timeout 和 route_config |
| 脱敏边界 | 用户可见事件不得泄露 `provider_runtime_ref`；完整模型快照只允许保存在服务端 interrupt payload 供确认后执行 |

后续约束：

- 验收 `系统 Skill`、`模型供应商`、`模型管理` 是否可用时，必须覆盖“后台配置落库 -> Agent RPC 消费 -> 用户 prompt 触发 run -> Agent DB 快照”。
