# 功能迭代文档入口

状态：active  
owner：文档与契约责任域  
更新时间：2026-06-28  
适用范围：进行中的技术功能迭代、变更记录和完成后归档规则

## 目标

本目录只存放当前或即将开始的功能迭代技术文档。已完成阶段的总结进入 `docs/releases/**`；过期设计进入 `docs/archive/**` 或标记 `archived`。

## 何时新建

- 新增或大改一个功能模块。
- 一次变更同时影响产品、后端、前端、契约、数据库或测试中的两类以上。
- 需要把历史归档结论迁移成当前 active 事实源。
- 需要为 GitHub issue、PR 或发布批次提供统一技术上下文。

## 命名规则

推荐格式：`YYYY-MM-DD-功能名迭代.md`。

## 内容要求

使用 `docs/templates/功能迭代文档模板.md`。至少包含产品来源、技术范围、契约影响、数据库影响、测试计划、实现状态、验证证据、归档条件。

## 生命周期

| 状态 | 说明 |
| --- | --- |
| draft | 范围未确认，只能澄清和补文档。 |
| review | 等待产品、契约或技术复核。 |
| active | 当前迭代事实源，可进入实现。 |
| archived | 迭代完成，结论已进入 release 或被更高层文档承接。 |
| deprecated | 方案废弃，必须指向替代文档。 |
