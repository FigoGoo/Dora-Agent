# Agent 工作台与 A2UI 组件视觉规范

状态：active
owner：产品与需求责任域
更新时间：2026-06-27
适用范围：Dora-Agent Web C 端统一 Agent 创作工作台、聊天区域、聊天外部 A2UI 组件、资产区、黑板、生成确认、进度和状态展示

相关契约：事件类型、顺序和 payload 以 AG-UI/A2UI 协议文档为准；本规范只定义视觉与交互呈现。

## 背景

统一 Agent 是 C 端核心体验。用户通过一个 Agent 完成音乐、图片、视频和组合创作。A2UI 组件需要在聊天区域和聊天外部区域共同展示 Agent 的中间结果、工具状态、Skill 输出、资产和黑板内容。如果缺少工作台和 A2UI 的视觉规范，前端容易把每个事件临时渲染成不同样式，导致体验割裂。

## 目标

- 明确 Agent 工作台的顶部项目工具条 + 三栏创作布局和弹性规则。
- 明确聊天区域组件和聊天外部组件的视觉边界。
- 统一 Thinking、Tool、Skill、Model、扣费确认、生成进度和资产/黑板展示。
- 保证 A2UI 组件跟随站点主题，不形成独立视觉系统。
- 工作台采用沉浸式创作结构：左侧故事板，中间预览区，右侧对话面板，顶部项目工具条。

## 非目标

- 不定义 AG-UI/A2UI 事件 payload。
- 不定义 Agent 路由、Skill 调用和工具执行逻辑。
- 不定义后端任务状态机。
- 不展示模型内部推理链路。

## 工作台布局

桌面默认：

```text
顶部项目工具条：项目标题 / 素材 / 项目 / 剪辑 / 复制 / 导出 / 积分 / 用户
左侧 StoryboardPanel：故事板、关键元素、分镜、素材缩略图、黑板结构化内容
中央 PreviewStage：当前元素 / 分镜预览、模型、尺寸、生成结果
右侧 ChatPanel
  Header：对话
  MessageList：消息流、Thinking、Tool/Skill 状态、Media Assets 结果卡、结果表格
  Composer：输入框、模型、Skill、资产库入口、发送按钮
```

比例：

| 显示环境 | StoryboardPanel | PreviewStage | ChatPanel |
| --- | --- | --- | --- |
| 笔记本小屏 | `28-32%`，必要时可折叠 | `34-38%` | `30-36%` |
| 标准桌面 | `30-34%` | `34-40%` | `28-32%` |
| 宽屏 | 固定舒适扫描宽度 | 居中扩大预览 | 固定舒适阅读宽度 |
| 超宽屏 | 不超过故事板最大宽度 | 预览不无限拉伸 | 不把消息无限拉宽 |

约束：

- StoryboardPanel 列表独立滚动，选中卡片必须有清晰描边。
- PreviewStage 必须保持足够留白，空态或生成预览居中展示。
- ChatPanel 输入区固定底部，消息列表独立滚动。
- 资产摘要在 ChatPanel 内以 MediaAssetsCard 呈现，详细资产和黑板结构在 StoryboardPanel / PreviewStage 呈现。
- 超宽屏不把对话气泡和文本无限拉长。

## A2UI 组件分区

用户已确认 A2UI 分为两类：

| 分区 | 组件 | 说明 |
| --- | --- | --- |
| 聊天区域组件 | 消息、思考状态、Tool/Skill Tag、确认卡、进度卡、表单控件 | 出现在对话流内部 |
| 聊天外部组件 | 故事板、预览区、资产、黑板 | 出现在 StoryboardPanel 和 PreviewStage |

规则：

- 不需要模型选择 A2UI 组件，模型选择在聊天输入框内完成。
- A2UI 组件样式必须符合站点主题。
- A2UI 支持思考状态的打印机输出。
- A2UI 支持 Tool、Skill 等平台内置组件 Tag 展示。
- 单选、多选、输入框等可出现在聊天区域作为交互控件。

## 聊天区域

### 消息布局

消息类型：

- 用户消息。
- Agent 文本消息。
- 系统状态消息。
- Tool 调用消息。
- Skill 运行消息。
- 确认消息。
- 错误消息。

视觉规则：

- 不做过大聊天气泡。
- 用户消息右侧或明显区分，Agent 消息左侧。
- 单条消息最大阅读宽度，宽屏下不无限拉伸。
- 消息间距 `12px` 到 `16px`。
- 消息时间和状态使用弱文本。

### Agent 文本流

状态：

- `message.started`：创建消息容器。
- `message.delta`：增量渲染。
- `message.completed`：固化消息。

视觉规则：

- 增量文本使用正常正文，不闪烁。
- 完成后不改变排版。
- 网络恢复后补齐内容，不重复消息。

### ThinkingTypewriter

用途：

- 展示 Agent 正在分析、规划、调用 Skill 或等待工具结果。

样式：

- 字号 `12px` 到 `13px`。
- 文本颜色 `rgba(255,255,255,0.50)`。
- 使用打印机输出效果。
- 容器可折叠。
- 完成后默认收起或弱化。

规则：

- 只展示公开处理状态，不展示模型内部推理链路。
- 文案使用“正在分析素材”“正在生成分镜”“正在调用图片生成工具”等外部可见状态。
- 不展示隐藏 Prompt、密钥、内部成本和工具原始参数。

### Tag 体系

Tag 类型：

| Tag | 用途 | 示例 |
| --- | --- | --- |
| `SkillTag` | 当前使用的 Skill | `MV 制作` |
| `ToolTag` | 当前调用的 Tool | `图片生成` |
| `ModelTag` | 当前生成模型 | `Seedance 2.0` |
| `StatusTag` | 当前状态 | `生成中` |
| `RiskTag` | 风险或安全状态 | `需确认` |
| `CreditTag` | 积分影响 | `预计 10 积分` |

样式：

- 高度 `20px` 到 `24px`。
- 字号 `11px` 到 `12px`。
- 圆角 `9999px` 或 `7px`。
- 背景半透明。
- 图标使用 `lucide-react`。

规则：

- Tag 必须短小，不承载长句。
- 多个 Tag 横向排列，超出换行或折叠。
- 风险 Tag 不能只靠颜色表达。

## 输入框 Composer

组成：

- 文本输入。
- 上传素材。
- 模型类型选择：图片、音乐、视频。
- 资产引用。
- 参数入口。
- 发送按钮。

规则：

- 模型选择在输入框内，不做独立 A2UI 组件。
- 用户选择的模型当前对话生效。
- 用户不选择模型时使用默认模型。
- 进入扣费确认后不允许修改模型。
- 输入框支持引用已经创作的资产。

视觉：

- 默认使用玻璃 Prompt 样式。
- 内部工具按钮使用 `25px` 高度。
- 选择的模型、资产、参数以 capsule 展示。
- capsule 可移除，但进入确认后锁定。

## 生成前确认卡

出现时机：

- 会消耗积分的生成任务。
- 需要用户确认模型、数量、时长或资产引用。

必须展示：

- 任务类型。
- 当前空间。
- 选择模型名称。
- 预计积分。
- 当前余额。
- 生成数量、首数或时长。
- 确认和取消。

视觉：

- 卡片宽度跟随消息区。
- 背景使用弱玻璃面板。
- 积分信息突出但不使用强警告色。
- 确认按钮使用主按钮。

规则：

- 确认卡出现后模型和关键参数锁定。
- 余额不足时确认按钮禁用。
- 企业空间必须显示企业名称。

## 生成进度卡

阶段：

- 排队中。
- 分析中。
- 生成中。
- 保存中。
- 完成。
- 部分完成。
- 失败。
- 已取消。

展示内容：

- 当前步骤。
- 步骤进度。
- 已完成产物摘要。
- 可取消入口。
- 错误原因和重试入口。

视觉：

- 使用纵向步骤或紧凑进度条。
- 长任务可折叠详情。
- 完成后转为产物摘要卡。

规则：

- 生成完成才扣费。
- 只要生成完成就扣费。
- 取消时已完成部分按规则展示。
- 保存失败要保留创作过程，提示重试保存。

## Tool 调用展示

Tool 调用状态：

- pending。
- running。
- succeeded。
- failed。
- cancelled。
- timed_out。

展示内容：

- Tool 名称。
- ToolTag。
- 状态。
- 耗时。
- 参数摘要。
- 结果摘要。

规则：

- 不展示敏感参数、密钥、内部 URL。
- 失败时展示可理解原因。
- 可重试 Tool 展示重试入口。
- Tool 风险等级用 RiskTag 展示。

## Skill 运行展示

Skill 状态：

- selected。
- running。
- needs_input。
- completed。
- failed。

展示内容：

- Skill 名称。
- SkillTag。
- 当前步骤。
- 所需用户输入。
- 输出摘要。

规则：

- Agent 根据意图选择不同 Skill。
- 没有匹配 Skill 时使用文本模型对话，不提示创建 Skill。
- Skill 输出的结构化元素优先进入黑板。

## 交互表单组件

聊天区域可出现：

- 单选。
- 多选。
- 输入框。
- Textarea。
- 参数确认。
- 文件选择。

规则：

- 表单组件必须使用公共组件规范。
- 提交后展示用户选择摘要。
- 表单过期或会话状态变化时禁用并说明原因。
- 不在聊天消息里塞复杂多页表单；复杂配置进入 Drawer 或 Builder。

## 右侧 Workspace Panel

结构：

```text
Panel Header
Tabs: 资产 / 黑板
Toolbar: 筛选 / 搜索 / 视图切换
Content
```

规则：

- 图片、音乐普通生成默认打开资产。
- 视频、MV、短视频、广告视频默认打开黑板。
- 用户手动切换后当前会话保持上次选择。
- Panel 可折叠，折叠后保留入口和未读/更新提示。

## 资产 Panel

资产类型：

- 图片。
- 音乐。
- 视频。
- 文本。
- 提示词。
- 分镜。
- 时间线。
- 上传素材。

展示形式：

- 网格卡。
- 列表行。
- 当前会话产物分组。
- 引用中资产分组。

卡片信息：

- 缩略图或类型图标。
- 名称。
- 类型。
- 状态。
- 生成来源。
- 操作：预览、引用、下载、更多。

规则：

- 新生成资产进入资产 Panel 并高亮一次。
- 资产保存失败展示错误状态和重试。
- 可以引用已经创作的资产继续创作。
- 音频和视频必须可预览。

## 黑板 Panel

用途：

- 展示 Skill 产出的素材、提示词和故事线。
- 按 Skill 给出的结构展示创作过程。
- 视频类按分镜展示图片、脚本、提示词等。

结构：

```text
故事线摘要
分镜列表
  分镜编号
  画面 / 素材
  脚本 / 旁白
  提示词
  生成状态
时间线摘要
```

元素类型：

- 文本。
- 长文本。
- 富文本。
- 图片。
- 音频。
- 视频。
- 提示词。
- 分镜。
- 时间线。
- 参数组。
- 标签组。

规则：

- 平台内置固定元素类型，但前端从配置读取，不写死。
- 元素卡片不套卡片。
- 分镜列表支持折叠和展开。
- 当前生成中的分镜有状态指示。
- 提示词默认可查看，但敏感内部 Prompt 不展示。

## 断线重连和恢复

视觉状态：

- reconnecting：顶部或消息区显示恢复中。
- restored：恢复成功，补齐缺失事件。
- restore_failed：恢复失败，提供刷新或返回项目。

规则：

- 重连时保留已有消息和资产。
- 不能重复渲染同一事件。
- 未知事件不导致页面崩溃，显示兼容提示或忽略。

## 安全和内容治理展示

第一版内容安全治理使用 LLM 进行提示词安全评估。

状态：

- safe。
- needs_revision。
- blocked。

展示规则：

- blocked 使用明确错误卡。
- needs_revision 提供修改建议。
- 不展示内部安全评分和模型判定细节。
- 用户可修改提示词后重新提交。

## 页面状态

| 状态 | 展示 |
| --- | --- |
| loading | 工作台骨架 |
| empty | 开始一次创作 |
| streaming | 文本增量和停止入口 |
| thinking | 打印机输出 |
| confirming | 扣费确认卡 |
| generating | 进度卡 |
| interrupted | 人工确认或补充输入 |
| reconnecting | 恢复状态条 |
| error | 错误卡和恢复入口 |

## 禁用项

- 禁止展示模型内部推理链路。
- 禁止展示 API Key、密钥、内部成本、内部 Tool 原始 URL。
- 禁止把模型选择做成 A2UI 独立组件。
- 禁止聊天消息里嵌入复杂后台表格。
- 禁止 Tool 和 Skill 状态只用纯文本无 Tag。
- 禁止右侧资产和黑板脱离主题样式。
- 禁止 hover、streaming、loading 导致工作台布局抖动。

## 前端验收清单

- [ ] 工作台三栏布局在 1440px、1920px、2560px 下可用。
- [ ] 小屏桌面右侧 Panel 可折叠或 Drawer 化。
- [ ] 消息流支持 started、delta、completed 增量渲染。
- [ ] Thinking 使用打印机状态且不展示内部推理。
- [ ] Tool、Skill、Model、Status 使用 Tag 展示。
- [ ] 扣费确认卡展示空间、模型、积分、余额和确认动作。
- [ ] 资产 Panel 能展示图片、音乐、视频和失败状态。
- [ ] 黑板能展示分镜、脚本、提示词和生成状态。
- [ ] 断线重连不重复渲染事件。
- [ ] 所有图标来自 `lucide-react`。
