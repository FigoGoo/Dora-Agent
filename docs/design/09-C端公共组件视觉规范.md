相关契约：前端组件展示字段以 API 契约和 AG-UI 事件协议为准，不在本规范中定义后端字段。

## 背景

C 端页面包含首页、Agent 创作工作台、项目/会话、资产库、Skill、积分、作品中心、精选作品、个人中心和企业空间。仅有视觉 token 不足以约束真实开发，必须补充公共组件的尺寸、状态和使用边界，避免不同页面各自发明按钮、卡片、表单和反馈样式。

## 目标

- 统一 C 端组件的尺寸、字重、圆角、透明度、交互状态和图标来源。
- 保证夜间主题和日间主题使用同一套组件语义。
- 让前端开发可以直接拆分基础组件和业务组件。
- 避免组件在 Agent 工作台、作品中心、资产库和个人中心中风格漂移。

## 非目标

- 不定义平台后台管理端组件规范。
- 不定义 API 字段、RPC 字段或 AG-UI payload。
- 不定义最终 Figma component variant 命名。
- 不覆盖复杂图表、数据分析和运营报表组件。

## 组件分层

| 层级 | 组件 |
| --- | --- |
| 基础组件 | Button、IconButton、Input、Textarea、Select、Checkbox、Radio、Switch、Slider、Tabs、Tooltip |
| 布局组件 | AppSideNav、PublicSideNav、ContextHeader、PageBody、ResponsiveDrawer |
| 反馈组件 | Toast、Alert、Empty、Skeleton、ErrorState、Progress |
| 浮层组件 | Modal、Drawer、Popover、DropdownMenu、LoginModal |
| 内容组件 | Card、AssetCard、WorkCard、SkillCard、ModelOption、CreditCard |
| Agent 组件 | ChatMessage、ThinkingTypewriter、ToolTag、SkillTag、ConfirmPanel、GenerationProgress |
| 选择器组件 | Upload、FilePicker、AssetPicker、ModelPicker、SkillPicker |

## 全局组件规则

- 所有组件必须使用 [视觉风格与设计 Token 规范](./08-视觉风格与设计Token规范.md) 中的颜色、字体、圆角和透明度。
- 所有图标统一使用 `lucide-react` 按需引入。
- 仅图标按钮必须有 `aria-label` 和 tooltip。
- 所有可交互组件必须覆盖 default、hover、focus-visible、active、disabled、loading。
- 所有表单组件必须覆盖 error、success、help text、required、disabled。
- 组件内部文本不得使用 `vw` 缩放，不得使用负字距。
- 组件不能因 hover、loading、长文本或图标切换导致布局尺寸跳动。
- C 端创作体验允许使用透明玻璃和胶囊；业务管理型页面保持克制、可扫描。

## Layout Components

### AppSideNav

用途：

- 登录后 C 端左侧全局导航。
- 承载 Logo、新建创作、首页、工作台、项目、资产库、Skill、作品中心、精选作品和积分。

规则：

- 桌面默认展开，小屏桌面可折叠为图标栏。
- 当前路由高亮使用图标、文字和弱背景组合，不只依赖颜色。
- 不承载复杂表单、筛选器或业务详情。
- 企业身份不替换导航结构，只通过当前空间和可见入口体现差异。

### PublicSideNav

用途：

- 未登录公开首页、精选作品中心和公开作品详情的左侧轻导航。

规则：

- 只展示 Logo、首页、精选作品、公开 Skill、登录和注册。
- 点击创作、点赞、资产库、项目、积分等登录态入口时触发 LoginModal。
- 登录成功后切换为 AppSideNav，但页面主体上下文保留。

### ContextHeader

用途：

- 右侧内容区顶部状态条。
- 展示页面标题、当前空间、积分、通知、主题和用户菜单。

规则：

- 不承载一级主导航。
- 当前空间和企业空间标识必须可见。
- 页面标题和面包屑过长时截断并提供 tooltip。

## Button

### 尺寸

| Size | 高度 | padding | 字号 | 用途 |
| --- | --- | --- | --- | --- |
| `xs` | `25px` | `0 10px` | `11px` | Prompt 内部工具按钮 |
| `sm` | `32px` | `0 14px` | `13px` | 次级操作、卡片操作 |
| `md` | `36px` | `0 16px` | `14px` | 默认按钮 |
| `lg` | `40px` | `0 20px` | `14px` | 主操作、确认动作 |

### 变体

| Variant | 用途 | 夜间样式 |
| --- | --- | --- |
| `primary` | 主要动作，如开始创作、确认扣费 | `background: var(--accent)`，深色文字 |
| `secondary` | 次级动作，如取消、保存草稿 | `rgba(255,255,255,0.08)` 背景 |
| `ghost` | 轻量动作，如更多、查看详情 | 透明背景，hover 显示弱面板 |
| `danger` | 删除、移除、撤销分享 | 风险色背景或风险色描边 |
| `glass` | 创作入口、Skill hover、Prompt 内按钮 | 半透明背景 + blur |

### 状态

- hover：只调整背景、边框和文字色，不改变尺寸。
- active：可降低亮度或增加内阴影。
- focus-visible：必须有可见描边，颜色使用 `--accent-soft`。
- disabled：透明度降至 `0.45`，鼠标不可点击。
- loading：保留原尺寸，左侧显示 loader 或替换图标，不改变按钮宽度。

## IconButton

尺寸：

| Size | 容器 | 图标 |
| --- | --- | --- |
| `sm` | `28px` | `16px` |
| `md` | `32px` | `18px` |
| `lg` | `36px` | `20px` |

规则：

- 图标来自 `lucide-react`。
- 必须提供 `aria-label`。
- 默认背景透明，hover 使用 `rgba(255,255,255,0.08)`。
- 工具栏中的 IconButton 必须有 tooltip。
- 危险操作图标不能只依赖颜色表达风险，tooltip 或确认弹层必须说明影响。

## LinkButton

用途：

- 查看全部。
- 返回列表。
- 打开详情。
- 跳转精选作品或个人作品。

规则：

- 字号 `13px` 或 `14px`。
- 默认使用 `var(--muted)`，hover 变为主文本色。
- 不把主要提交动作做成 LinkButton。
- 外链必须明确打开方式，必要时使用外链图标。

## LoginModal

用途：

- 未登录访客触发创作、点赞、创建项目、查看个人资产、作品中心、积分或通知等需要登录的动作。
- API 返回 `UNAUTHENTICATED` 且当前页面可以保留上下文时。

规则：

- 作为 Modal 展示，不默认整页跳转。
- 展示登录和注册入口，支持关闭后留在当前公开页面。
- 必须保留触发前的页面、Prompt、作品、Skill 或目标动作。
- 登录成功后继续原动作；无法继续时返回触发位置并提示用户重新确认。
- 不在弹窗内展示企业空间、积分、资产等登录后私有数据。

## Input

尺寸：

| Size | 高度 | padding | 字号 |
| --- | --- | --- | --- |
| `sm` | `32px` | `0 12px` | `13px` |
| `md` | `36px` | `0 14px` | `14px` |
| `lg` | `40px` | `0 16px` | `14px` |

夜间样式：

- 背景 `rgba(255,255,255,0.05)` 或 `rgba(0,0,0,0.30)`。
- 边框 `rgba(255,255,255,0.10)`。
- placeholder `rgba(255,255,255,0.38)`。
- focus 边框 `var(--accent-soft)`。

规则：

- 输入错误时显示错误文字和错误边框。
- 搜索框左侧使用 `Search` 图标。
- 密码框右侧使用 `Eye` / `EyeOff` 图标。
- 输入框高度不因错误文案出现而改变同一行其他控件布局。

## Textarea

用途：

- Prompt 草稿。
- Skill 描述。
- 作品简介。
- 审核意见和反馈。

规则：

- 最小高度 `96px`，长文本编辑可 `160px` 起。
- 支持自动增高时必须设置最大高度，超过后内部滚动。
- 不使用大圆角，默认 `8px` 或 `12px`。
- Prompt 主输入可使用 `20px` 圆角和玻璃样式。

## Select / Combobox

用途：

- 模型类型筛选。
- 作品分类。
- 资产类型。
- 企业空间或身份选择。

规则：

- 默认高度 `36px`。
- 下拉面板使用 Popover 样式。
- 选项文字超长省略。
- 支持搜索的选择器使用 Combobox。
- 不能用浏览器默认 select 裸样式。

## Checkbox / Radio / Switch

| 组件 | 尺寸 | 用途 |
| --- | --- | --- |
| Checkbox | `16px` | 多选、协议、批量选择 |
| Radio | `16px` | 单选模型、分类、可见范围 |
| Switch | `36px x 20px` | 开关设置，如公开分享、启用记忆 |

规则：

- 选择控件必须可键盘操作。
- Label 点击区域包含文字和控件。
- Switch 只用于即时状态开关，不用于表单内多选。

## Slider / Stepper

用途：

- 视频时长。
- 图片数量。
- 音乐首数。
- 生成数量。

规则：

- 数值必须同时可输入，不能只靠拖动。
- 有最小值、最大值、步进值和单位。
- 高风险或扣费相关数值必须进入确认卡后锁定。

## Tabs / SegmentedControl

Tabs 用于页面或面板级切换：

- 资产 / 黑板。
- 图片 / 音乐 / 视频。
- 全部 / 已完成 / 失败。
- 个人作品 / 分享作品。

SegmentedControl 用于局部模式：

- 网格 / 列表。
- 当前对话 / 当前项目。
- 简洁 / 详细。

规则：

- Tab 高度 `31px` 到 `36px`。
- Active 背景可使用 `rgba(255,255,255,0.12)`。
- Tab 数量过多时横向滚动，不压缩到不可读。

## DropdownMenu / Popover

用途：

- 更多操作。
- 模型选择。
- 作品分类筛选。
- 用户菜单。
- 当前空间切换。

样式：

- 背景 `rgba(15,15,15,0.92)`。
- 边框 `rgba(255,255,255,0.10)`。
- blur `16px`。
- 圆角 `10px`。

规则：

- 菜单项高度 `32px` 到 `36px`。
- 危险项放在底部并使用风险色。
- 点击外部关闭。
- 支持 Esc 关闭和键盘导航。

## Modal

用途：

- 扣费确认。
- 删除确认。
- 转让企业拥有者。
- 审核提交。
- 兑换码兑换结果。

尺寸：

| Size | 宽度 |
| --- | --- |
| `sm` | `420px` |
| `md` | `560px` |
| `lg` | `720px` |

规则：

- Modal 必须有标题。
- 高风险动作必须写清影响对象和不可逆后果。
- 主按钮放右侧，取消按钮在主按钮左侧。
- 内容超过视口时内部滚动，底部操作固定。

## Drawer

用途：

- 资产详情。
- 作品详情侧栏。
- Skill 配置详情。
- 消耗明细筛选。

规则：

- 右侧 Drawer 宽度 `420px`、`560px`、`720px` 三档。
- 小屏时可全屏。
- Drawer 不用于主流程确认，确认仍用 Modal 或 ConfirmPanel。

## Tooltip

规则：

- 仅图标按钮必须有 tooltip。
- tooltip 文案不超过一行，必要时改为 Popover。
- 延迟显示，避免鼠标扫过时大量闪烁。
- 不在 tooltip 中放交互控件。

## Toast / Alert

Toast 用于短反馈：

- 保存成功。
- 复制成功。
- 已加入作品中心。
- 兑换成功。

Alert 用于页面内提示：

- 余额不足。
- 上传失败。
- 安全评估未通过。
- 当前企业空间权限不足。

规则：

- Toast 不承载复杂决策。
- 错误 Toast 必须提供可理解原因。
- 需要用户处理的问题用 Alert，不只弹 Toast。

## Empty / Skeleton / ErrorState

Empty 必须包含：

- 空态标题。
- 一句原因或引导。
- 一个主操作或返回入口。

Skeleton 使用场景：

- 首次加载列表。
- 作品网格加载。
- 资产卡加载。
- 项目卡加载。

ErrorState 必须包含：

- 用户可理解错误。
- 重试或返回入口。
- 必要时提供错误详情展开，不默认暴露技术栈。

## Card

通用 Card：

- 圆角 `8px` 到 `12px`。
- 夜间边框 `rgba(255,255,255,0.08)`。
- 背景 `rgba(255,255,255,0.04)` 到 `rgba(255,255,255,0.05)`。

规则：

- 不做卡片套卡片。
- 可点击卡片 hover 不改变尺寸。
- 卡片内标题和操作区布局固定，避免加载状态跳动。

## AssetCard

展示内容：

- 缩略图或媒体类型图标。
- 资产名称。
- 类型 Tag：图片、音乐、视频、文本、分镜、提示词。
- 状态：保存中、可用、失败。
- 操作：预览、引用、下载、更多。

尺寸：

- 网格卡最小宽度 `180px`。
- 列表行高度 `64px` 到 `80px`。

规则：

- 音频资产需要播放/暂停状态。
- 视频资产展示封面和时长。
- 失败资产保留位置，展示重试或删除。

## WorkCard

用于个人作品中心和精选作品。

展示内容：

- 封面。
- 作品标题。
- 作品类型。
- 分类和标签。
- 点赞、分享、公开状态。
- 创建或更新时间。

规则：

- 精选作品卡以封面为主，底部暗遮罩承托文字。
- 个人作品卡可更偏管理，显示公开/私密、编辑和删除。
- 标题最多两行，超出省略。

## SkillCard

展示内容：

- Skill 名称。
- 来源：系统、个人、企业。
- 审核状态。
- 可见范围。
- 简短描述。
- 操作：使用、编辑、提交审核、版本。

规则：

- 首页热门 Skill 使用视觉规范中的玻璃卡。
- Skill 管理列表不使用大预览 hover，保持可扫描。
- 审核状态必须使用 StatusBadge，不只靠文字颜色。

## Tag / Badge

Tag 类型：

- `SkillTag`
- `ToolTag`
- `ModelTag`
- `StatusBadge`
- `RiskBadge`
- `AssetTypeTag`
- `WorkCategoryTag`

尺寸：

- 高度 `20px`、`22px`、`24px` 三档。
- 字号 `11px` 或 `12px`。
- 圆角 `9999px` 或 `7px`。

规则：

- Tag 文案短小，不超过 8 个中文字符优先。
- Tag 超长必须省略。
- 风险和状态 Tag 不能只靠颜色，必要时加图标。

## Upload / FilePicker / AssetPicker

Upload 状态：

- idle。
- dragging。
- uploading。
- success。
- failed。
- blocked。

规则：

- 上传区域必须显示大小和格式约束。
- 上传失败保留错误原因和重试入口。
- 安全评估失败使用 blocked 状态。
- AssetPicker 支持按类型、时间、来源筛选。
- 选择资产后以 capsule 或小卡片展示，支持移除。

## CreditEstimate / ConfirmPanel

用于生成前确认扣费。

必须展示：

- 生成类型。
- 选择模型名称。
- 预计扣费。
- 当前余额。
- 生成数量或时长。
- 确认和取消。

规则：

- 进入确认后不允许修改模型。
- 余额不足时主按钮禁用并提供去兑换或联系管理员入口。
- 企业空间扣费必须显示当前空间名称。

## GenerationProgress

展示内容：

- 当前步骤。
- 总步骤或阶段。
- 已生成产物摘要。
- 可取消入口。
- 错误和重试。

规则：

- 生成完成前不得把产物误标为可用。
- 取消后展示已完成部分和未完成状态。
- 长任务必须能回到项目或会话历史继续查看。

## 可访问性

- 所有可点击元素必须可键盘访问。
- focus-visible 状态必须清晰。
- 仅图标按钮必须有 `aria-label`。
- 表单错误必须关联输入框。
- 文本和背景对比度必须满足基础可读。
- 动效必须支持降低动效偏好。

## 前端验收清单

- [ ] 组件使用统一 token。
- [ ] 图标只使用 `lucide-react`。
- [ ] 所有交互态齐全。
- [ ] loading 不改变布局宽高。
- [ ] 长文本不溢出。
- [ ] 错误态有可理解原因和恢复入口。
- [ ] 夜间主题和日间主题均可渲染。
- [ ] 关键组件在 1440px、1920px 和小屏桌面截图验收。
