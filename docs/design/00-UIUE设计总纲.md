## 背景

Dora-Agent 第一版是 Web 端 AIGC Agent 创作工具。用户通过统一 Agent 完成音乐、图片、视频和组合式创作，并通过资产、黑板、个人作品中心和精选作品中心沉淀结果。

用户已确认：

- 参考 Flova 的样式色系。
- 结构也可以参考 Flova。
- 需要日间和夜间两种深浅主题。
- 需要弹性布局，适应不同显示器。
- 只在 Web 端运行。
- 用户端主视觉方向调整为暗色电影级创作控制台。
- 用户端桌面 Web 改为左右结构，左侧为全局导航，右侧为页面内容和工作台。

## 设计目标

- 建立专业、沉浸但不花哨的 AIGC 创作工具体验。
- 让用户打开 Web 后可以快速进入 Agent 创作、项目/会话、资产库、Skill 和作品中心。
- 让用户端形成稳定的左侧导航 + 右侧内容结构，减少顶部横向导航在创作场景中的干扰。
- 让未登录访客可以先浏览首页公开作品和精选作品，降低首访门槛。
- 让统一 Agent 工作台同时承载对话、生成确认、进度、资产和黑板。
- 让精选作品中心具备公开浏览和作品展示感。
- 让平台后台使用独立管理端主题，保持高密度、可扫描、低装饰的管理端体验。
- 让日间/夜间主题拥有一致语义 token，而不是两套独立样式。

## 产品气质

第一版采用“暗色电影级创作控制台”作为用户端主气质：

- 首页：左侧轻导航，右侧黑底 Prompt、少量快捷入口和公开作品露出，保持极简。
- 创作工作台：左侧全局导航稳定存在，右侧沉浸、面板清晰，像 AI 导演控制台。
- 作品中心：更有流媒体内容广场感，支持图片、音乐、视频卡片。
- 精选作品中心：类似作品画廊/案例库，不做强社区娱乐流。
- 平台后台：独立浅色默认管理端主题，高密度表格和表单，不复用 C 端暗色电影级创作视觉。

## 主题策略

第一版必须支持：

- 夜间主题：用户端默认气质，适合沉浸式创作和视频/音乐工作流。
- 日间主题：用于长时间整理、后台管理和可访问性补充。

主题切换规则：

- 用户端和 Agent 工作台支持主题切换。
- 平台后台支持日间/夜间，但默认日间；夜间主题是低亮度办公模式，不复用用户端沉浸式暗色视觉。
- 未登录公开首页和精选作品中心跟随系统或默认日间，后续可支持用户选择。
- 所有颜色必须使用语义 token，不允许页面内硬编码主题色。

## Web-only 约束

- 第一版只设计 Web 端。
- 不设计 iOS / Android 原生 App。
- 移动端只保证浏览、查看和基础对话可用，不作为完整创作主场景。
- Agent 创作、后台管理、Skill Builder、资产/黑板以桌面 Web 为主。

## 弹性布局原则

支持以下显示环境：

| 环境 | 设计策略 |
| --- | --- |
| 13-14 寸笔记本 | 紧凑布局，右侧工作区可折叠 |
| 15-16 寸笔记本 | 标准双栏或三栏工作台 |
| 24-27 寸显示器 | 左侧全局导航 + 右侧页面内容；工作台内容区可继续分 Agent 和右工作区 |
| 宽屏 / 超宽屏 | 多面板并列，内容不无限拉伸 |

布局约束：

- 页面主体不使用固定死宽。
- 使用容器断点和弹性栅格。
- 用户端桌面默认采用左右结构：左侧 `AppSideNav` 固定或可折叠，右侧 `PageContent` 承载页面标题、状态区、主体内容和弹窗触发点。
- Agent 工作台聊天区、资产区、黑板区支持可调宽或预设比例。
- 精选作品中心使用响应式卡片网格。
- 后台表格允许横向滚动、固定关键列、隐藏低优先级列。
- 超宽屏下使用最大阅读宽度或增加辅助面板，不把文本拉长。

## 设计原则

- 不用说明文字代替交互。状态、按钮和布局本身要清楚。
- 所有高风险、扣费、业务写入动作必须有确认态。
- A2UI 组件跟随站点主题样式，不做独立视觉系统。
- 思考状态只展示可公开处理状态，不展示模型内部推理链路。
- Tag 展示 Skill、Tool、Model、Risk、Status 等公开信息，不展示密钥、成本、内部 ID。
- 作品公开展示不暴露源会话、黑板、提示词、积分、模型成本和用户隐私。
- 未登录访客触发创作、点赞或个人数据入口时使用登录弹窗承接，不丢失当前公开页面上下文。
- 用户端主导航不放在顶部横向展开；顶部只保留右侧内容区内的页面上下文、当前空间、积分、通知、主题和用户菜单。
- 后台不使用营销式英雄区，不使用大面积装饰图。

## 非目标

- 本阶段不做可交互前端实现，PNG 视觉稿仅用于评审。
- 本阶段不写前端代码。
- 本阶段不定义最终 API 字段。
- 本阶段不做移动 App 设计。
- 本阶段不做推荐算法、社区互动、评论等复杂视觉设计。
