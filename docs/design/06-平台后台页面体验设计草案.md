## 背景

平台后台用于配置平台级能力，不是普通用户创作入口。第一版平台后台必须独立入口、独立代码、独立登录态，并覆盖管理员账号、用户管理、系统 Skill、Skill 审核、模型、Tool、积分、兑换码、精选作品管理和审计日志。

`02-平台后台与运营管理 PRD` 已确认 Done，本文件作为页面体验和工程映射输入；正式字段以 API、RPC 和 AG-UI 契约为准。

## 设计目标

- 让后台保持高密度、可扫描、低装饰的管理端体验。
- 对所有高风险后台操作提供确认、原因、结果反馈和审计提示。
- 不展示用户私有创作内容、密钥明文、兑换码完整列表或内部推理链路。
- 为后续 API、RPC、审计日志和前端页面拆分提供页面级输入。

## 非目标

- 第一版不做平台后台 RBAC。
- 不做运营数据看板、推荐算法、榜单运营或公开作品审核队列。
- 不允许平台管理员代登录普通用户。
- 不在用户管理中查看用户私有资产、会话、黑板、生成记录和提示词。
- 不定义后台 API 字段、数据库字段或审计字段。

## 后台 App Shell

```text
左侧后台导航
  概览
  账号与用户
  Skill
  模型
  Tool
  积分
  内容展示
  系统

顶部账号栏
  当前管理员
  退出登录

主内容区
  页面标题
  筛选 / 表格 / 表单
  Drawer / Modal
  状态反馈
```

规则：

- 后台默认日间主题，可支持夜间主题，但不使用 C 端沉浸式玻璃视觉。
- 表格页必须支持筛选、分页、加载、空态、错误态。
- 危险操作使用确认弹层，不只用 Toast。
- Secret、兑换码和用户隐私信息默认脱敏。

### 平台管理员账号

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/admin/admins` |
| 页面结构 | 管理员列表、新增管理员、停用确认 |

### 用户管理

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/admin/users` |
| 页面结构 | 筛选栏、用户表格、用户基础详情、启用/禁用确认 |
| 核心组件 | FilterBar、UserTable、UserDetailDrawer、StatusConfirm |
| 状态 | loading、empty、filtered_empty、enabled、disabled、confirming、permission_denied、error、success |
| 依赖 | API / RPC：用户查询、账号状态变更、审计日志 |

规则：

- 展示账号基础信息、身份状态、所属企业摘要、启用状态和时间信息。
- 禁用用户前必须填写操作原因并确认影响。
- 不展示用户私有资产、会话、黑板、生成记录、提示词、上传私有素材和创作内容。
- 不支持代登录，不支持直接修改用户密码。

### 系统 Skill 管理

| 项 | 设计 |

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/admin/skills/reviews` |

| 核心组件 | ReviewQueue、ReviewDetail、ReviewActionPanel |

## 通用状态与组件

| 状态 | 后台表现 |
| --- | --- |
| loading | 表格 Skeleton、按钮 loading、表单提交中。 |
| empty | 表格空态，提供创建、清除筛选或返回入口。 |
| error | 页面内 Alert，提供重试和 trace_id。 |
| success | Toast 或结果区，重要操作保留审计提示。 |
| confirming | 危险、停用、下架、导出、发放等动作必须确认。 |
| permission_denied | 普通用户访问后台或登录态异常时拒绝访问。 |

后台基础组件：

- DataTable / FilterBar / Pagination。
- Drawer / Modal / ConfirmModal。
- StatusBadge / RiskBadge / SecretInput。
- AuditHint / TraceIdCopy。
- EmptyState / ErrorState / Skeleton。

## 边界检查

- 平台后台不进入普通用户 Agent 工作台。
- 平台管理员账号不复用普通用户登录态。
- 用户管理不是内容审查后台，不展示用户私有创作内容。
- Secret、兑换码、用户隐私和内部链路必须脱敏。
- 后台字段以 API、RPC 和审计契约为准，不在设计文档中定义后端字段。

## 验收标准

- [ ] 后台页面覆盖管理员账号、用户管理、系统 Skill、Skill 审核、模型、Tool、积分、兑换码、精选作品和审计日志。
- [ ] 每个后台页面都有主任务、核心组件、状态和契约依赖。
- [ ] loading、empty、error、success 和 confirming 已覆盖。
- [ ] 后台不消费 AG-UI 流式事件，不混入普通用户创作入口。
- [ ] 敏感信息和用户私有创作内容展示边界明确。
