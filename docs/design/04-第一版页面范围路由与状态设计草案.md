## 背景

现有设计文档已经覆盖视觉基线、C 端页面模板、公共组件和统一 Agent 工作台。为了让后续前端开发工程师能够按页面拆分工作，还需要把 PRD 中的页面范围整理为第一版路由草案、页面组件、状态覆盖和契约依赖。

本文件是页面设计和需求映射输入。当前 PRD 套件已确认 Done，可进入需求映射矩阵、契约设计和正式代码开发；正式字段以 API、RPC 和 AG-UI 契约为准。

## 目标

- 补齐第一版站点信息架构的页面级落点。
- 给出普通用户端、公开页、企业空间和平台后台的路由草案。
- 明确每个页面的主任务、核心组件和必要状态。
- 标明哪些页面依赖 API、RPC 或 AG-UI 契约。
- 明确前端不得自行发明后端字段、权限字段、AG-UI payload 或 RPC 字段。

## 非目标

- 不定义最终前端技术选型。
- 不定义数据库字段、API 字段、RPC 字段或 AG-UI payload。
- 不做代码实现。
- 不设计 iOS / Android 原生端。
- 不把 Draft PRD 宣称为正式开发依据。

## 路由命名原则

| 原则 | 说明 |
| --- | --- |
| C 端与后台隔离 | 普通用户端使用 `/app`，公开页使用 `/explore`，平台后台使用 `/admin`。 |
| 工程目录隔离 | 用户端落在 `frontend/`，管理端落在 `admin_frontend/`。 |
| 用户端左右结构 | C 端桌面默认左侧 `AppSideNav` / `PublicSideNav` + 右侧 `PageContent`，不使用顶部横向主导航。 |
| 公开内容免登录 | 首页公开作品、精选作品和公开作品详情可匿名访问；个人化内容必须登录后展示。 |
| 当前空间不写死在路由 | 当前空间来自登录身份和 API 返回上下文，不通过前端拼接字段决定。 |
| 项目是创作容器 | 会话、资产、黑板和作品通过项目聚合；项目路由表达用户可见归属。 |
| 企业空间复用 C 端工作台 | 企业身份下仍进入同一个 Agent 工作台，企业管理使用独立二级页面。 |
| 详情优先 Drawer | 资产、Skill、通知等管理型详情优先用 Drawer，公开作品详情使用详情页。 |
| 路由为草案 | 最终路由由前端开发工程师结合框架约定确认。 |

## 全局状态覆盖

| 状态 | 页面范围 | 设计要求 |
| --- | --- | --- |
| loading | 所有页面 | 使用 Skeleton 或轻量加载，不出现长时间空白。 |
| empty | 列表、库、中心页 | 说明为空原因，并提供开始创作、上传、创建、清除筛选等入口。 |
| error | 所有页面 | 展示用户可理解原因和重试入口，技术细节默认不暴露。 |
| success | 表单、确认、保存 | 使用 Toast 或结果区域反馈，不改变已确认数据语义。 |
| streaming | Agent 工作台 | 增量渲染消息，保留停止入口，不让布局抖动。 |
| interrupt | Agent 工作台 | 用确认面板或补充输入承接 `interrupt.required`，展示影响范围。 |
| resume | Agent 工作台、会话恢复 | 断线重连时保留已有内容，补偿事件或使用快照恢复。 |
| permission_denied | 企业、后台、私有详情 | 不展示越权内容，说明当前身份和恢复路径。 |
| login_required | 首页公开内容、精选作品、所有私有入口 | 未登录访客触发创作、点赞、个人数据入口时弹出登录弹窗，保留当前意图。 |
| blocked | 生成、作品分享、上传文本 | 内容安全阻断时不进入扣费或公开分享。 |
| archived_readonly | 项目详情、工作台、项目资产和会话历史 | 项目归档后只允许查看历史、资产、黑板和作品；禁用继续创作、上传到项目、确认扣费和保存新资产。 |
| insufficient | 积分和生成确认 | 不展示确认按钮或禁用确认，并引导兑换或联系企业拥有者。 |
