## 当前设计结论

- 只做 Web 端，不做 iOS/Android 原生端。
- 支持日间主题和夜间主题。
- 使用弹性布局和容器断点，适配笔记本、标准桌面、宽屏和超宽屏。
- 用户端桌面采用左侧导航 + 右侧内容结构；未登录公开页和登录后 C 端保持同一骨架。
- 结构可参考 Flova，但 Dora-Agent 的导航命名和业务模块以本项目 PRD 为准。
- 首页是轻量入口，不承载完整工作台、黑板、资产库列表或后台内容。
- 首页公开作品和精选作品支持未登录浏览；创作、点赞和个人数据入口通过登录弹窗承接。
- 项目是用户创作目标容器；首页开始创作默认进入项目，工作台、资产、黑板和作品都需要能解释项目归属。
- 风格偏专业创作工具，兼顾内容作品展示，不做纯营销页。
- 后台采用独立管理端主题，简约白色主题、黑色元素点缀、高密度、表格化、可扫描，不复用 C 端暗色电影级、玻璃拟态和作品氛围视觉。
- C 端页面开发必须同时遵循视觉 Token、公共组件、页面模板和 Agent 工作台/A2UI 四类规范。
- 管理端页面开发必须同时遵循平台后台页面体验设计和管理端主题样式设计规范。
- 04-07 保留为第一版页面范围与需求映射输入；进入前端阶段前需要结合当前契约和实现再次复核。
