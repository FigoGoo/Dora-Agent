# 平台后台管理页草图

状态：archived
owner：产品与需求责任域
更新时间：2026-06-28
适用范围：平台后台 `/admin`、用户管理、Skill、模型、Tool、积分、兑换码、精选作品管理和审计日志低保真草图
相关代码路径：用户端 `frontend/**`；管理端 `admin_frontend/**`
相关契约：[C 端与后台业务 API 契约草案](../../contracts/api/C端与后台业务API契约草案.md)、[业务微服务 RPC 契约草案](../../contracts/rpc/业务微服务RPC契约草案.md)；后台不消费 AG-UI 流式事件

## 目标用户

- 平台管理员：配置平台能力、审核 Skill、发放积分、管理公开作品风险和查看审计日志。

## 主任务

用独立后台完成平台运营配置和审计追踪，不进入普通用户工作台，不查看用户私有创作内容。

## 页面区域

| 区域 | 作用 |
| --- | --- |
| 后台独立 Shell | 独立入口、独立登录态、独立导航。 |
| 左侧后台导航 | 概览、账号与用户、Skill、模型、Tool、积分、内容展示、系统。 |
| 筛选栏 | 支持按模块需要筛选，字段以后续契约为准。 |
| 表格区 | 高密度可扫描列表，支持分页、空态、错误态。 |
| Drawer | 查看基础详情，不展示敏感或私有内容。 |
| Confirm Modal | 禁用、下架、导出、发放、风险变更等高影响动作。 |
| 审计提示 | 关键操作成功或失败后展示审计记录提示。 |

## 关键组件

| 组件 | 用途 |
| --- | --- |
| AdminShell | 后台 App Shell。 |
| AdminLoginForm | 平台管理员独立登录。 |
| DataTable / FilterBar / Pagination | 管理端列表页基础结构。 |
| StatusBadge / RiskBadge | 状态和风险展示，不能只靠颜色。 |
| SecretInput | API Key 等敏感信息脱敏输入和展示。 |
| ConfirmModal | 停用、下架、导出、积分发放等确认。 |
| AuditHint / TraceIdCopy | 审计提示和 trace_id 复制。 |
| DetailDrawer | 用户基础信息、公开作品摘要、审计摘要等详情。 |

## 状态覆盖

| 状态 | 后台表现 |
| --- | --- |
| loading | 表格 Skeleton、按钮 loading、连通性测试 testing。 |
| empty | 无用户、无审核、无模型、无审计时展示空态和恢复入口。 |
| error | 页面内 Alert，展示可理解错误和 trace_id 摘要。 |
| success | Toast 或结果区，并提示关键操作已进入审计。 |
| streaming | 后台不消费 AG-UI 流式事件。 |
| interrupt | 后台高风险动作使用 ConfirmModal，不使用 AG-UI interrupt。 |
| resume | 后台不恢复 Agent run；表格重载使用普通 API 状态。 |

## AG-UI / API / RPC 依赖

| 依赖 | 说明 |
| --- | --- |
| API | 后台登录、列表查询、详情查询、审计查询、导出动作等。 |
| RPC | 用户启停、Skill 审核、模型/Tool 配置、积分发放、兑换码、公开作品下架等业务能力。 |
| AG-UI | 无。后台页面不消费 Agent 流式事件。 |

## 敏感信息边界

后台不得展示：

- API Key 明文。
- 完整兑换码列表进入审计日志。
- 用户私有资产、会话、黑板、生成记录、提示词或上传私有素材。
- 模型内部推理链路、系统提示词、供应商原始响应全文。
- 用户完整手机号、完整邮箱等敏感个人信息。

## 模块映射

| 模块 | 页面重点 |
| --- | --- |
| 用户管理 | 基础账号状态、启用/禁用确认和原因。 |
| Skill 审核 | 只读审核内容，通过/拒绝，意见非必填。 |
| 模型供应商 | API Key 脱敏、连通性测试、启停。 |
| 模型管理 | 类型、默认模型、积分单价、停用默认模型前确认。 |
| Tool 管理 | 启停、白名单、风险、超时、影响提示。 |
| 积分 / 兑换码 | 手动发放、有效期、批量导出确认。 |
| 精选作品管理 | 公开作品查询和下架，不删除源资产。 |
| 审计日志 | 操作摘要、结果、时间、trace_id。 |

## 待确认项

- 后台首页摘要是否只展示待审核和最近审计，还是需要模块健康状态。
- 审计日志筛选维度、trace_id 展示规则和保留范围需契约确认。
- 兑换码导出方式和导出结果交付方式需产品与后端确认。

## 边界声明

本草图不定义后台字段或审计字段。平台后台是运营配置入口，不是内容审查后台，也不是普通用户代操作入口。
