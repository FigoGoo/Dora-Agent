## 背景

企业空间第一版不是复杂组织管理系统，核心价值是让多个企业成员共享企业积分和企业 Skill。企业身份下的创作仍使用统一 Agent 工作台，通过当前空间切换改变 Skill 池、积分账户、产物归属和历史可见范围。

本设计草案只补企业空间页面体验，不新增企业资产库、企业会话管理、企业黑板管理或成员创作记录后台。

## 设计目标

- 让用户清楚识别当前处于企业空间。
- 让企业拥有者完成邀请成员、移除成员、转让拥有者、管理企业积分和企业 Skill。
- 让企业成员在企业身份下使用统一 Agent 工作台和企业积分。
- 明确企业拥有者不额外查看成员资产、会话、任务、黑板和生成记录。
- 让身份切换的影响可见，避免串用个人空间和企业空间数据。

## 非目标

- 不做部门、岗位、多企业归属和复杂 RBAC。
- 不做企业资产、会话、黑板、任务或生成记录管理后台。
- 不允许企业成员主动退出企业。
- 不让企业拥有者查看成员私有创作内容。
- 不定义企业 API、RPC 或数据库字段。

## 信息架构

```text
个人中心
  身份与空间
    个人身份
    企业身份

企业空间
  企业概览
  企业成员
  企业 Skill
  企业积分
  本人企业消耗

统一 Agent 工作台
  当前空间 = 企业空间
  Skill 池 = 系统 Skill + 企业 Skill + 个人 Skill
  积分账户 = 企业积分账户
  资产可见 = 当前用户本人创建的企业空间资产
```

## 入口与空间识别

| 位置 | 设计要求 |
| --- | --- |
| 右侧 ContextHeader | 展示企业名称和“企业空间”标识，积分入口指向企业积分。 |
| 工作台 Composer | 在扣费确认前展示当前空间，企业空间必须显示企业名称。 |
| 资产 / 会话 / 作品 | 只展示当前登录用户本人在企业空间创建的数据。 |
| 个人中心 | 身份切换必须提示将重新登录目标身份并刷新空间上下文。 |
| 错误页 | 权限不足时展示当前身份与空间，不展示越权内容。 |

## 页面设计

### 企业注册

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/enterprise/register` |
| 适用角色 | 普通用户 |
| 页面结构 | 企业信息表单、创建说明、提交结果 |
| 关键组件 | EnterpriseRegisterForm、FormError、SuccessResult |
| 关键状态 | loading、duplicate_enterprise、error、success |
| 契约依赖 | API / RPC：企业创建、创建者成为拥有者 |

交互规则：

- 创建成功后，创建者成为企业拥有者。
- 创建成功不自动暴露成员资产入口。
- 失败时保留用户已填内容，展示可理解错误。

### 企业概览

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/app/enterprise` |
| 适用角色 | 企业拥有者、企业成员 |
| 页面结构 | 企业摘要、当前权限、快捷入口、企业积分摘要 |
| 关键组件 | EnterpriseSummary、RoleBadge、QuickLinks、CreditSummary |
| 关键状态 | loading、empty、permission_denied、error、success |
| 契约依赖 | API：当前企业身份、企业摘要、积分摘要 |

交互规则：

- 企业拥有者看到成员、企业 Skill、企业积分入口。
- 企业成员只看到企业工作台和本人企业消耗入口。
- 不展示成员资产、成员会话或成员创作记录入口。

### 企业成员管理

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/app/enterprise/members` |
| 适用角色 | 企业拥有者 |
| 页面结构 | 成员列表、邀请成员、邀请状态、移除操作 |
| 关键组件 | MemberTable、InviteDialog、InviteStatusBadge、RemoveConfirm |
| 关键状态 | loading、empty、invite_pending、confirming、permission_denied、error、success |
| 契约依赖 | API / RPC：邀请、邀请状态、移除成员 |

交互规则：

- 邀请未注册用户时展示邀请状态，不在前端创建成员关系。
- 移除成员前必须确认影响。
- 不能移除企业拥有者。
- 企业成员访问该页展示无权限状态。

### 拥有者转让

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/app/enterprise/owner-transfer` |
| 适用角色 | 企业拥有者 |
| 页面结构 | 成员选择、影响说明、确认转让 |
| 关键组件 | MemberSelect、ImpactNotice、ConfirmModal |
| 关键状态 | loading、selecting_member、confirming、failed、success |
| 契约依赖 | API / RPC：拥有者转让 |

交互规则：

- 只能选择当前企业成员。
- 确认弹层说明原拥有者会变为普通企业成员。
- 转让成功后刷新当前权限和企业入口。

### 企业 Skill

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/app/enterprise/skills` |
| 适用角色 | 企业拥有者创建和管理；企业成员只读使用 |
| 页面结构 | 企业 Skill 列表、审核状态、创建入口、详情 Drawer |
| 关键组件 | SkillTabs、SkillCard、StatusBadge、SkillDrawer、SkillBuilderEntry |
| 关键状态 | loading、empty、draft、testing、pending_review、rejected、published、deprecated、permission_denied |
| 契约依赖 | API / RPC：Skill CRUD、测试、提交审核、版本 |

交互规则：

- 企业成员不能创建企业 Skill。
- 企业 Skill 审核结果通过站内信通知创建者。
- Draft、PendingReview、Rejected、Deprecated 不参与 Agent 路由。
- 已发布企业 Skill 对企业成员全部可见，不做部门或成员级范围。

### 企业积分

| 项 | 设计 |
| --- | --- |
| 路由草案 | `/app/enterprise/credits` |
| 适用角色 | 企业拥有者 |
| 页面结构 | 企业余额、即将过期、兑换码、流水、成员消耗明细 |
| 关键组件 | CreditOverview、ExpiringCreditNotice、RedeemForm、LedgerList、MemberUsageList |
| 关键状态 | loading、empty、insufficient、redeem_success、redeem_failed、permission_denied、error |
| 契约依赖 | API / RPC：企业积分、兑换码、流水、成员消耗 |

交互规则：

- 企业兑换码只能由企业拥有者兑换到当前企业积分账户。
- 企业空间生成时只扣企业积分，不自动切换个人积分。
- 企业成员看不到企业总账和其他成员消耗。

## 企业身份下的 Agent 工作台

| 区域 | 企业空间规则 |
| --- | --- |
| 当前空间 | 顶部和确认卡展示企业名称。 |
| Skill 池 | 系统 Skill、企业 Skill、个人 Skill。 |
| 模型选择 | 仍在聊天输入框内完成，当前对话生效。 |
| 积分确认 | 显示企业积分账户、预计积分、当前余额和即将过期积分。 |
| 资产视图 | 只展示当前用户本人有权访问的企业空间资产。 |
| 黑板视图 | 展示当前会话内 Skill 输出元素和草稿态创作过程。 |
| 断线恢复 | 使用 AG-UI 续传或快照恢复，不假定任务失败。 |

## 状态覆盖

| 状态 | 企业空间表现 |
| --- | --- |
| loading | 企业身份、成员、积分、Skill 列表加载时使用 Skeleton。 |
| empty | 无成员、无企业 Skill、无消耗明细时提供对应主操作。 |
| error | 显示可理解原因和重试入口，不暴露后端细节。 |
| success | 邀请、移除、转让、兑换、保存成功使用 Toast 或结果区。 |
| streaming | 企业工作台内同 C 端 Agent 流式输出。 |
| interrupt | 企业工作台中扣费、高风险 Tool、业务写入确认。 |
| resume | 企业工作台断线重连和历史会话恢复。 |
| permission_denied | 非拥有者访问成员、转让、企业总积分时展示无权限。 |

## 边界检查

- 企业拥有者不额外查看成员资产、产物、会话、任务、黑板或生成记录。
- 企业成员不能创建企业 Skill，不能兑换企业码，不能主动退出企业。
- 当前空间事实由后端返回的登录身份和权限上下文决定。
- 前端不以本地缓存自行判断企业权限，不自行发明企业字段。
- 企业空间业务事实由业务微服务维护；Agent 微服务只保存 Agent Runtime 数据和创作过程引用。

## 验收标准

- [ ] 企业空间页面覆盖企业注册、概览、成员、转让、Skill、积分和本人消耗。
- [ ] 企业身份下统一 Agent 工作台复用 C 端工作台，并明确企业空间上下文。
- [ ] loading、empty、error、success、streaming、interrupt、resume 已覆盖。
- [ ] 企业拥有者和企业成员的页面权限差异清楚。
- [ ] 明确 API、RPC、AG-UI 契约未完成前不能发明后端字段。
