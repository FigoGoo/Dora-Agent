## 背景

用户端采用暗色电影级创作控制台风格，服务于内容创作、作品浏览和 Agent 工作台沉浸体验。平台后台的核心任务不同：管理员需要高频筛选、扫描表格、编辑配置、处理审核、查看审计和执行高风险操作。

因此管理端必须有独立主题样式，不沿用用户端的玻璃拟态、暗色大视觉、作品氛围图、霓虹能量点和营销式首屏。

## 设计目标

- 建立独立于 C 端的管理端视觉语言：清晰、克制、高密度、可扫描。
- 默认使用浅色主题，支持长时间配置、审核和表格处理。
- 为后台表格、筛选、表单、Drawer、Modal、审计和风险操作提供稳定组件规则。
- 让高风险操作、权限边界、脱敏信息和审计提示在视觉上足够明确。
- 保证未来 `admin_frontend/**` 实现时可以直接落到语义 token 和组件规范。

## 非目标

- 不复用 C 端暗色电影级创作主题。
- 不定义后台 API 字段、RPC 字段、数据库字段或审计日志字段。
- 不设计运营大屏、营销落地页、社区内容流或沉浸式作品展示。
- 不为第一版后台定义复杂 RBAC 可视化。
- 不要求管理端与用户端拥有相同背景、卡片、按钮和动效风格。

## 风格定位

关键词：

- Operational
- Dense but calm
- Table first
- Audit friendly
- Low decoration
- Clear hierarchy
- Risk visible

一句话原则：

> 管理端不是创作空间，而是平台配置、审核、排障和审计的工作台。

视觉上应呈现：

- 浅灰页面背景、白色内容面板、清晰边框和稳定间距。
- 表格和筛选是主视觉，不用大图、英雄区或装饰背景抢占注意力。
- 彩色只表达状态、风险和主要操作，不作为页面氛围铺色。
- 信息密度高于 C 端，但文字、行高和点击目标仍保持可用。
- 危险操作在按钮、确认弹层、结果反馈和审计提示中形成一致视觉链路。

## 主题策略

### 默认主题

管理端默认使用浅色主题：

- 适合长时间阅读和表格处理。
- 与 C 端默认暗色创作氛围区分。
- 便于展示启用、停用、失败、风险、审计等状态。

### 夜间主题

管理端可支持夜间主题，但夜间主题只是低亮度办公模式，不是 C 端沉浸式视觉：

- 不使用黑底作品墙。
- 不使用玻璃拟态 Skill 卡片。
- 不使用大面积渐变、光晕、影像背景。
- 保持同样的信息密度、表格边框、状态色和表单层级。

## 布局骨架

### Admin App Shell

```text
AdminSideNav 232px，可折叠为 64px
  ↓
AdminTopBar 56px
  ↓
AdminPage
  PageHeader
  FilterBar / ActionBar
  DataTable / Form / DetailPanel
  Drawer / Modal / Toast
```

核心尺寸：

| 区域 | 参数 |
| --- | --- |
| 左侧导航展开宽度 | `232px` |
| 左侧导航折叠宽度 | `64px` |
| 顶部栏高度 | `56px` |
| 页面水平内边距 | `24px`，窄屏 `16px` |
| 页面垂直内边距 | `20px` |
| PageHeader 高度 | 内容自适应，通常 `52px` 到 `72px` |
| FilterBar 最小高度 | `48px` |
| 表格行高 | 默认 `44px`，高信息行 `52px` |
| Drawer 宽度 | `480px` 到 `720px` |
| Modal 宽度 | `420px`、`560px`、`720px` 三档 |

布局规则：

- 后台不使用营销式 Hero。
- 页面标题区不放超大字号口号。
- 表格页优先使用 `PageHeader + FilterBar + DataTable + Pagination`。
- 表单页优先使用单列或双列栅格，不使用 C 端卡片瀑布流。
- 宽屏不把表单无限拉宽；表单主体最大宽度 `960px`。
- 表格允许横向滚动和固定关键列，不强行压缩所有列。

颜色规则：

- 管理端主色使用蓝色系，表达工具和配置，不使用 C 端 AI Lime 作为主色。
- 状态色必须语义化：成功绿色、警告琥珀、危险红色、信息青色。
- 主按钮一页最多一个主要操作；次要操作使用默认按钮或文本按钮。
- 表格行 hover 只改变背景，不增加阴影或动画位移。
- 禁止大面积渐变、霓虹光晕、彩色玻璃和背景图装饰。

## 字体与密度

字体栈：

```css
--admin-font:
  Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
  "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", Arial, sans-serif;
```

字号规则：

| 场景 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 页面标题 | `20px` | `600` | `28px` |
| 二级标题 | `16px` | `600` | `24px` |
| 正文 | `14px` | `400` | `22px` |
| 表格正文 | `13px` | `400` | `20px` |
| 表头 | `12px` | `600` | `18px` |
| 表单 Label | `13px` | `500` | `18px` |
| 辅助说明 | `12px` | `400` | `18px` |
| 状态 Badge | `12px` | `500` | `18px` |

密度规则：

- 使用 `4px` 基础间距网格。
- 常用间距：`4px`、`8px`、`12px`、`16px`、`20px`、`24px`。
- 后台卡片圆角不超过 `8px`。
- 表格、筛选栏、表单控件不使用大圆角胶囊。
- 字体不使用 viewport width 缩放。
- 字距保持 `letter-spacing: 0`。

## 组件规范

### 导航

- 左侧导航展示后台模块，不混入 C 端创作入口。
- 当前模块使用 `--admin-surface-selected` 背景和 `--admin-primary` 文字或图标。
- 二级导航最多两层；超过两层时在页面内使用 Tabs 或侧向二级菜单。
- 平台后台入口不得出现在 C 端普通导航中。

### 页面头部

- `PageHeader` 包含标题、简短说明、主操作按钮。
- 主操作按钮放在右侧，危险操作不得作为默认主按钮。
- 面包屑用于深层详情页，不在一级列表页强制展示。
- 不使用大面积封面图、主视觉图或欢迎语横幅。

### 筛选栏

- 筛选栏优先横向排列：搜索、状态、类型、时间、操作者。
- 筛选项过多时使用“更多筛选”Popover 或 Drawer。
- 已选筛选条件必须可见并可一键清除。
- 筛选变更不得丢失分页和排序语义，具体行为以后续 API 契约为准。

### 表格

- 表格是后台默认信息承载组件。
- 表头使用弱背景和底部分隔线。
- 默认分页每页 `10` 条；可选 `20`、`50`，上限以后续 API 契约为准。
- 支持 loading、empty、filtered_empty、error、success 状态。
- 关键列可固定，低优先级列可隐藏到详情 Drawer。
- 批量操作必须有明确选中数量和二次确认。
- 危险行操作使用文字按钮或更多菜单，不放成高亮主按钮。
- 表格内不展示密钥、完整兑换码、用户私有内容、完整提示词或供应商原始响应。

### 表单

- 普通配置表单使用垂直 Label。
- 密集编辑表单可使用两列布局，但字段组必须清楚。
- 必填、错误、禁用、只读、提交中状态必须可区分。
- Secret 输入默认脱敏，允许重新填写，不允许明文回显。
- 高风险配置保存前使用确认弹层说明影响范围。
- 表单错误贴近字段展示；全局错误放在表单顶部 Alert。

### Drawer 与 Modal

- 列表详情优先使用 Drawer，避免离开当前筛选上下文。
- 新增、编辑和高风险确认使用 Modal 或独立表单页。
- Drawer 内可展示审计摘要、状态历史和只读详情。
- 危险确认 Modal 必须包含操作对象、影响说明、原因输入或确认动作。

### 状态与 Badge

| 状态 | 视觉 |
| --- | --- |
| 启用 / 成功 | 绿色文字 + 浅绿背景 |
| 停用 / 失败 | 红色文字 + 浅红背景 |
| 待审核 / 待处理 | 琥珀文字 + 浅黄背景 |
| 测试中 / 处理中 | 青色文字 + 浅青背景 |
| 草稿 / 未配置 | 灰色文字 + 浅灰背景 |
| 高风险 | 红色边框或红色 Badge，不使用绿色或蓝色弱化 |

### 反馈

- Toast 只用于轻量成功或非阻塞提示。
- 高风险操作结果必须在 Modal、Drawer 或页面 Alert 中保留可读结果。
- 审计相关操作成功后提示“已记录审计”，但不展示内部审计字段结构。
- 失败提示应给出用户可执行动作，例如重试、修改配置、联系管理员。

## 禁用项

管理端禁止使用：

- C 端暗色电影级背景和作品氛围图。
- 玻璃拟态 Skill 卡片、霓虹光效、彩色光斑和大面积渐变。
- 营销式 Hero、超大标题、情绪化插画首屏。
- 卡片瀑布流承载后台主列表。
- 仅靠 Toast 完成危险操作确认。
- 用颜色表达权限或风险但不配文字说明。
- 展示 Secret 明文、完整兑换码、内部成本明细给无关页面。
- 展示用户私有资产、会话、黑板、生成记录或提示词。

## 响应式规则

| 断点 | 规则 |
| --- | --- |
| `>= 1280px` | 左侧导航展开，表格完整展示，详情使用右侧 Drawer |
| `1024px-1279px` | 左侧导航可折叠，表格隐藏低优先级列 |
| `768px-1023px` | 筛选栏换行，Drawer 接近全宽，表格横向滚动 |
| `< 768px` | 仅保证基础查看、筛选和关键操作；复杂配置建议桌面端完成 |

后台移动端不是第一主场景，但必须保证：

- 不出现文本和按钮重叠。
- 表单字段可读可点。
- 危险确认完整展示。
- 横向滚动区域有明确边界。

## 风险

- 如果管理端复用 C 端主题，会导致表格密度不足、操作层级不清和审核效率下降。
- 如果管理端过度追求高密度，会损失可访问性和错误识别能力。
- 如果状态色不语义化，管理员可能误判停用、失败、待审核和高风险操作。
- 如果 Secret、兑换码和用户隐私信息没有统一视觉规则，后续实现容易出现安全展示缺陷。

## 验收标准

- 后台默认浅色主题，与 C 端暗色创作主题明显区分。
- 列表页可以用 `PageHeader + FilterBar + DataTable + Pagination` 落地。
- 表格、表单、Drawer、Modal、Badge、Alert 均有可实现的主题 token。
- 高风险操作具备危险色、确认弹层、原因或影响说明、结果反馈和审计提示。
- Secret、兑换码、用户私有内容和供应商敏感响应默认不明文展示。
- 管理端页面不使用 C 端玻璃拟态、作品背景、霓虹光效和营销式 Hero。
