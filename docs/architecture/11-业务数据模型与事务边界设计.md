## 背景

业务系统微服务需要保存账户、企业、项目、模型、Tool、Skill、积分、资产、通知、作品和审计等业务事实。本文在 PRD 评审阶段给出数据模型和事务边界草案，供后续 RPC 契约、SQL 脚本和服务端测试使用。

本轮不创建 migration，不写 Go 代码。所有表名、字段和索引仍是候选设计，正式落地前需要按功能点拆分到迭代 SQL，并经过主控 Codex 和对应契约评审。

## 目标

- 明确业务数据库候选表和领域模型状态。
- 明确权限、租户、空间和资源访问边界。
- 明确关键写操作事务边界、回滚条件和幂等策略。
- 明确无数据库级外键约束、列表分页、批量查询和索引设计原则。
- 输出 contract test、集成测试和数据库验证映射。

## 非目标

- 不保存 Agent Runtime 数据，例如 session、run、message、event、tool_call、interrupt、memory、blackboard draft。
- 不保存业务系统无关的 Eino 编排状态或 AG-UI 事件。
- 不设计支付、订单、发票、退款、评论、收藏、关注、排行榜或推荐算法。
- 不创建数据库级外键约束，不使用 GORM AutoMigrate。

## 数据边界原则

- 业务数据库保存业务事实，Agent 领域数据库保存 Agent Runtime 数据，两者至少独立 schema，生产环境优先独立 database 或实例。
- 所有业务表按场景带 `tenant_id`、`space_id`、`project_id`、`owner_user_id`、`enterprise_id` 或等价隔离字段。
- 数据库 schema 一律不添加 `FOREIGN KEY`、`REFERENCES` 或等价外键约束。
- 跨表一致性通过 application/domain 校验、事务、唯一约束、普通索引、幂等记录和测试保证。
- 列表查询必须分页，默认 `page_size=10`，建议最大 `page_size=50`。
- 避免在 `for` 循环中逐条查询数据库或 RPC；使用批量查询、`IN` 查询、JOIN、预加载、批量 RPC 或批次缓存。
- DTO 不暴露表结构，业务表字段不能直接成为 RPC 对象。

## 权限、租户和空间边界

| 主体 | 可访问范围 | 写权限 | 特殊限制 |
| --- | --- | --- | --- |
| 个人用户 | 个人空间项目、资产、作品、积分、个人 Skill、通知 | 创建项目、创建个人 Skill、上传资产、创建作品、兑换个人码 | 不能访问企业空间资源 |
| 企业拥有者 | 企业空间本人项目、资产和作品、企业 Skill、企业积分、成员管理 | 邀请、移除成员、转让拥有者、创建企业 Skill、兑换企业码 | 不额外查看成员项目、资产、会话、黑板或创作过程 |
| 企业成员 | 企业空间本人项目、资产、作品、本人消耗、企业 Skill 使用 | 使用企业积分创作、创建本人项目和作品 | 不能创建企业 Skill，不能主动退出企业 |
| 平台管理员 | 平台后台配置、用户基础信息、公开作品管理、审计日志 | 管理模型、Tool、系统 Skill、审核、积分发放、兑换码、下架公开作品 | 不查看用户私有资产、会话、黑板、提示词或私有素材 |
| 未登录访客 | 首页公开作品、Shared 状态公开作品列表和详情、公开展示信息 | 无 | 不能点赞，不能创作，不能访问个人项目、私有资产、私有 TOS 原始链接 |

权限校验必须在 business application/domain 层执行，repository 只做数据访问。

## 领域状态机

| 领域 | 状态 | 状态说明 |
| --- | --- | --- |
| 用户 | active、disabled | disabled 不能登录个人或企业空间，不自动删除公开作品 |
| 企业邀请 | pending、accepted、expired、cancelled | 过期或取消后不能加入企业 |
| 企业成员 | active、removed | removed 后失去企业空间访问权限 |
| 模型供应商连通性 | untested、connected、failed | failed 不启用相关模型 |
| 模型 | enabled、disabled | 默认模型停用前必须切换 |
| Tool | enabled、disabled | disabled 不能绑定或执行 |
| Skill | draft、pending_review、published、rejected、deprecated | 只有 published 参与 Agent 路由 |
| 项目 | active、archived | archived 不出现在默认最近项目；是否允许继续创作待确认 |
| 积分批次 | active、exhausted、expired | 扣减优先消耗最早过期 active 批次 |
| 积分冻结 | frozen、charged、released | charged 可使用冻结时已过期的积分，released 不延长有效期 |
| 兑换码 | active、redeemed、expired、disabled | 第一版一次性兑换 |
| 资产 | uploading、available、save_failed | available 才可预览、下载、引用、创建作品 |
| 通知 | unread、read | 通知不是业务事实来源 |
| 作品 | private、shared、taken_down | shared 生成公开快照，taken_down 公开链接不可访问 |

## 关键事务边界

| 事务 | 原子写入内容 | 事务外动作 | 回滚或补偿 |
| --- | --- | --- | --- |
| 个人注册 | `business_users`、个人 `business_spaces`、个人 `credit_accounts`、幂等记录 | 发送欢迎通知如后续需要 | 任一写入失败则全部回滚 |
| 企业创建 | `enterprises`、企业 `business_spaces`、拥有者 `enterprise_members`、企业 `credit_accounts`、审计 | 无 | 任一写入失败则全部回滚 |
| 创建邀请 | `enterprise_invites`、审计 | 外部通知第一版不做 | 重复邀请按业务规则返回已有或冲突 |
| 接受邀请 | 邀请状态、`enterprise_members`、审计 | 登录态刷新由调用方处理 | 用户已有 active 企业身份则拒绝 |
| 移除成员 | 成员状态变为 removed、审计 | Agent 会话访问失效由后续查询权限体现 | 不能移除 owner，失败全部回滚 |
| 转让拥有者 | 原 owner role 更新、新 owner role 更新、审计 | 当前用户权限刷新由调用方处理 | 目标不是 active 成员则拒绝 |
| 用户禁用启用 | 用户状态、审计 | 现有 session 失效策略待确认 | 审计写失败时状态变更应回滚 |
| API Key 更新 | 加密凭证、供应商状态、审计 | 连通性测试应在事务外执行后保存结果 | 不记录密钥明文 |
| 默认模型切换 | `default_models`、审计 | 无 | 新模型不可用则拒绝 |
| Tool 策略变更 | `tool_policies`、`tool_whitelist_rules`、审计 | 影响 Skill 的提示可异步计算 | 审计失败回滚 |
| Skill 提交审核 | skill/version 状态、测试样例校验结果、审计 | 无 | 测试样例不足拒绝 |
| Skill 审核 | review record、skill 状态、审计、通知写入或通知 outbox | 站内信补偿发送 | 通知写入失败不回滚审核事实，但必须有失败审计或补偿记录 |
| 项目创建 | `projects`、幂等记录、审计可选 | 自动标题可由 Agent 在事务前给出建议 | 空间或用户无权限则拒绝 |
| 项目归档 / 恢复 | project status、审计、幂等记录 | 资产和作品不删除 | 无权限或状态冲突则拒绝 |
| 积分发放 | credit batch、ledger、审计、幂等记录 | 无 | 任一写入失败回滚 |
| 兑换码创建 | code hash、批次元数据、审计 | 批量导出明文只在创建响应或安全导出流程提供 | 审计不得保存完整明文 |
| 兑换码兑换 | redeem status、redemption、credit batch、ledger、审计 | 无 | code 已用、过期、停用则拒绝 |
| 积分冻结 | 按最早过期批次减少 available、增加 frozen、freeze record、ledger、幂等记录 | Agent 执行生成在事务外 | 余额不足拒绝，不部分冻结 |
| 资产保存并扣费 | asset、project_assets、asset_elements、storage object、freeze charged、批次扣减、ledger、幂等记录 | TOS 文件上传应先完成或使用已存在 object key | 资产写入失败则不扣费；扣费失败则资产不可变为 available |
| 释放冻结 | freeze released、批次 frozen 减少、available 按原过期时间恢复、ledger、幂等记录 | 无 | 已 charged 的 freeze 不能释放 |
| 上传素材登记 | asset、project_assets、storage object、asset_elements 可选、审计 | 文件上传和病毒/格式校验在事务前完成 | 校验失败不创建资产 |
| 创建作品 | works、project_works、work_assets、审计 | 安全评估由 Agent 或调用方先完成 | 源资产不可用、无权限或不属于当前项目则拒绝 |
| 分享作品 | public snapshot、work shared 状态、审计 | 公开 CDN 或代理缓存刷新可事务后执行 | 快照失败保持 private |
| 取消分享 | work private 状态、snapshot disabled、审计 | 公开缓存失效事务后执行 | 不删除源资产 |
| 点赞取消点赞 | work_likes 状态、计数更新或查询聚合、幂等记录 | 无 | 未登录拒绝，重复点赞幂等 |
| 平台下架 | work taken_down 状态、snapshot disabled、审计 | 公开缓存失效事务后执行 | 不删除源资产 |

## 幂等键策略

| 场景 | 幂等 scope | 重复请求行为 | 冲突判定 |
| --- | --- | --- | --- |
| 注册、企业创建、邀请、接受邀请 | `identity` / `enterprise` | 返回同一业务结果或当前状态 | 同键不同请求摘要返回 `IDEMPOTENCY_CONFLICT` |
| 项目创建、归档、恢复 | `project:<space_id>` / `project:<project_id>` | 返回同一项目或状态变更结果 | 标题、目标状态或空间不同为冲突 |
| 管理员配置写入 | `admin:<action>` | 返回同一配置变更结果 | 操作对象或目标状态不同为冲突 |
| Skill 提交、审核、发布、回滚 | `skill:<skill_id>` | 返回同一状态变更结果 | 版本号或审核结果不同为冲突 |
| 积分冻结、释放、扣减 | `credit:<account_id>` | 返回同一 freeze 或 ledger 结果 | 金额、资源、模型、数量不同为冲突 |
| 兑换码兑换 | `redeem:<code_hash>` | 已兑换返回既有 redemption 结果 | 目标账户不同为冲突 |
| 资产保存并扣费 | `asset_commit:<run_id>:<artifact_id>` | 返回同一 asset 和 charge 结果 | object key、freeze_id 或金额不同为冲突 |
| 作品分享、取消分享、下架 | `work:<work_id>` | 返回同一分享状态或下架状态 | 目标状态、快照内容摘要不同为冲突 |
| 通知创建、已读 | `notification:<source_event>` | 避免重复通知或重复已读 | recipient 或 related resource 不同为冲突 |

`idempotency_records` 建议保存 `request_hash`、`result_ref`、`status`、`expires_at`。保留时间按业务风险确定，积分、资产扣费、兑换码等资金等价操作保留更长时间。

## 查询和分页设计

| 查询 | 分页和排序 | 批量策略 |
| --- | --- | --- |
| 用户列表、管理员列表、审计日志 | 默认 10 条，按创建或操作时间倒序 | 管理后台筛选条件建组合索引 |
| Skill 列表和路由池 | 默认 10 条或按 Agent 路由需要限制返回 | 用 `scope_type IN (...)` 和 `status=published` 批量加载 |
| 项目列表 | 默认 10 条，按 `last_activity_at` 倒序 | 企业空间第一版按 `owner_user_id` 过滤本人项目 |
| 模型列表 | 按类型和状态过滤，默认 10 条 | 一次性加载同类型可用模型和默认模型 |
| Tool 可绑定列表 | 按空间、企业、用户等级、状态过滤 | 批量加载 Tool 和 policy，避免逐 Tool 查询白名单 |
| 积分流水和成员消耗 | 默认 10 条，按时间倒序 | 企业成员消耗按 user_id 聚合或分页查询 |
| 资产列表 | 默认 10 条，按创建时间倒序 | 引用校验使用 `BatchCheckAssetAccess` |
| 通知列表 | 默认 10 条，按时间倒序 | 未读数使用索引条件统计 |
| 精选作品列表 | 默认 10 条，按分享时间或筛选条件 | 标签和分类筛选走普通索引或后续搜索能力 |

## 审计与脱敏

| 场景 | 必须审计 | 脱敏规则 |
| --- | --- | --- |
| 管理员登录、失败、退出 | 是 | 登录失败只记录账号脱敏摘要 |
| API Key 新增编辑和连通性测试 | 是 | 只记录 key fingerprint 和结果，不记录明文 |
| 兑换码创建、导出、停用 | 是 | 审计不保存完整兑换码列表 |
| 用户禁用启用 | 是 | 记录原因和状态变化，不记录用户私密内容 |
| Skill 审核 | 是 | 审核意见可记录，但不记录系统提示词或敏感策略 |
| 积分发放、冻结、扣减、释放 | 是 | 记录金额、资源摘要、trace_id、idempotency_key |
| 项目创建、归档、恢复、重命名 | 归档和恢复必须审计，创建和重命名建议审计 | 记录项目 ID、空间、状态变化，不记录完整提示词 |
| 作品分享、取消分享、下架 | 是 | 记录公开字段摘要，不记录源提示词、黑板、私有素材 |

## 数据库约束和索引原则

- 不使用数据库级外键；所有关联字段只作为普通业务字段、查询过滤条件或审计信息。
- 使用唯一约束表达业务唯一性，例如一个 active 用户最多一个企业、兑换码 hash 一次性、同一用户对同一作品一次点赞。
- 积分账户、冻结、扣减需要在事务内使用行级锁或等价并发控制，避免余额超扣。
- 价格、Tool 策略、Skill 版本、公开快照等影响历史结果的配置应保存快照或版本引用。
- 大表索引按查询路径创建，避免无依据索引；审计日志、积分流水、资产、作品按时间和 owner/space 组合索引。
- 后续 migration 必须成对提供 `up.sql` 和 `down.sql`，并说明锁风险和回滚方式。

## 风险与待确认事项

- 当前 PRD 套件已确认 Done。后续由主控 Codex 明确契约、开发计划和 subagent 需求映射矩阵。
- 平台后台访问路径需要确认。如果后台直接调用业务服务，需要补充 direct HTTP 或 BFF 边界；如果经 Agent API 适配层，需要确保不混入 Agent Runtime。
- 项目封面、自动标题和归档后是否允许继续创作需要产品确认。
- 内容安全结果的业务保存口径未最终定义。建议业务写操作只接收安全通过证据摘要，不保存模型推理链路或安全策略细节。
- 模型供应商 API Key 加密方案、密钥轮换和本地开发占位策略需要在安全设计或配置规范中补充。
- 积分冻结并发控制需要实现阶段验证 PostgreSQL 锁策略和事务隔离级别。
- 公开作品快照的媒体代理、签名 URL 有效期和缓存失效策略需要前端、Agent 和业务服务进一步对齐。
- 第一版不做资产删除，后续如果补删除和回收站，需要重新设计资产、作品、积分和公开快照的影响边界。
