## 可稳定引用的原则

以下原则已经在 PRD、规范、测试用例和 active 契约之间形成一致口径，可以作为后续补契约和开发前置约束：

- RPC 表达业务能力，不暴露数据库表 CRUD。
- 写操作必须有幂等键；高风险写操作需要 preview / confirm 或等价确认语义。
- AG-UI canonical 字段使用 `type`，`event_type` 仅允许兼容读取。
- Agent DB 只保存 Agent Runtime 数据，不保存业务事实。
- 业务事实通过业务服务和业务数据库维护，Agent 服务只能保存业务引用和脱敏摘要。
- SQL 不添加数据库级外键约束，不使用 `FOREIGN KEY` 或 `REFERENCES`。
- 项目 `archived` 后只读，不允许创建 run、继续生成、上传到该项目、保存新资产或确认扣费。
- TOS object key、上传签名和公共 URL 由业务服务生成或授权返回，前端和 Agent 不自造 object key，不接触 AK/SK。

## active 条件

契约文档保持 `active` 需持续满足：

- 关闭本文档记录的 P0 缺口，或在契约正文中明确延期且不影响本次实现范围。
- 每个新增或变更的 RPC/API/AG-UI/数据表都有字段级定义、错误码、权限、幂等、分页或状态机说明。
- 对应代码事实源已明确，例如 Thrift、OpenAPI、AG-UI schema、migration 或 fixture 路径。
- 对应测试入口已明确，至少能定位到 contract test、服务级测试或手工验证报告。
- 契约 README 的状态、正文状态和目录索引一致。
