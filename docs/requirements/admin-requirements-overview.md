> 更新日期：2026-07-13

> 关联文档：[共通业务规则与验收基线](common-requirements-baseline.md)、[用户端需求总览](user-requirements-overview.md)、[服务端需求总览](server-requirements-overview.md)

## 1. 文档目的

本文定义 Dora 生产版管理端的目标用户、职责范围、菜单结构、运营与治理能力、敏感操作控制和验收口径，为管理端交互设计、服务端领域设计、权限设计、测试、运营、财务、风控和审计提供共同需求基线。

管理端仅面向平台内部人员，不属于本期排除的企业用户功能。

## 2. 需求背景与管理目标

Dora 用户端包含个人创作、结构化 Skill 创建、多 Skill 调用、按次积分消费、发布者收益、精选作品和平台公告全局弹窗。生产版管理端必须覆盖这些业务的完整治理闭环，而不能仅作为数据查询工具。

核心目标：

1. 保证用户、发布者、Skill、公开 Tool 和精选作品可审核、可追溯、可暂停和可下架。
2. 保证 Agent 运行、异步生成、积分消费和发布者收益可查询、可恢复、可对账。
3. 为运营、审核、财务、客服、安全和风控提供职责隔离的工作台。
4. 使平台公告全局弹窗可配置、可预览、可排期、可撤回。
5. 对所有敏感查看和操作执行最小权限控制与完整审计。

## 3. 管理端用户与角色

| 角色 | 主要职责 |
| --- | --- |
| 超级管理员 | 管理管理员账号、角色、权限和平台级功能配置。 |
| 平台运营 | 运营 Skill 市场、精选作品、分类、标签、推荐位、活动和全局公告。 |
| Skill 审核员 | 审核 Skill 首次发布、版本更新和违规整改。 |
| Tool/AI 管理员 | 管理公开 Tool、模型、Provider、路由、配额和成本。 |
| 内容安全审核员 | 审核 Skill、精选作品、用户输入和生成内容，处理版权与隐私问题。 |
| 风控人员 | 处理账号滥用、刷调用、自买、虚假点赞、支付和收益回收风险。 |
| 客服人员 | 处理工单、计费申诉、投诉、举报和补偿申请。 |
| 财务人员 | 管理积分账务、结算批次、收益回收、打款和对账。 |
| 数据分析人员 | 查看经营、市场、作品、Agent 运行和财务报表。 |
| 审计人员 | 只读查看敏感操作、财务记录、数据导出和审计日志。 |

同一管理员可以具有多个角色，但人工调账、收益冻结、收益回收审批和打款等高风险操作应支持职责分离和双人复核。

## 4. 管理范围与约束

### 4.1 本期范围

- 个人用户和个人发布者管理。
- Skill 创建、测试、审核、版本、市场和违规治理。
- 用户在 Skill 表单中可通过 `@` 引用的公开 Tool 目录管理。
- Project、Agent、Skill 调用和 AIGC Worker 运行治理。
- 积分、订单、发布者收益积分、结算、回收、打款和对账。
- 精选作品审核、推荐、标签、创作过程审核、点赞风控和下架。
- 平台公告全局弹窗配置。
- 内容安全、版权、风控、客服、数据和系统权限。

### 4.2 本期不包含

- 企业、组织、团队、企业 SSO、组织钱包和企业私有市场管理。
- 移动端管理后台。
- 用户消息中心、站内信、私信和定向消息发送。
- 精选作品评论及评论审核。
- 用户上传任意可执行代码或任意自定义 Tool 的审核体系。
- 管理员绕过状态机直接修改账本、运行结果或用户业务事实。

## 6. 核心管理需求

### 6.1 经营概览

经营概览需要按权限汇总：

- 新增、活跃、留存用户和发布者数量。
- Skill 数量、审核量、实际调用量、成功率和市场转化。
- 快速创作成功率、Project 数量、Agent/Worker 运行质量。
- 精选作品发布量、浏览、点赞和 Skill 导流。
- 积分充值、扣费、Graph Tool 汇总、账务冲正、平台收入和 Provider 成本。
- 发布者收益积分待结算、可用、冻结、五折回收、冲正和对账差异。
- 安全、风控、客服和异常任务待办。

Dashboard 数据只能来自业务事实和已定义的聚合口径，不允许人工修改统计结果。

### 6.2 用户与发布者管理

- 查询个人用户资料、账号状态、实名认证、积分状态、Project、创建的 Skill、风险和工单记录。
- 管理个人发布者的发布权限、收益权限、收益回收权限和收款信息状态。
- 支持限制登录、限制 Agent 使用、限制 Skill 发布、冻结收益和封禁账号。
- 支持注销申请审核、法定保留数据处理和业务数据匿名化/删除状态查看。
- 管理员不得直接改写 Project、对话、作品和账务事实；处置必须使用受控命令并填写原因。

建议状态：

```text
用户：正常 → 受限 → 禁用 → 注销中 → 已注销
发布者：未开通 → 待认证 → 正常 → 受限/冻结 → 已关闭
```

### 6.3 Skill 治理

Skill 审核页必须展示创建者提交的完整结构化配置：

- Skill 名称、简介、分类、标签和市场展示内容。
- 输入与输出。
- Skill 调用规则。
- 流程规划。
- 素材分析。
- 故事板设计。
- 媒体生成。
- 提示词写法。
- 视频剪辑。
- 通过 `@` 引用的公开 Tool、权限和费用。
- 平台计费配置、测试扣费结果、发布者收益规则、示例作品和版本差异；审核员和发布者均不得覆盖平台可信价格。

审核规则：

1. Skill 只能引用当前允许创建者使用的公开 Tool。
2. 提交审核时冻结 Skill 配置、Tool、权限、计费和安全检查快照。
3. 已发布 Skill 的修改生成新版本，审核通过前不影响线上版本。
4. 审核员只能通过、驳回、暂停、恢复或下架，不得替创建者改写 Skill。
5. 驳回、暂停和下架必须填写原因；创建者在“我的 Skill”查看。
6. Tool 被暂停或废弃时，管理端必须列出受影响 Skill，并阻止不安全的新调用。
7. Skill 的调用规则需要检查过宽、冲突、循环选择和误导行为。

```text
Skill 版本：草稿 → 测试中 → 待审核 → 审核中 → 已发布/已驳回
已发布 → 已暂停 → 恢复发布/已下架
```

### 6.4 公开 Tool 与 AI 能力

公开 Tool 是 Skill 创建页面通过 `@` 引用的受控平台能力。每个 Tool 至少配置：

- 名称、能力说明、适用场景和示例。
- 输入输出 Schema。
- 权限范围、数据使用说明和风险等级。
- 单次费用或费用估算规则。
- 超时、并发、频率和最大调用限制。
- 依赖模型、Provider 或 Connector。
- 版本、兼容策略和当前状态。
- 是否对 Skill 创建者开放。

```text
Tool：草稿 → 测试中 → 已开放 → 已暂停 → 已废弃
```

- Tool 密钥和平台凭证必须受控保存，不得在普通管理页明文展示。
- Tool 改名不得改变已有 Skill 中的稳定引用。
- Tool 改价或升级应形成版本或生效快照，不能静默改写历史调用账单。
- 本期不审核用户上传的任意代码或任意自定义 Tool；该能力需要独立安全方案。

### 6.5 Skill 市场运营

- 管理分类、标签、推荐位、榜单、搜索词和活动。
- 配置 Skill 市场曝光规则，但不得绕过 Skill 发布与安全状态。
- 运营人员不得直接修改发布者的 Skill 指令、Tool 引用或定价历史。
- Skill 市场必须明确“Skill 本身不另收固定完成费”与模型、媒体、公开 Tool 和 Graph Tool 实际可计费执行积分的区别。
- 所有推荐、榜单干预和搜索降权操作需要记录原因和操作人。

### 6.6 “我的 Skill”Dashboard 数据治理

管理端需要提供用户端 Skill Dashboard 的数据源及口径查询：

- 曝光量、详情访问量、启用次数。
- 实际 Skill 调用次数和独立使用用户数。
- 成功、失败、取消、部分成功和账务冲正次数。
- 用户扣费积分、冲正积分和平均单次积分。
- 发布者待结算、可用、冻结、冲正和已回收收益积分。
- 关联精选作品及其浏览和点赞贡献。

调用量必须来自实际 Skill 调用事实，而不是启用次数、Agent 读取调用规则次数或技术重试次数。异常流量可以从经营指标和收益中剔除，但原始事实及剔除原因必须保留审计记录。

### 6.7 精选作品运营

精选作品可以由作者主动提交或平台从已获授权的作品中发起邀请，公开前必须完成审核：

- 作品内容、封面、标题和标签。
- 作者公开资料。
- 使用的 Skill 及其版本。
- 作者确认公开的创作过程快照。
- 版权、隐私、内容安全和冒用风险。

“查看创作过程”只能展示安全的公开投影，例如作者允许公开的 Prompt、故事板版本、选定资产和高层 Skill 步骤。管理端必须拦截：

- 模型隐藏推理过程和系统提示词。
- Skill 未公开配置与 Tool 内部参数。
- 私有 Chat、密钥、Provider 请求和内部日志。
- 未授权、已删除或存在版权风险的资产。
- 内部费用、风控和用户隐私数据。

运营能力包括：

- 审核通过、驳回、暂停、下架和恢复。
- 配置推荐位、排序权重和作品标签。
- 预览用户端实际公开内容。
- 查看点赞明细、异常流量和风控结论。
- 处理作者撤回、用户举报、版权投诉和安全处置。

点赞采用账号级唯一关系，异常点赞可从展示计数中剔除。本期不存在评论对象，因此不建设评论审核或评论管理。

```text
精选作品：草稿 → 待审核 → 审核中 → 已发布/已驳回
已发布 → 已暂停/已下架
```

### 6.8 Agent 运行中心

支持按照用户、Project、Skill、Skill 版本、Skill 调用、Agent Turn、Operation、Batch 和 Job 查询运行链路：

- 已启用 Skill 与实际选择的 Skill。
- Skill 选择原因和实际调用序号。
- ChatModel、公开 Tool 和 Graph Tool 运行状态。
- 图片、视频、音频和装配任务状态。
- 逐项积分扣费、Graph Tool 汇总、账务冲正和发布者收益积分关联。
- 错误、重试、取消、降级、等待用户和部分成功状态。

人工取消、重试、恢复或补偿必须具有独立权限、稳定幂等标识和审计记录。管理员不得修改用户对话、伪造 Skill 调用、伪造运行结果或绕过财务结算。

### 6.9 积分与订单

- 管理充值商品、支付订单、支付回调、积分账户和不可变积分流水。
- 管理模型、媒体、公开 Tool 和 Graph Tool 的版本化积分配置；发布者、管理员、客户端和模型不得在运行请求中覆盖可信价格。
- 查询每个 Skill Invocation、Graph Tool、Operation/Batch/Job 的扣费明细、终态汇总、收益归属和账务冲正。
- 平台不提供充值退款或消费退款。客服可以处理计费申诉并解释账单；只有重复扣费、错扣或无对应可计费执行等平台账务差错才能发起冲正。
- 人工调账需要填写原因，并支持双人复核和额度限制。
- 冲正、人工调账和补偿必须关联工单或风险事件，使用稳定幂等键并由不可变账本追加记录。
- 客户端、管理员或模型不得直接提交可信最终积分值，最终积分由冻结计费配置和受信模型/Provider/Tool 回执确定。

```text
充值订单：待支付 → 已支付/已关闭/支付异常核对
用户积分：可用 → 扣费中 → 已扣费 → 已冲正（仅账务差错）
```

### 6.10 发布者收益、结算与回收

- 用户消费积分账户与发布者收益积分账户分离。
- 每次实际 Skill Invocation 冻结 Skill 版本、收益规则版本、归属发布者和风险规则；收益积分不得直接等同于用户总扣费或 Provider 成本。
- 合格调用生成待结算收益积分；结算周期和风控通过后转为可用，申诉、风险和异常流量可以触发冻结或收益冲正。
- 平台按 `50%` 系数回收可用收益积分；不足最小结算单位的余数留在收益账户。
- 管理结算批次、回收申请、风控审核、财务审核、打款、失败重试和财务对账。
- 发布者自测、自买、刷量、虚假用户和风险流量不得制造可回收收益。
- 发布者将可用收益积分转换为消费积分和申请收益回收是两个独立操作。

```text
发布者收益积分：待结算 → 可用/冻结/已冲正
                      可用 → 冻结/已回收/已冲正
                      冻结 → 可用/已冲正
收益回收申请：已提交 → 风控审核 → 财务审核 → 打款中 → 已完成
                                             └→ 已驳回/打款失败
```

管理端、用户端和服务端必须使用[共通状态机与用户投影](common-requirements-baseline.md#4-统一状态机与用户投影)，不得在报表或人工处置中创建含义不同的同名状态。

### 6.11 内容安全、风控与客服

- 内容安全覆盖 Skill 表单、Tool 引用、用户输入、AIGC 输出、精选作品和公开创作过程。
- 风控覆盖批量注册、资源滥用、Skill 刷量、发布者自买、虚假点赞、支付盗刷、冲正滥用和异常收益回收。
- 客服处理计费申诉、Skill/作品投诉、侵权举报、发布者申诉和补偿。
- 工单详情承载沟通记录，不扩展为用户消息中心。
- 账务冲正、补偿、解封、恢复发布和解除冻结必须关联工单、风险事件或审核单。

### 6.12 平台公告全局弹窗

公告仅通过用户端全局弹窗展示，管理端不提供站内信或用户定向消息能力。

管理员可以配置：

- 标题、正文、图片和平台内跳转链接。
- 生效时间、失效时间和时区。
- 展示优先级。
- 每个公告版本仅一次、每次登录或平台允许的频率规则。
- 是否允许关闭。
- 草稿、预览、排期、发布、撤回和紧急停用。

同一时刻多个有效公告必须按确定的优先级和发布时间排序。用户展示/关闭记录只用于频控和统计，不形成收件箱、未读红点或个人消息。

```text
公告：草稿 → 已排期 → 展示中 → 已结束
展示中 → 已撤回
```

### 6.13 系统、数据与审计

- 管理管理员账号、内部角色和资源级权限。
- 管理功能开关、平台参数、协议、隐私政策、安全策略和数据保留周期。
- 提供用户、Skill、精选作品、Agent 质量、积分、收益、冲正和客诉报表。
- 数据导出必须进行权限校验、脱敏、用途记录和审计。
- 系统监控告警属于内部运维能力，不属于用户消息产品。

## 7. 审计要求

以下操作必须写入不可篡改审计记录：

- 用户封禁、解封、注销和权限限制。
- Skill 审核、暂停、下架、恢复和版本回滚。
- Tool 开放、暂停、废弃、改价和版本更新。
- 精选作品审核、推荐、隐藏、恢复和下架。
- Agent 人工取消、重试、恢复和补偿。
- 积分调账、账务冲正、收益冻结、收益冲正和追回。
- 收益回收审核、打款及失败处理。
- 公告发布、撤回、紧急替换和展示规则修改。
- 管理员、角色和权限变更。
- 高敏数据查看和数据导出。

审计记录至少包含管理员、角色、目标对象、目标版本、操作前后值、操作原因、关联工单、请求标识、时间和来源地址。

## 8. 数据与非功能要求

- 管理端所有列表支持稳定分页、组合筛选、排序和权限范围内导出；普通列表查询 `P95 ≤ 1s`。
- 聚合指标需要标注口径、统计周期、数据延迟和是否剔除风险流量。
- 敏感字段默认脱敏，查看明文需要更高权限和二次审计。
- 人工处置必须幂等，刷新、超时和重试不得重复冲正、打款、下架或补偿。
- 账务、审核、Skill 版本和 Tool 版本必须支持历史追溯。
- 后台故障不得影响已提交的运行、结算和公告撤回最终生效。
- 重要任务积压、账务不平、Provider 故障和异常点赞需要进入内部监控告警。
- 高敏查看和数据导出 `100%` 写入审计；账务不平在发现后 `5min` 内告警，并阻止相关对象重复结算。
- 超过 `10,000` 行或预计执行超过 `10s` 的导出必须转为异步任务，不得占用同步请求；导出文件使用短期授权并记录下载审计。

其余可用性、恢复、容量、数据保留和安全指标遵循[共通非功能基线](common-requirements-baseline.md#6-v1-量化非功能基线)。

## 9. 核心验收标准

以下用例继承[共通可执行验收规范与 Evidence 要求](common-requirements-baseline.md#7-可执行验收规范)，缺少规定证据时不得判定通过。

| ID | Given / When | Then |
| --- | --- | --- |
| ADM-RBAC-001 | Given 两名管理员具有不同角色与数据范围；When 访问同一菜单、对象和敏感字段 | 只返回授权资源；越权访问失败并记录审计，不可通过导出绕过。 |
| ADM-SKILL-001 | Given Skill 处于任一合法审核状态；When 提交、通过、驳回、暂停、恢复或下架 | 仅允许共通状态机中的合法迁移，原因、版本、操作人和审核快照可回放。 |
| ADM-SKILL-002 | Given Skill 提交新版本；When 审核员打开详情 | 完整展示结构化字段、`@Tool` 稳定引用、权限、平台计费配置、测试扣费结果和版本差异。 |
| ADM-TOOL-001 | Given Tool 已暂停、废弃或改价；When 管理员发布变更 | 新 Skill/新运行按版本失败关闭，运行中执行按冻结版本处理，受影响 Skill 可完整定位。 |
| ADM-DASH-001 | Given 存在启用、规则读取、技术重试和真实 Invocation；When 查询 Dashboard | 调用量只等于真实 Invocation，扣费等于去重明细之和，冲正和收益积分可逐笔追踪。 |
| ADM-BILL-001 | Given 同一扣费幂等键被重复处理；When 查询账本和 Graph Tool 汇总 | 只存在一笔正常扣费，汇总不追加完成费；平台不提供失败、取消或申诉退款入口。 |
| ADM-BILL-002 | Given 财务确认重复扣费或错扣；When 双人复核通过冲正 | 原账本保持不变，新增唯一冲正和关联收益冲正，重复操作返回原结果。 |
| ADM-EARN-001 | Given 发布者有合格可用收益积分；When 审批五折回收并触发重复回调 | 只扣减一次收益积分并只打款一次，50% 系数、余数、税费和状态均可对账。 |
| ADM-WORK-001 | Given 作者提交公开快照；When 审核员预览和发布 | 用户端展示内容与管理端最终预览一致，私有 Chat、隐藏指令、Tool 参数和未授权资产不可见。 |
| ADM-LIKE-001 | Given 同一账号并发点赞且部分流量被判定异常；When 查看运营数据 | 关系真源最多一个，原始事实保留，风险流量只从公开计数和经营指标中剔除。 |
| ADM-ANN-001 | Given 多个公告排期且其中一个被撤回；When 用户进入页面 | 按优先级和发布时间展示；撤回公告在缓存失效 SLA 内停止展示并有完整审计。 |
| ADM-AGENT-001 | Given Agent/Worker 处于运行、等待用户、核对或 Dead 状态；When 管理员取消、重试、恢复或人工重放 | 操作具有权限、幂等键、原因和审计；终态任务通过新记录重放，不原地重置。 |
| ADM-NOSOCIAL-001 | Given 管理员查看全部菜单和接口；When 搜索精选作品相关能力 | 不存在评论对象、评论审核、评论通知或面向用户的消息发送能力。 |

## 10. 待确认参数

1. 管理员角色的最终拆分及双人复核额度。
2. Skill 审核时限、申诉时限和自动安全检查规则。
3. Tool 版本兼容、暂停后的运行中 Skill 处理策略。
4. 精选作品采用纯人工审核还是安全机审加人工复核。
5. 公告默认展示频率、允许连续展示的最大公告数和紧急公告关闭规则。
6. 发布者收益积分生成规则、结算周期、最低回收积分、积分兑付币种/法币面值、税费、打款渠道和财务对账周期；五折回收系数已经确认。
