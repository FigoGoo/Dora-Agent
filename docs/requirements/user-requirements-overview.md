## 1. 文档目的

本文定义 Dora 生产版桌面 Web 用户端的目标用户、产品范围、菜单结构、核心流程、业务规则与验收口径，作为后续交互设计、视觉设计、服务端设计、测试和运营验收的共同需求基线。

本文是需求总览，不约束具体前端框架、页面路由、组件实现或接口协议。

## 2. 需求背景与产品目标

Dora 是一个由 Skill 驱动的 AIGC Agent 平台。用户通过自然语言发起创作，在同一个 Project 中使用 Chat、A2UI、故事板和资产完成图片、视频、音频等内容生产；个人创作者可以通过结构化表单创建 Skill、发布到 Skill 市场，并按照有效 Skill 调用及平台收益规则获得收益积分。

本期用户端目标：

1. 让用户从一句提示词快速进入可持续编辑的创作工作台。
2. 让用户能够在同一 Project 中启用一个或多个 Skill，由 AI 根据 Skill 调用规则选择实际执行的 Skill。
3. 让普通用户可以用结构化表单创建 Skill，不要求编写代码。
4. 让 Skill 发布者能够管理已发布与未发布 Skill，并查看调用、积分和收益数据。
5. 通过精选作品展示结果、所用 Skill 和安全处理后的创作过程，促进 Skill 发现与复用。
6. 按平台模型与能力计费配置，在实际生成或可计费执行开始时扣除积分，并为发布者提供独立的收益积分与五折回收能力。

## 3. 目标用户

| 用户 | 能力与目标 |
| --- | --- |
| 游客 | 浏览首页、Skill 市场、Skill 详情、精选作品及全局公告；登录后才能创作、点赞或创建 Skill。 |
| 个人用户 | 快速创作、管理 Project 和资产、启用 Skill、充值并消费积分、提交工单。 |
| Skill 创建者 | 使用结构化表单创建和测试 Skill，管理已发布与未发布 Skill。 |
| 个人发布者 | 发布 Skill、查看 Skill Dashboard、获得收益积分并申请五折回收。 |

Skill 创建者、发布者和 Skill 使用者共用同一个个人账号，不形成企业、组织或团队角色体系。建议允许登录用户先创建草稿，在提交市场审核或申请收益回收前完成实名认证。

## 4. 产品范围与约束

### 4.1 本期范围

- 仅提供桌面 Web。
- 仅支持个人账号和个人发布者。
- 支持快速创作、Project、资产中心、Skill 市场、我的 Skill、精选作品、积分钱包、帮助反馈和账号设置。
- 支持 Chat、A2UI、动态故事板和异步媒体生成。
- 支持同一 Project 启用多个 Skill，按照实际 Skill 调用次数计费。
- Skill 使用结构化表单创建，通过 `@` 引用平台向用户开放的 Tool。
- 平台公告仅以全局弹窗展示。
- 精选作品支持点赞和查看公开创作过程，不支持评论。

### 4.2 本期不包含

- 移动端 App、移动 Web 和 PWA。
- 企业、组织、团队、成员、项目协作、企业 RBAC、SSO、组织钱包和私有市场。
- 消息中心、站内信、私信、业务通知聚合和公告列表页。
- 精选作品评论、回复、评论通知、关注关系和好友关系。
- 用户上传或运行任意代码、二进制和不受控脚本型 Skill。
- 用户自建未审核 Tool、模型 Provider 或支付渠道。

## 6. 核心功能需求

### 6.1 快速创作

#### 6.1.1 功能目的

降低首次创作成本，让用户从一句提示词直接进入完整工作台，同时保证每次创作都有可管理、可恢复的 `project_id`。

#### 6.1.2 入口与对话框

点击“快速创作”后必须先显示轻量对话框：

- 提供一个多行提示词输入框。
- 提示词允许为空。
- 提供“开始创作”和“取消”操作。
- 点击“开始创作”后进入提交状态，防止重复点击。
- 创建失败时保留已输入内容并允许重试，不得留下用户不可见的重复 Project。

#### 6.1.3 创建规则

```text
点击快速创作
→ 展示提示词对话框
→ 用户输入提示词或留空
→ 创建新的 project_id 和默认 Session
→ 打开新的独立创作页面
→ 有提示词：作为第一条用户消息自动开始 Agent Turn
→ 无提示词：展示空白工作台，等待用户在右侧 Chat 输入
```

补充规则：

- 空提示词或纯空白字符不得触发 Agent、Skill 调用或积分扣费。
- 有提示词时，第一条消息只能提交一次；网络重试不得重复创建 Project、消息或计费。
- Project 名称可根据首条提示词生成临时名称；空提示词使用默认名称，用户后续可修改。
- Project 创建成功后，即使 Agent 暂时未启动，用户也必须能够凭 `project_id` 进入并恢复工作台。

#### 6.1.4 创作页面布局

创作页面采用一个 Project 对应一个独立工作区：

```text
创作页面
├─ 顶部：Project 名称、已启用 Skill、运行状态、积分摘要
├─ 左侧工作区
│  ├─ 故事板
│  └─ 资产
└─ 右侧工作区
   ├─ Chat 消息流
   ├─ A2UI 表单/确认卡
   ├─ Skill 调用状态
   └─ Prompt 输入框
```

- 故事板与资产属于同一 Project，不新建独立数据空间。
- 不需要故事板的创作允许显示空状态或隐藏故事板内容，但不能影响 Chat 和资产使用。
- Chat 只展示用户可理解的 Skill/Graph Tool 高层运行状态，不重复展示底层每个 Job、内部日志或 Provider 参数。
- 用户可以在 Project 中启用或停用多个 Skill；启用 Skill 本身不收费。
- AI 实际选择并执行 Skill 时，需要在工作台展示 Skill 名称、Graph Tool 运行状态、本次已扣积分和费用汇总。
- Prompt 编辑、资产绑定、候选确认和指定目标重新生成等确定性操作可以在左侧直接完成，不要求重新发起一轮自由 Agent 推理。

### 6.2 项目中心

Project 是长期保存的创作容器，包含 Session、对话、已启用 Skill、Skill 调用、故事板、资产、生成任务、版本、交付物和费用汇总。

项目中心必须支持：

- 按状态查看 Project。
- 按名称、创建时间和更新时间搜索或排序。
- 继续创作、重命名、归档、恢复和移入回收站。
- 查看创作历史、使用过的 Skill、积分消耗和最终交付物。
- 从有权且满足发布条件的 Project 提交精选作品。
- 默认仅项目所有者可访问，不提供团队共享或协同编辑。

### 6.3 资产中心

原“素材中心”统一更名为“资产中心”，用户界面统一使用“资产”一词。

资产中心管理用户上传和 AI 生成的图片、视频、音频、文本、结构化数据及最终交付物，支持：

- 类型、来源、Project、Skill、时间和状态筛选。
- 图片/视频/音频预览及必要的元数据查看。
- 下载、收藏、复用、重命名、删除和回收站恢复。
- 查看资产来源 Project、Skill、Skill 版本和生成时间。
- 区分候选资产、当前采用资产、已被新版本替代资产和最终交付物。
- 默认私有访问；只有用户主动提交并通过审核的精选作品公开快照可以对外展示。

### 6.4 Skill 市场

Skill 市场用于发现、比较、使用和创建 Skill。

Skill 详情至少展示：

- Skill 名称、简介、分类、标签、发布者和当前版本。
- 适用场景、输入要求、输出结果和示例作品。
- 支持的故事板、资产、媒体类型和交互方式。
- 需要使用的公开 Tool、权限和数据使用说明。
- 平台计费配置下的预计积分范围、可能使用的模型/媒体/公开 Tool 费用及最大积分提示；发布者不能自行覆盖平台价格。
- 调用次数、成功率等经平台确认可公开的数据。
- 收藏、启用/使用和投诉举报入口。

“收藏与最近使用”归入 Skill 市场，不再使用“我的 Skill”表示消费者收藏。

### 6.5 创建 Skill

#### 6.5.1 产品形态

“Skill Studio”不再作为用户端一级菜单。Skill 市场提供“创建 Skill”子功能，点击后打开独立的 Skill 创建页面。

本期 Skill 以结构化表单表示，不提供代码编辑器，不允许上传任意可执行代码或不受控脚本。

#### 6.5.2 结构化表单

| 表单字段 | 需求说明 |
| --- | --- |
| Skill 名称 | Skill 的公开名称，必须清晰、可识别且不得重复冒用他人品牌。 |
| 简介、分类与标签 | 帮助用户理解适用场景，并用于市场搜索和筛选。 |
| 输入与输出 | 描述 Skill 接收的文字/图片/视频/音频等输入及预期输出。 |
| Skill 调用规则 | 当开启多个 Skill 时，告诉 AI 在什么情况下应该调用这个 Skill。 |
| 流程规划 | 告诉 AI 按什么顺序推进、步骤之间有什么依赖，以及应该怎样和用户交互。 |
| 素材分析 | 告诉 AI 看完素材后要产出什么，例如提取分镜、整理人物描述或做截帧等基础处理。 |
| 故事板设计 | 告诉 AI 怎样编写故事板，包括应包含哪些元素、如何设计镜头，以及如何描述画面和镜头语言。 |
| 媒体生成 | 告诉 AI 怎样生成图片、视频和音频，包括使用哪个模型、参考哪些资产，以及如何确定输出设置。 |
| 提示词写法 | 告诉 AI 生成媒体时提示词应该怎样写，包括图片提示词、视频提示词和提升效果的技巧。 |
| 视频剪辑 | 告诉 AI 剪辑时要注意什么，包括如何裁剪资产、怎样对齐音视频，以及如何保证成片连贯。 |
| 示例与开场提示 | 提供用户可直接使用的示例输入和预期效果。 |
| 市场展示 | 封面、详情介绍、示例作品、版权声明和用户可见的使用说明。 |
| 计费信息 | 查看平台为相关模型、媒体、公开 Tool 和 Graph Tool 配置的预计用户积分、费用上限及发布者收益规则；发布者不能自行定价。 |

“流程规划”至“视频剪辑”均应作为独立结构化字段存在。对于确实不适用的字段，可选择“不适用”并填写原因，不能使用一个无结构的大文本框替代全部字段。

#### 6.5.3 `@Tool` 引用

上述指令字段支持输入 `@` 引用 Tool：

1. 输入 `@` 后展示当前用户可引用的公开 Tool 列表，支持按名称和能力搜索。
2. Tool 列表展示名称、能力说明、适用场景、权限、费用提示和状态。
3. 选中 Tool 后插入可识别的引用标记；保存时必须关联稳定 Tool 标识，不能只保存显示名称。
4. 只能引用平台明确向 Skill 创建者开放且当前可用的 Tool。
5. 未知、无权限、已暂停或已废弃 Tool 必须给出字段级错误，并阻止提交发布。
6. Tool 的密钥、内部参数和平台凭证不得展示给 Skill 创建者。
7. Skill 测试时应明确提示 Tool 授权范围和可能产生的积分成本。

#### 6.5.4 创建与发布流程

```text
创建 Skill
→ 填写结构化表单
→ 自动保存草稿
→ 表单与 @Tool 校验
→ 测试/预览
→ 提交审核
→ 审核通过后发布到 Skill 市场
→ 后续修改生成新版本并重新审核
```

- 测试调用不得计入公开调用量或产生发布者收益积分；测试中实际开始的模型或生成执行仍按平台计费配置扣除测试账号积分。
- 已发布版本不得原地覆盖；编辑必须创建新版本。
- 提交审核时，应向创建者展示将要公开的内容、Tool 权限和计费信息。
- 驳回、暂停或下架原因必须在“我的 Skill”对应详情中查看，不依赖消息中心。

### 6.6 我的 Skill

原“发布者中心”调整为一级菜单“我的 Skill”，用于管理自己创建和发布的 Skill，不再承载消费者收藏。

分组以“当前是否存在可公开使用的线上版本”为准：存在有效线上版本的 Skill 归入“已发布”；不存在有效线上版本的草稿、测试、首次审核、驳回、暂停或下架 Skill 归入“未发布”。已有线上版本且新版本正在审核时仍归入“已发布”，并单独标识更新审核状态。

#### 6.6.1 已发布的 Skill

- 展示当前公开版本、状态、发布时间和市场链接。
- 支持查看详情、创建新版本、主动暂停或申请下架。
- 展示当前 Tool 依赖和不可用风险。
- 查看关联精选作品及用户使用情况。

#### 6.6.2 未发布的 Skill

包含草稿、测试中、待审核、审核中、已驳回、已暂停和已下架等状态，并支持：

- 继续编辑、复制、删除草稿。
- 查看测试结果、审核进度和驳回原因。
- 修改后重新提交审核。
- 对违规处置或审核结果发起申诉。

#### 6.6.3 Skill Dashboard

Dashboard 支持总览和单个 Skill 视图，并支持时间范围与 Skill 版本筛选。指标至少包括：

- 市场曝光量、详情访问量和启用次数。
- 实际 Skill 调用次数、独立用户数、成功率、失败率和取消率。
- 消耗积分、账务冲正积分和平均单次积分。
- 发布者待结算收益积分、可用收益积分、冻结收益积分、已回收收益积分和已冲正收益积分。
- 关联精选作品数量及其浏览、点赞贡献。
- 版本、审核、暂停、下架和违规记录。

发布者收益积分、结算、收款账户和五折回收入口统一放在 Skill Dashboard 的收益区域。消费积分与发布者收益积分必须在界面和账务上区分。

### 6.7 多 Skill 启用与调用

- 同一 Project 可以启用一个或多个已发布 Skill。
- Skill 调用规则用于告诉 AI 在什么情况下选择该 Skill。
- 启用、停用、展示或读取 Skill 规则不扣除积分。
- AI 实际选择并开始执行某个 Skill 时，产生一次 Skill Invocation；积分按该 Invocation 下实际开始的模型、媒体、公开 Tool 或 Graph Tool 可计费执行逐项扣除。
- 同一轮确实执行多个 Skill 时，可以产生多笔独立 Skill Invocation、扣费明细和收益归属记录。
- 每个 Graph Tool 到达业务终态后汇总其去重后的扣费明细，不额外收取 Tool 完成费；异步 Graph Tool 返回“已接受”时费用汇总可以尚未完成。
- 同一次业务执行的技术重试、恢复、Checkpoint/Receipt 重放和异步媒体回流不得重复扣费；用户主动重新生成属于新业务操作，需要重新扣费。
- 达到用户设置或平台规定的积分上限时，应在新的可计费执行开始前暂停并要求用户正式确认，不得静默超额消费。

### 6.8 精选作品

#### 6.8.1 功能目的

展示高质量作品及其所用 Skill 和可理解的创作过程，帮助用户发现 Skill 和学习创作方法，但不建设评论型社交社区。

#### 6.8.2 作品列表与详情

精选作品至少展示：

- 作品封面、标题和作品内容。
- 标签。
- 作者公开名称与头像。
- 使用的一个或多个 Skill，并可进入对应 Skill 详情。
- 点赞数及当前用户点赞状态。
- 查看创作过程入口。

支持按标签、作品类型和所用 Skill 浏览或筛选。游客可以浏览；点赞前必须登录。

作品所用 Skill 必须从对应 Project 的实际 Skill 调用记录生成，作者不能添加未实际调用的 Skill。

#### 6.8.3 点赞

- 同一账号对同一作品最多存在一个有效点赞。
- 支持点赞和取消点赞。
- 重复点击、网络重试和并发请求不得重复累计。
- 异常点赞可能被平台风控剔除。
- 点赞不产生私信、消息通知或关注关系。

#### 6.8.4 查看创作过程

“创作过程”展示作者提交并确认过的公开快照，可以包含：

- 作者同意公开的初始提示词和关键修改说明。
- 选定的故事板版本和关键镜头。
- 公开资产的演进和最终采用结果。
- 实际使用的 Skill、Skill 版本和高层创作步骤。

不得展示：

- 完整私有 Chat 和作者未选择公开的内容。
- 系统提示词、Skill 隐藏指令、模型隐藏推理过程。
- Tool 密钥、内部参数、Provider 请求和内部日志。
- 未获授权的上传资产、已删除资产和财务/风控信息。

作者提交精选作品前必须预览并确认公开快照。Project 后续修改不得自动改变已发布作品；更新公开内容需要形成新快照并重新审核。

#### 6.8.5 发布与审核

- 作者从自己有权访问的 Project 或最终交付物提交精选作品。
- 提交时选择封面、标题、标签和允许公开的创作过程；系统自动附加实际调用的 Skill 及版本。
- 发布前进行平台审核、内容安全和版权检查。
- 作者可以撤回；平台可以因违规、版权或安全问题下架。
- 本期不提供评论入口、评论数量、回复、`@用户` 或评论管理。

### 6.9 平台公告全局弹窗

平台公告不设置用户端菜单、公告列表或消息收件箱，统一由应用外壳以全局弹窗展示。

- 用户进入任意用户端页面时，都可以展示当前有效公告。
- 支持标题、正文、图片和平台内链接。
- 支持仅展示一次、每次登录展示或按平台配置频率展示。
- 多个有效公告按平台优先级顺序展示。
- 用户关闭后按该公告版本记录展示状态；公告形成新版本时可以重新展示。
- 已撤回或过期公告不得继续弹出。
- 公告查看记录只用于展示频控和统计，不形成未读红点或消息中心。
- Skill 审核、任务、收益回收/打款和工单状态仍在各自业务页面查看。

### 6.10 积分钱包与发布者收益

消费积分不使用预占，按实际可计费执行直接扣除：

```text
可用积分 → 扣费中 → 已扣费
                    └→ 已冲正（仅平台账务差错）
```

- 用户积分由平台模型、媒体、公开 Tool 或 Graph Tool 的计费配置确定；发布者、用户和模型均不能修改可信最终积分。
- 每个可计费执行正式开始前原子扣除积分；余额不足时不得开始对应模型、Provider 或公开 Tool 执行。
- Graph Tool 到达业务终态后汇总其去重后的扣费明细，不额外收取 Tool 完成费。
- 用户必须能够查看模型/媒体/公开 Tool、Graph Tool、所属 Skill、计费配置版本和积分汇总。
- 失败、部分成功、取消、结果被替代或用户未采用均不退款；开始前取消不扣费，开始后取消保留已发生扣费。
- 技术重试和恢复不重复扣费；用户主动重新生成是新的可计费操作。
- 平台不提供充值退款或消费退款。重复扣费、错扣等平台账务差错通过追加式冲正纠错，不删除原记录。
- 消费积分不可转账或提现。
- 发布者收益积分在“我的 Skill → Skill Dashboard”中单独展示，至少区分待结算、可用、冻结、已回收和已冲正。
- 发布者收益积分由平台收益规则生成，不直接等同于用户总扣费或 Provider 成本；Skill 自测、自买和风险流量不产生可回收收益。
- 平台按 `50%` 系数回收可用收益积分；不足最小结算单位的余数保留在收益账户。
- 收益积分转消费积分与收益回收是两个独立操作，分别展示记录和结果。

完整计费真值表以[共通业务规则与验收基线](common-requirements-baseline.md#32-计费真值表)为准。

### 6.11 帮助、工单与账号

- 工单、计费申诉、投诉、侵权举报和产品反馈均在对应页面查看状态与处理记录。
- 工单沟通不形成独立消息中心。
- 账号设置支持个人资料、实名、账号安全、第三方授权撤销、隐私数据处理和注销。
- 实名、收益回收/打款、删除数据等高风险操作需要二次确认和必要的安全校验。

## 7. 关键状态

| 对象/状态轴 | 用户可见状态 |
| --- | --- |
| Project 生命周期 | 使用中、已归档、回收站、已删除 |
| Project 最近运行 | 空闲、排队中、进行中、等待我处理、等待异步结果、已完成、部分失败、失败、已取消 |
| Skill 版本 | 草稿、测试中、待审核、审核中、已发布、已驳回、已暂停、已下架、已废弃 |
| Skill Invocation 执行 | 已创建、运行中、等待我处理、等待异步结果、成功、部分失败、失败、已取消 |
| Skill Invocation 账务 | 未扣费、扣费中、已扣费、部分扣费、已冲正 |
| Graph Tool | 已创建、运行中、等待我处理、已接受异步执行、已完成、部分失败、失败、已取消 |
| 生成任务 | 排队中、生成中、等待重试、结果核对中、成功、失败、已取消 |
| 精选作品 | 草稿、待审核、审核中、已发布、已驳回、已暂停、已下架 |
| 发布者收益积分 | 待结算、可用、冻结、已回收、已冲正 |
| 收益回收申请 | 已提交、风控审核、财务审核、打款中、已完成、已驳回、打款失败 |
| 工单 | 待受理、处理中、待补充、已解决、已关闭 |

上述中文状态是[共通状态机](common-requirements-baseline.md#4-统一状态机与用户投影)的确定性投影。由于没有消息中心，状态必须在业务页面持续可见，并提供原因、最后更新时间和可执行的下一步。

## 8. 数据、安全与体验要求

- 默认保护 Project、Chat、故事板和资产隐私；公开必须由用户主动提交并确认。
- 前端不得向用户展示系统提示词、隐藏推理过程、Tool 密钥和内部错误堆栈。
- 所有创建、点赞、积分扣费、收益回收和提交操作需要防重复提交。
- Agent 与异步任务状态必须支持刷新后恢复；SSE 断线后能够继续接收或回读进度。
- 费用、审核、失败和权限错误必须使用用户可理解的说明，并提供查看详情或恢复入口。
- 快速创建 Project `P95 ≤ 1s` 返回 `project_id`；接受持久化输入后 `P95 ≤ 2s` 返回首个状态事件；SSE 重连 `P95 ≤ 3s` 完成缺口回放。
- 用户提交后 `200ms` 内必须出现本地反馈，不允许用长时间无状态 Loading 隐藏异步处理。
- 桌面 Web 关键创作、计费、审批和账号流程达到 WCAG 2.1 AA 的键盘、焦点、语义和对比度要求。
- 页面不得因为没有移动端而引入移动端专属流程或 PWA 安装入口。

其余可用性、容量、恢复、保留和财务对账指标统一遵循[共通非功能基线](common-requirements-baseline.md#6-v1-量化非功能基线)。

## 9. 核心验收标准

以下用例继承[共通可执行验收规范与 Evidence 要求](common-requirements-baseline.md#7-可执行验收规范)，缺少规定证据时不得判定通过。

| ID | Given / When | Then |
| --- | --- | --- |
| USR-CREATE-001 | Given 用户打开快速创作；When 输入提示词并并发点击或重试 100 次 | 只创建一个 Project、默认 Session、首消息和 Agent Input，返回同一 `project_id`，P95 在 1 秒内打开工作台。 |
| USR-CREATE-002 | Given 提示词为空或纯空白；When 开始创作 | 创建空工作台，但不创建消息、Agent Turn、Skill Invocation 或扣费记录。 |
| USR-WORKSPACE-001 | Given Project 已有 Chat、故事板、资产和运行事件；When 刷新页面或 SSE 断线重连 | 恢复同一权威状态，3 秒 P95 内完成事件缺口回放，不重复执行或扣费。 |
| USR-ASSET-001 | Given 用户进入任意资产相关页面；When 查看菜单、标题和操作文案 | 全部统一显示“资产中心/资产”，不存在旧“素材中心”用户文案。 |
| USR-SKILL-001 | Given 登录用户进入 Skill 市场；When 点击“创建 Skill” | 打开独立结构化表单，完整包含调用规则、流程规划、素材分析、故事板设计、媒体生成、提示词写法和视频剪辑字段。 |
| USR-SKILL-002 | Given Tool 未开放、已暂停、已废弃或用户无权限；When 通过 `@` 选择或提交 Skill | 前端显示字段级错误，服务端拒绝发布，不能通过修改请求绕过。 |
| USR-SKILL-003 | Given Skill 已有线上版本且新版本审核中；When 查看“我的 Skill” | Skill 仍归“已发布”，同时独立展示新版本审核状态、版本差异和原因。 |
| USR-BILL-001 | Given 可计费执行价格已冻结且余额充足；When 模型或生成正式开始 | 原子扣除配置积分并展示明细；Graph Tool 终态汇总等于去重明细之和，不追加完成费。 |
| USR-BILL-002 | Given 同一执行发生技术重试、恢复或重复回调；When 状态最终完成 | 页面只展示一笔有效扣费；用户主动重新生成时展示一笔新的扣费。 |
| USR-BILL-003 | Given 执行开始后失败、部分成功或取消；When 查看账单和发布者收益 | 已发生扣费不退款，运行状态和账务状态分别展示；失败或取消子项不生成收益，已成功合格子项正常结算，只有平台账务差错显示冲正记录。 |
| USR-EARN-001 | Given 发布者有合格可用收益积分；When 提交收益回收并重复提交 | 只创建一笔回收申请，按 50% 系数计算结算基数，余数保留并可追踪打款状态。 |
| USR-WORK-001 | Given 作者提交精选作品；When 审核通过并公开 | 只展示确认快照、实际使用 Skill、作者资料和脱敏过程，私有 Chat、隐藏指令、Tool 内部参数和未授权资产不可访问。 |
| USR-LIKE-001 | Given 同一用户对同一作品；When 并发点赞或取消 100 次 | 最多存在一个有效点赞关系，计数可从关系真源恢复。 |
| USR-NOSOCIAL-001 | Given 游客或登录用户浏览精选作品；When 查看列表、详情和账号设置 | 不存在评论、回复、评论数量、评论通知或关注关系入口。 |
| USR-ANN-001 | Given 公告处于有效版本；When 用户进入任意页面、关闭、刷新或公告被撤回 | 按版本和频率展示；撤回或过期后停止展示，且不存在公告菜单、收件箱或红点。 |

## 10. 待确认参数

以下参数不改变本文已确认的计费和状态规则，但必须在对应详细 PRD 或设计评审中确认：

1. Skill 各结构化字段的必填矩阵与字数上限。
2. 模型、媒体、公开 Tool 和 Graph Tool 的具体积分配置、计费单位和版本生效规则。
3. 发布者收益积分生成规则、结算周期、最低回收积分、积分兑付币种/法币面值、税费和收益转消费积分比例；五折回收系数已经确认，不再作为待定项。
4. 精选作品是否允许未登录用户查看完整创作过程。
5. 公告默认展示频率及多个公告的最大连续弹窗数量。
6. 独立创作页面采用当前标签页路由跳转还是新浏览器标签页打开。
