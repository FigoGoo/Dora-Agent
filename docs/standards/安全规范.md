# 安全规范

状态：active
owner：文档与契约责任域
适用范围：前端、智能体微服务、业务微服务、测试和配置
更新时间：2026-06-30

## 目标

为 AIGC Agent SaaS 建立基础安全边界，覆盖认证授权、租户隔离、密钥管理、Agent Tool 安全、数据保护和日志脱敏。

## 认证与授权

- 所有用户请求必须有认证上下文。
- 业务权限由业务微服务最终校验。
- 智能体微服务只能传递权限上下文，不替业务服务做最终业务授权。
- 前端只展示权限结果，不绕过服务端权限。

### 管理员越权红线（与业务归属对称）

- 平台管理员与业务用户是两套独立鉴权上下文：管理员身份（`AdminAuth`）只承载平台管理能力（管理员、用户、审计等平台级元数据），不得携带或被赋予业务空间归属（`space_id`、业务 user 归属、enterprise 归属）。
- 管理员不得以管理员身份跨入业务空间归属上下文读写业务空间内资产、项目、作品、积分等归属数据；业务空间归属数据只接受业务用户鉴权上下文（`AuthContext`）。
- 管理员不得设置、重置或改写业务用户密码；平台后台只能变更用户状态等平台治理字段。用户密码变更必须走用户侧认证/找回密码闭环，且由对应用户鉴权或专用安全流程负责。
- 管理员如确需查看用户业务空间内容，必须走独立的、留痕的只读通道并单独鉴权，不得复用业务用户归属；该通道未建立前，平台管理通道一律不展开业务空间归属明细（如用户详情中的空间、企业成员、归属审计明细保持不暴露）。
- 上述红线与业务用户侧「当前空间=归属总开关、跨空间访问禁止」对称：二者任一缺失即视为越权红线缺口。

## 租户隔离

- 所有业务数据和 Agent Runtime 数据必须带 tenant_id 或等价隔离字段。
- 跨租户访问必须禁止。
- 测试数据也必须区分租户，避免误判权限。

## Secret 管理

- 禁止提交真实 `.env`、AK/SK、Token、数据库密码、私钥。
- `.env.example` 只写占位值。
- 本地真实密钥放 `.env.local` 或开发者本机安全配置。
- 线上密钥通过环境变量或安全配置系统注入。

## Agent Tool 安全

- Tool 必须有明确输入 schema、输出 schema、权限要求和错误分类。
- 能改变业务事实的 Tool 必须通过业务 RPC，并按业务需要由后端生成或校验内部业务幂等键。
- 高风险 Tool 必须支持人工确认或由产品设计明确风险豁免。
- Skill 绑定 Tool 只代表设计期声明，Agent 执行前必须逐 Tool 调用业务服务策略/白名单校验；不得复用缓存的 `tool_bindings` 结果绕过运行期复检。
- 禁止 Agent 直接执行任意系统命令、任意 SQL、任意外部 HTTP 请求。

## Prompt Injection 防护

- 不信任用户输入、网页内容、文件内容或检索结果中的指令。
- Tool 调用权限不能由模型输出单独决定。
- 系统提示词、密钥、内部配置不得暴露给用户或模型生成内容。
- 检索内容要标注来源，不能直接覆盖系统策略。

## 文件与媒体安全

- 上传文件必须限制大小、类型和处理方式。
- 媒体生成产物必须记录来源、用户、任务和策略版本。
- 外部 URL 下载必须有白名单、超时和大小限制。
- TOS 使用公共桶时，对象 key 必须不可猜测，业务服务必须在签发上传凭证前校验登录态、空间、项目和文件约束。
- 前端直传 TOS 只能使用短期上传凭证，不得接触 AK/SK。
- 公开桶对象 URL 一旦泄露即可能被直接访问；第一版接入 CDN 且不做缓存失效，取消分享或下架后只能保证业务列表和详情不可访问，已缓存 URL 不承诺即时失效。
- 已下架媒体如需重新公开，必须换 object key。
- object key 目录和命名规则以 `docs/standards/TOS对象存储规范.md` 为准。

## 日志与隐私

- 日志必须脱敏。
- 禁止记录完整 Token、密码、AK/SK、支付凭证、未脱敏个人敏感信息。
- Agent 消息、提示词、产物和记忆按产品策略和用户授权保存。

## 数据库安全

- 业务 DB 与 Agent DB 隔离。
- DAO 不拼接未校验 SQL。
- migration SQL 必须评审。
- 删除、归档、数据保留必须遵循产品与隐私要求。

## 前端安全

- 不在前端保存长期敏感密钥。
- API 错误信息不得泄露内部实现。
- 对富文本、Markdown、HTML 展示进行 XSS 防护。
- WebSocket/SSE 连接必须校验认证上下文。
