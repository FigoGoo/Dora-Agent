# 智能体配置化规范

状态：active  
owner：Agent 服务责任域
适用范围：Agent Runtime、Agent 配置、Tool、Skill、Workflow、Graph、Memory、模型参数  
更新时间：2026-06-22  

## 目标

智能体必须尽可能配置化，避免把产品策略、提示词、模型参数、工具开关、工作流策略硬编码到代码里。

## 产品输入

产品与需求责任域在《产品系统设计》中定义：

- Agent 能力目标。
- 支持的输入、输出和产物类型。
- 可用 Tool / Skill。
- 是否需要 Workflow / Graph。
- 是否需要 Memory。
- 是否需要人工确认。
- 安全和内容策略。
- 验收标准。

## 配置项范围

建议配置化：

- Agent 名称、描述、版本。
- 模型供应商、模型名、温度、最大 token、超时。
- System prompt、角色约束、输出格式。
- Tool 白名单、Tool 参数约束、Tool 权限要求。
- Skill 白名单。
- Retriever 数据源和检索参数。
- Memory 开关、作用域、保留策略。
- Workflow / Graph 选择和节点配置。
- Interrupt / Resume 策略。
- AG-UI 事件开关和展示策略。
- 成本、速率、并发和超时限制。

## 存储规则

- Agent 配置保存在 Agent 领域数据库，例如 `agent_runtime_configs`。
- 配置必须有版本号、状态、owner、更新时间、审计字段。
- 配置变更必须可回滚。
- 敏感密钥不进入 Agent 配置表，只引用安全配置 key。

## 代码规则

- 代码实现配置解释器和执行器，不把具体业务 Agent 策略写死。
- 默认配置可以存在，但必须被显式版本化。
- 代码里允许保留安全底线，例如禁止任意 SQL、禁止任意系统命令。

## 测试规则

- 每个 Agent 配置版本必须有最小验证集。
- Tool 权限、人工确认、输出格式和失败路径必须可测试。
- 产品验收标准映射到配置验证项。

## 检查表

- [ ] 产品是否定义 Agent 能力和配置边界。
- [ ] 是否避免硬编码提示词和工具策略。
- [ ] 是否有配置版本和审计。
- [ ] 是否有回滚方式。
- [ ] 是否有配置验证项。
