# 本地开发与配置规范

状态：active
owner：文档与契约责任域
适用范围：本地 macOS + Docker 开发环境
更新时间：2026-06-22

## 环境边界

- 本地开发环境：macOS + Docker。
- 线上目标环境：CentOS 8 单机。
- 当前阶段以本地开发和联调为主，不定义线上 CI/CD、告警、SLO 和自动发布。

## 本地依赖

本地依赖建议通过 Docker 启动：

- PostgreSQL：业务数据库和 Agent 领域数据库可以用不同 database 或 schema。
- etcd：服务发现和非敏感配置。
- mock 服务：外部 AIGC、对象存储、支付或第三方服务未就绪时使用。

## env 文件

- `.env.example` 是本地配置项模板。
- 开发者复制为 `.env.local` 或本机约定文件。
- `.env.local`、`.env`、包含真实密钥的文件不得提交。
- 新增配置项必须同步 `.env.example` 和本规范。

## 配置分层

```text
默认配置
  -> .env.example 中声明的本地配置
  -> .env.local 中的开发者本机配置
  -> etcd 中的非敏感运行配置
  -> 安全配置系统或环境变量中的密钥
```

## 服务端口建议

- 业务 Kitex server：`19001`
- 智能体 Gin HTTP：`18080`
- 智能体 Kitex client 不固定监听端口。
- 前端开发服务：`3000`
- PostgreSQL：`5432`
- etcd：`2379`

实际端口以 `.env.example` 为准。

## 数据库配置

- 业务数据库和 Agent 领域数据库必须分离。
- 本地可使用同一 PostgreSQL 实例下的不同 database：`dora_business`、`dora_agent`。
- 本地 migration 可以使用 golang-migrate。
- 线上 migration 上线后由用户手动执行。

## 服务发现配置

- 本地 etcd endpoint 默认：`http://127.0.0.1:2379`。
- Kitex server 注册到 etcd。
- Kitex client 通过 etcd resolver 发现业务服务。

## 安全配置

- 真实 AK/SK、Token、数据库密码不得写入 `.env.example`。
- `.env.example` 只放占位值。
- 本地开发可使用弱密码，但不得复用线上密码。
- TOS 公共桶的 endpoint、bucket、region、base URL 可以作为非密配置写入 `.env.example`；`TOS_ACCESS_KEY_ID`、`TOS_SECRET_ACCESS_KEY` 只能写入 `.env.local` 或部署密钥。
- 短信服务的 AK/SK 只能写入 `.env.local` 或部署密钥；短信签名、模板 ID、账号 ID 如涉及生产资源，也按敏感运行配置处理，不写入仓库。
- 前端直传 TOS 时只能使用后端按对象 key、MIME、大小和有效期签发的上传凭证，不允许把 TOS AK/SK 下发给前端。
- TOS object key 和目录规则以 `docs/standards/TOS对象存储规范.md` 为准。
