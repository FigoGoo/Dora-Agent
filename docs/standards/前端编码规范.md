# 前端编码规范

状态：active
owner：主控 Codex 汇总维护
适用范围：`/web/**`、`/frontend/**` 和相关前端代码

## 页面

- 页面按用户流程组织，不做营销页式空壳。
- 核心工作流首屏可用，任务状态和历史可追踪。

## 组件

- 组件职责清晰：输入、参数、消息、Tool 状态、人工确认、产物、任务列表分开。
- 组件 props 使用契约字段，不发明后端字段。
- 图标统一使用 `lucide-react` 按需引入；不得混用其他 icon 库、emoji、图片图标或手写内联 SVG 作为常规功能图标。
- 仅图标按钮必须提供 `aria-label` 和 tooltip。

## 状态管理

- 必须覆盖 loading、empty、error、success、streaming、interrupt、resume。
- Agent run 状态和 UI 局部状态分开管理。

## API 集成

- HTTP API 以 API 契约为准。
- 处理鉴权、状态码、错误码、分页、重试和取消。

## AG-UI 事件消费

- 以 AG-UI 事件规范为准。
- 按 event_id 去重，按 sequence 合并。
- 未知事件不应导致页面崩溃。

## 流式渲染

- message.delta 增量渲染。
- message.completed 固化消息。
- 断线重连后可补偿缺失事件或查询 run 状态。

## 错误态

- 错误展示用户可理解信息。
- 提供重试、返回、联系支持或查看详情的合理入口。

## 加载态

- 区分首次加载、提交中、流式生成中、工具调用中和恢复中。

## 人工确认

- interrupt.required 必须展示确认对象、风险、可选动作和过期提示。
- resume 后展示恢复中和恢复结果。

## 测试

- 组件测试覆盖渲染和交互。
- 状态测试覆盖 AG-UI 事件合并。
- 浏览器测试覆盖主路径、错误路径、中断恢复和断线重连。
