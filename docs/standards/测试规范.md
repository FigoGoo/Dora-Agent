# 测试规范

状态：active  
owner：主控 Codex 汇总维护  
更新时间：2026-06-25
适用范围：所有服务、前端、契约、数据库和端到端测试  

## 单元测试

- 覆盖纯函数、领域规则、状态机、Graph 节点、Tool 输入输出和前端组件。
- 错误分支和边界输入必须覆盖。

## 集成测试

- 覆盖 repository、application、RPC client/server、事件生产和前端 API 集成。
- 使用隔离测试数据，避免污染真实环境。

## 契约测试

- RPC 契约测试覆盖场景化 DTO、分页默认值、错误码、权限、超时、重试、幂等和版本兼容。
- API 契约测试覆盖请求、响应、状态码、鉴权、分页默认 10 条每页和错误。
- AG-UI 契约测试覆盖事件类型、payload、顺序、幂等和未知事件兼容。

## 浏览器测试

- 覆盖用户主路径、创建任务、流式输出、Tool 调用展示、人工确认、任务完成和失败重试。
- 记录关键截图、日志和网络证据。

## RPC 链路测试

- 验证智能体服务 RPC client 到业务服务 RPC server 的正常路径、业务错误、权限错误、超时和幂等。

## Agent 行为测试

- 覆盖 Agent 推理入口、Tool 选择、Graph / Workflow、TurnLoop、中断、恢复、长任务状态和事件输出。

## Agent 领域数据库验证

- 验证 agent_sessions、agent_runs、agent_messages、agent_events、agent_tool_calls、agent_tasks、agent_interrupts、agent_artifacts、agent_memories、agent_runtime_configs。
- 验证状态机、索引查询、幂等写、数据保留、无数据库级外键约束、列表分页和批量查询路径。

## 业务数据库验证

- 只验证业务服务产生的业务事实。
- 不要求或允许智能体服务直接访问业务数据库。
- 验证业务库 schema 未创建数据库级外键约束，列表查询分页且默认 10 条每页。

## 回归测试

- 修复缺陷后复现原问题并验证相关路径未退化。
- 高风险修复要补充相邻场景。

## 缺陷报告

- 必须包含严重级别、复现步骤、期望结果、实际结果、浏览器表现、AG-UI 证据、API/RPC 证据、数据库证据、初步定位和建议 owner。
