# 文档规范

状态：active
owner：主控 Codex 汇总维护
适用范围：docs 下所有项目文档

## 文档分类

- PRD：产品目标、用户流程、Agent 能力和验收标准。
- 产品系统设计：产品体验设计师交付给工程角色 owner 的产品目标、用户流程、Agent 能力、页面结构、业务规则、异常场景、验收标准和交付说明。
- ADR：架构或技术决策。
- RPC 契约：智能体微服务和业务微服务边界。
- API 契约：前端和智能体微服务边界。
- AG-UI 事件协议：智能体微服务和前端实时事件边界。
- Agent 领域数据模型：Agent Runtime 数据库表和状态机。
- 测试报告：测试范围、结果、风险和上线建议。
- 缺陷报告：可复现问题和证据。
- 产品设计交接规范：见 `docs/standards/产品设计交接与开发约束规范.md`。
- 开发流程规范：产品 Done Gate、按功能点开发、本地开发和 SQL 准备规则。
- GitHub 协作规范：分支、提交、PR、评审和合并规则。
- 安全规范：认证、授权、租户、Secret、Agent Tool 和日志脱敏规则。
- 本地开发与配置规范：macOS + Docker 本地环境和 env 模板。
- 迭代 SQL 脚本规范：本地 SQL 和上线手动迁移准备规则。
- TOS 对象存储规范：公共桶、object key、上传签名、前端直传和公开媒体快照规则。

## 文档路径

- 产品文档：`docs/product/`
- 设计文档：`docs/design/`
- 架构文档：`docs/architecture/`
- 契约文档：`docs/contracts/`

- 模板文档：`docs/templates/`
- 项目计划：`docs/product/` 或 `docs/project/`

## 状态

- draft：草案，未作为实现依据。
- active：当前有效事实源。
- deprecated：已废弃，必须说明替代文档或废弃原因。

## 必填元信息

- owner：负责人或维护角色。
- 更新时间：使用 `YYYY-MM-DD`。
- 适用范围：说明服务、模块、页面、接口或数据范围。
- 相关代码路径：列出主要目录或文件。
- 相关契约：列出 RPC、API、AG-UI 或数据模型文档。

## 内容要求

- 背景：为什么需要这份文档。
- 目标：要达成的结果。
- 非目标：明确本次不覆盖的内容。
- 约束：服务边界、技术、权限、事务、事件、数据或测试约束。
- 方案：结构化说明设计、契约、流程或验证。
- 风险：列出影响范围和缓解方式。
- 验收标准：必须可验证。

## 更新规则

- 契约变更先更新文档，再改实现。
- 代码行为与文档不一致时，必须同步修正文档或代码。
- 文档更新要说明影响范围。

## 废弃规则

- deprecated 文档不得删除，除非主控 Codex 明确清理。
- 废弃文档必须指向替代文档或说明历史原因。
