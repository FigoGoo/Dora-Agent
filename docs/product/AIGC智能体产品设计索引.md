## 背景

Dora-Agent 是一个基于 Go + Eino 的 AIGC 智能体 SaaS 应用，用户可以通过 Agent 生成音乐、图片、视频等内容。

本索引记录产品设计文档拆分关系、正式工程设计入口和开发依据口径。各功能点的产品规则、验收标准和待确认事项以对应子文档为准。

当前所有产品文档均为 `product_status: Done`，允许工程角色 owner 进入正式开发。正式开发设计计划以 `code-plan/README.md`、`code-plan/agent/**`、`code-plan/business/**`、`code-plan/tests/**` 为准，不再要求额外输出 `docs/project/` 项目计划。本次开发不涉及前端开发和部署上线文档，`docs/design/**` 仅作为 Agent、业务 API、AG-UI 和测试验收的产品体验输入。

## 设计关系

```text
统一 Agent
  -> 基于用户意图选择 Skill
  -> 基于当前空间加载可用 Skill 池
  -> 覆盖音乐创作、MV 制作、短视频制作、电商商品图、电商广告视频、文旅宣传视频、品牌 LOGO 等典型场景
  -> Skill 只能绑定平台开放 Tool
  -> 生成资源时使用模型选择
  -> 扣费类 Tool 调用前进入积分预估与确认

Skill Builder
  -> 创建系统 / 企业 / 个人 Skill
  -> 引用 Tool 边界
  -> 声明由平台内置固定资产元素类型组成的输出元素结构
  -> 被统一 Agent 路由使用

模型供应商
  -> 由平台管理员配置
  -> 向模型选择提供可用模型
  -> 普通用户不可见供应商与 API Key

模型选择
  -> 只对图片 / 音乐 / 视频生成开放
  -> 只对当前对话生效
  -> 与积分扣费共同完成生成前确认

Tool 边界
  -> 第一版只允许平台内置 Tool 开关和白名单
  -> 约束 Skill 可调用能力
  -> Tool 风险等级由平台管理员配置
  -> Tool 可按企业、用户等级或套餐限制可用范围
  -> 业务事实必须通过业务 RPC Tool
  -> 高风险 Tool 触发人工确认

内容安全治理
  -> 第一版只使用 LLM 做提示词安全评估
  -> 评估用户提示词和生成前提示词
  -> 安全评估通过后才进入积分冻结和生成执行
  -> 不做外部内容安全服务、人工复核或生成文件内容审核

积分扣费
  -> 属于业务域
  -> 基于当前空间选择个人或企业积分账户
  -> 使用积分来源与有效期规则计算可用余额
  -> 由业务微服务负责账户、冻结、扣减、释放和审计

积分来源与有效期
  -> 第一版不做在线购买积分包
  -> 平台管理员后台手动发放积分
  -> 用户或企业拥有者使用兑换码兑换积分
  -> 积分按批次拥有有效期

账户体系
  -> 用户默认拥有个人空间
  -> 登录页面支持选择个人登录或企业登录
  -> 企业通过专用企业注册页面创建，创建者默认成为企业拥有者
  -> 企业成员只能通过邀请加入
  -> 第一版一个用户同一时间最多属于一个企业
  -> 用户通过个人中心切换身份，等价于重新登录目标身份并切换当前空间
  -> 企业空间本质是多账户共享企业积分
  -> 第一版不做独立企业管理员角色，企业管理权限由企业拥有者承担
  -> 企业拥有者不额外查看成员资产、产物、会话、任务、黑板或生成记录
  -> 资源、Skill、积分和历史记录按空间归属

项目与资产归属
  -> 项目是用户创作目标的上层容器，不是新的 Agent 或企业项目管理工具
  -> 用户从首页或工作台开始新创作时默认创建项目
  -> 会话、run、黑板过程、生成资产和上传素材挂在项目下
  -> 资产保留来源会话、来源 run 或来源上传动作摘要
  -> 作品从项目资产中整理出来，公开展示由作品公开快照承担
  -> 项目业务事实由业务微服务维护，Agent 只保存 project_id 引用和 Runtime 过程

资产与创作过程保存
  -> 业务微服务保存用户可见资产事实
  -> 资产必须归属项目，并保留来源会话或 run 引用
  -> 最终资产元素随用户可见资产由业务微服务保存
  -> Agent 微服务保存黑板中的草稿态资产元素
  -> Agent 微服务保存会话、消息、事件、Tool 调用、黑板和资产引用
  -> 生成完成且资产保存成功后进入资产视图、产物历史和产物详情
  -> 用户可以从会话历史进入原会话，并在会话中引用已保存资产继续创作

作品中心与精选作品
  -> 个人作品中心是当前登录用户自己的作品集
  -> 用户可以从已保存资产创建作品，编辑标题、封面、简介、分类和标签
  -> 用户分享作品后进入精选作品中心
  -> 首页公开作品和精选作品中心支持免登录查看列表和详情
  -> 精选作品支持点赞、复制分享链接、作品标签和作品分类
  -> 点赞、开始创作、创建项目、查看个人数据等需要登录的动作由登录弹窗承接
  -> 公开作品不得展示源会话、黑板、提示词、积分、模型成本或用户隐私信息

AG-UI / A2UI
  -> 单独设计组件样式、组件作用和交互契约
  -> 组件样式必须符合站点主题样式
  -> 组件分为聊天区域组件和聊天外部组件
  -> 支持思考状态打印机输出，但不暴露模型内部推理链路
  -> 支持 Skill、Tool 等平台能力 Tag 展示，但不暴露内部配置
  -> 模型选择在聊天输入框内完成，不作为独立 A2UI 组件
  -> 聊天外部只有一个外部工作区组件，承载资产视图和黑板视图
  -> 单独定义事件类型、事件顺序和 payload
  -> 承接统一 Agent 的流式输出、Tool 状态、人工确认、任务进度和产物展示

页面范围与交互状态
  -> 定义用户端、企业空间和平台后台的页面清单
  -> 普通用户端和企业空间共用统一 Agent 创作工作台
  -> 平台管理员后台采用独立入口、独立代码
  -> 平台管理员后台默认存在初始管理员账号，可新增平台管理员账号
  -> 平台管理员后台第一版只有一个平台管理员角色，不做 RBAC
  -> 平台审计日志覆盖后台关键操作
  -> 定义加载、空状态、错误、权限不足、生成中、确认中、取消和重连等状态
  -> 为前端设计系统和页面开发提供产品输入

工程交付输入清单
  -> 在产品 Done 后交付工程角色 owner
  -> 拆分 Agent、业务后端、前端、测试和项目管理输入
  -> 明确 RPC、API、AG-UI、Agent 数据模型和 SQL 脚本的后续契约入口
  -> 正式开发设计计划统一放在 code-plan/
```

## 统一约束

- 产品层面只有两类顶层用户：平台管理员、普通用户。
- 系统初始化时默认存在一个初始平台管理员账号，可新增平台管理员账号。
- 第一版平台管理员不做 RBAC，所有平台管理员拥有相同后台权限。
- 平台管理员后台支持用户管理，用于查询用户、查看基础信息、启用或禁用用户账号。
- 用户管理不允许平台管理员代登录普通用户账号。
- 用户管理不展示用户私有资产、会话、黑板、生成记录、提示词、上传私有素材或创作内容。
- 企业拥有者、企业普通成员、个人用户是普通用户在不同空间和权限下的细分角色。
- 第一版不做独立企业管理员角色，企业管理权限默认由企业拥有者承担。
- 普通个人用户接受企业邀请后成为企业用户；企业用户被移除企业后恢复为普通个人用户。
- 登录页面支持个人登录和企业登录；个人登录进入个人空间，企业登录进入企业空间。
- 用户每次使用 Agent 时必须处于一个当前空间：个人空间或企业空间。
- 当前空间通过登录页面身份选择或个人中心身份切换确定；身份切换等价于退出当前登录身份并重新登录目标身份。
- 用户从首页或工作台开始新创作时默认创建项目；项目归属于当前空间和创建者。
- 会话必须归属项目；run 通过会话归属项目。
- 生成资产和上传素材必须归属项目，并保留来源会话、来源 run 或来源上传动作摘要。
- 黑板以项目为用户可见聚合口径，同时保留 run 级快照用于恢复。
- 作品从项目资产中创建；项目本身不公开，公开展示由作品公开快照承担。
- 企业空间第一版本质是多账户共享企业积分，不做企业资产、产物、会话、任务、黑板或生成记录管理。
- 企业拥有者不额外查看成员资产和生成记录。
- 业务微服务保存用户可见资产事实，Agent 微服务保存创作过程和资产引用。
- 个人作品中心保存当前登录用户自己的作品集，作品引用已保存资产。
- 分享后的作品进入精选作品中心，也可作为首页公开作品展示，并支持免登录查看。
- 未登录访客可以访问与用户无关的公开数据，例如首页公开作品、精选作品列表和公开作品详情。
- 点赞、开始创作、创建项目、查看个人资产 / 作品 / 积分 / 通知等需要登录的动作必须弹出登录弹窗，并保留当前页面和动作意图。
- 免登录访问只包含首页公开作品摘要、浏览列表、查看公开详情和复制分享链接。
- 公开作品不得暴露源会话、黑板、提示词、积分、模型成本、内部用户 ID、手机号、邮箱或私有素材。
- 资产元素不按具体创作场景写死；平台内置固定元素类型，Skill 声明输出元素结构，前端按元素类型渲染。
- 资产元素类型第一版不提供后台配置，但不能散落硬编码在业务代码中，应作为平台内置配置或系统字典维护。
- 生成完成且资产保存成功后，才作为可用资产展示并确认该资产对应扣费。
- 内容安全治理第一版只使用 LLM 做提示词安全评估，且必须在积分冻结和生成执行前完成。
- Agent 配置能力不干涉业务规则。
- Agent 使用什么能力，由用户意图和可用 Skill 决定。
- 音乐创作、MV 制作、短视频制作、电商商品图、电商广告视频、文旅宣传视频、品牌 LOGO 等是第一版统一 Agent 需要覆盖的典型创作场景，不单独拆成场景矩阵文档。
- 普通用户不能接入自己的模型 API Key。
- 所有 Skill 只能使用平台开放 Tool。
- 涉及扣费、业务事实写入、高风险 Tool 的任务必须人工确认。
- Agent Runtime 不直接访问业务数据库，不保存订单、支付、积分账户、业务权限等业务事实。
