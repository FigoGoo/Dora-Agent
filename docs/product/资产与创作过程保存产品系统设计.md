## 背景

统一 Agent 会生成图片、音乐、视频、脚本、提示词、分镜和中间素材。产品需要明确哪些内容保存为用户可见资产，哪些内容保存为创作过程，避免资产事实、Agent Runtime 过程数据和业务权限边界混在一起。

第一版补充项目 Project 作为创作目标容器。资产和创作过程仍绑定当前身份和当前空间，同时需要挂在项目下，避免用户在多次对话后无法理解资产归属。

## 目标

- 定义第一版需要保存哪些资产。
- 定义第一版资产元素的配置化保存规则。
- 定义第一版需要保存哪些创作过程。
- 明确资产归属、可见范围和引用规则。
- 明确项目、会话、run、资产和黑板之间的归属关系。
- 明确生成完成、失败、取消、断线重连时的保存规则。
- 明确业务微服务和 Agent 微服务的保存边界。
- 支持用户在会话中引用已创作资产继续创作。

## 核心概念

| 概念 | 定义 | 主要保存方 |
| --- | --- | --- |
| 项目 Project | 用户围绕一个创作目标形成的容器，聚合会话、资产、黑板和作品 | 业务微服务 |
| 资产 Asset | 用户可见、可预览、可下载、可在会话中引用的图片、音乐、视频、文件或上传素材 | 业务微服务 |
| 资产元素 Asset Element | 资产内部可展示、可编辑、可引用的结构化内容，例如标题、歌词、脚本、分镜、提示词、参考素材引用等 | 最终资产元素由业务微服务保存，草稿态元素由 Agent 微服务保存 |
| 资产元素类型 Asset Element Type | 平台内置给 Skill 使用的固定通用元素类型，例如短文本、长文本、图片引用、音频引用、视频引用、分镜、时间线、参数组等 | 平台内置配置 / 系统字典 |
| Skill 输出元素结构 Skill Output Schema | Skill 声明本次创作会产出哪些资产元素，以及这些元素如何在黑板和资产详情中展示 | Skill 配置 |
| 产物 Artifact | Agent 生成任务产出的结果引用，可以转为用户资产 | Agent 微服务保存引用，业务微服务保存资产事实 |
| 创作过程 Process | 会话、消息、运行、事件、Tool 调用、模型选择、积分确认、生成任务、黑板、分镜、脚本、提示词等过程数据 | Agent 微服务 |
| 黑板 Blackboard | Skill 在会话外部工作区展示的故事线、分镜、脚本、提示词、参考素材和中间结果 | Agent 微服务 |
| 资产引用 Asset Reference | 创作过程里引用某个资产的关系，例如参考图、已生成音乐、分镜图片 | Agent 微服务保存引用关系，业务微服务校验资产权限 |

## 资产保存范围

第一版保存以下资产：

| 资产类型 | 来源 | 是否自动保存 | 可见位置 |
| --- | --- | --- | --- |
| 生成图片 | 图片生成模型 Tool | 是 | 资产视图、产物历史、产物详情 |
| 生成音乐 | 音乐生成模型 Tool | 是 | 资产视图、产物历史、产物详情 |
| 生成视频 | 视频生成模型 Tool | 是 | 资产视图、产物历史、产物详情 |
| 上传素材 | 用户上传或选择 | 是 | 资产视图、聊天输入素材选择 |
| 中间素材 | Skill 生成的分镜图、草稿音频、脚本文件等 | 需要展示或复用时保存 | 黑板视图、资产视图 |

## 上传素材约束

上传素材在上传时完成约束校验。第一版建议规则如下，最终数值可在产品确认时调整：

| 素材类型 | 允许格式 | 单文件大小上限 | 失败处理 |
| --- | --- | --- | --- |
| 图片素材 | jpg、jpeg、png、webp | 20MB | 拒绝上传并提示格式或大小原因 |
| 音频素材 | mp3、wav、m4a | 100MB | 拒绝上传并提示格式或大小原因 |
| 视频素材 | mp4、mov、webm | 500MB | 拒绝上传并提示格式或大小原因 |
| 文档 / 文本素材 | txt、md、pdf、doc、docx | 50MB | 拒绝上传并提示格式或大小原因 |

上传规则：

- 上传时校验文件大小、扩展名、MIME 类型、空文件和明显损坏文件。
- 上传失败不创建用户可见资产。
- 上传失败不产生积分扣费。
- 上传素材的标题、说明、标签等文本会按 [内容安全治理产品系统设计](./内容安全治理产品系统设计.md) 做 LLM 提示词安全评估。
- 第一版不对上传图片、音频、视频文件本身做多模态内容安全审核。

## 资产元素规则

资产元素不能按具体业务场景写死。音乐、MV、短视频、电商商品图、电商广告视频、文旅宣传视频、品牌 LOGO 等场景只是典型使用方式，不决定固定字段。

第一版采用以下规则：

- 平台内置固定资产元素类型。
- 第一版不提供资产元素类型后台配置入口。
- 资产元素类型不能散落硬编码在业务代码中，应作为平台内置配置或系统字典维护。
- Skill 只能从平台内置固定资产元素类型中声明输出结构。
- Skill 输出元素结构决定黑板视图和资产详情中展示哪些元素。
- Agent 创作过程中先把草稿态元素保存到黑板和创作过程。
- 用户确认、生成完成或需要复用时，草稿态元素可以转为最终资产元素。
- 最终资产元素作为用户可见资产事实的一部分，由业务微服务保存。
- 前端根据资产元素类型渲染，不按具体场景硬编码页面字段。

第一版平台内置的通用资产元素类型建议包括：

| 元素类型 | 用途示例 |
| --- | --- |
| 短文本 | 标题、名称、口号、关键词 |
| 长文本 | 歌词、剧本详情、文案、旁白稿 |
| 富文本 | 带段落结构的脚本、说明、方案 |
| 结构化对象 | 商品信息、品牌信息、景点信息 |
| 列表 | 卖点列表、镜头列表、素材列表 |
| 图片引用 | 参考图、分镜图、封面、生成图片 |
| 音频引用 | 音乐、BGM、旁白、音效 |
| 视频引用 | 成片、片段、参考视频 |
| 文件引用 | 可下载文件、附件、工程文件 |
| 提示词 | 图片提示词、视频提示词、音乐提示词 |
| 分镜 | 镜头画面、脚本、提示词、状态 |
| 时间线 | 视频段落、音频段落、字幕段落 |
| 标签组 | 风格、场景、情绪、受众 |
| 参数组 | 生成参数、可公开配置摘要 |

平台内置资产元素类型需要做到相对完善，覆盖常见 AIGC 创作的文本、媒体引用、结构化内容、分镜、时间线、标签和参数。后续新增元素类型时，应通过平台内置配置或系统字典发布，不通过每个业务场景单独写死字段。

示例，不作为固定字段：

| 场景 | 资产 | 可能的资产元素 |
| --- | --- | --- |
| 音乐创作 | 音乐资产 | 标题、歌词、曲风、封面、音频引用、提示词 |
| MV 制作 | 视频资产 | 故事线、分镜、歌词引用、音乐引用、视频引用 |
| 电商商品图 | 图片资产 | 商品信息、卖点文案、参考图、提示词、生成图片 |
| 电商广告视频 | 视频资产 | 商品卖点、脚本、分镜、字幕、音乐引用、视频引用 |
| 文旅宣传视频 | 视频资产 | 景点信息、宣传文案、镜头规划、配音稿、视频引用 |
| 品牌 LOGO | 图片资产 | 品牌名、品牌关键词、色板、字体建议、LOGO 图片 |

第一版不保存为用户资产的内容：

- Agent 内部推理链路。
- 系统提示词。
- 平台内部配置。
- 供应商 API Key、endpoint、内部模型 ID。
- 未完成且不可预览、不可复用的临时文件。

## 创作过程保存范围

第一版保存以下创作过程：

- 会话信息。
- 用户消息和 Agent 可见回复。
- Agent run。
- AG-UI / A2UI 事件。
- Skill 路由结果。
- 当前对话模型选择。
- 用户输入参数摘要。
- 积分预估、确认、冻结、扣减、释放结果引用。
- Tool 调用状态、输入摘要、输出摘要和错误摘要。
- 生成任务状态。
- 产物引用。
- Skill 输出元素结构快照。
- 黑板中的草稿态资产元素。
- 已转为资产的最终资产元素引用。
- 黑板故事线。
- 分镜、脚本、提示词、参考素材和生成状态。
- 会话中引用资产的关系。

创作过程保存只保存用户可见或排障必要信息，不保存模型内部推理链路、系统提示词、密钥、供应商内部响应原文或平台安全策略细节。

## 归属与可见范围

资产和创作过程都必须绑定当前登录身份、当前空间和当前项目。

项目：

- 用户从首页或工作台开始新创作时默认创建项目。
- 一个会话必须归属于一个项目。
- 一个 run 必须归属于一个会话，并通过会话归属于项目。
- 用户在会话中生成或上传的资产默认归属于当前项目。
- 资产必须保留来源会话、来源 run 或来源上传动作摘要。
- 黑板以项目为用户可见聚合口径，同时保留 run 级快照用于恢复。

个人空间：

- 资产归属个人空间。
- 创作过程归属个人空间。
- 本人可见本人资产、会话、黑板和生成记录。
- 个人资产可在本人后续会话中引用。

企业空间：

- 资产归属企业空间。
- 创作过程归属企业空间。
- 第一版企业成员只能查看自己的资产、会话、黑板和生成记录。
- 企业拥有者不额外查看成员资产、会话、任务、黑板或生成记录。
- 成员被移出企业后，企业空间资产和创作过程仍留在企业，原成员失去访问权限。

## 预览与下载规则

- 用户有权限访问的资产都支持预览和下载。
- 资产文件存储在火山引擎 TOS。
- 第一版资产文件不设置业务有效期，不因文件链接时效导致资产不可用。
- 预览和下载权限只由当前空间、资产归属和用户可见范围决定。
- 预览失败时展示可理解错误和重试入口。
- 下载失败时展示可理解错误和重试入口。
- 被移出企业后，用户不能预览或下载企业空间资产，个人空间资产不受影响。

## 保存边界

业务微服务负责：

- 项目事实、项目状态、项目与资产 / 作品的业务关联。
- 用户可见资产事实。
- 最终资产元素。
- 平台内置资产元素类型的业务可见规则。
- 资产归属和权限。
- 资产预览、下载、历史列表和详情。
- 上传素材登记。
- 生成资产入库。
- 资产与积分账单、消耗明细的业务关联。

Agent 微服务负责：

- 会话、运行、消息、事件、Tool 调用、任务和黑板。
- 在会话、run、事件、产物引用中保存 `project_id` 引用。
- 黑板中的草稿态资产元素。
- 产物引用和资产引用关系。
- AG-UI / A2UI 事件流恢复。
- 创作过程快照。

Agent 微服务不能直接写业务项目或业务资产数据库。生成完成或上传素材需要成为用户可见资产时，必须通过业务微服务 API / RPC 创建或确认资产，并绑定到当前项目。

## 保存时机

| 时机 | 保存内容 | 保存规则 |
| --- | --- | --- |
| 会话创建 | 会话、当前登录身份、当前空间、当前项目 | 自动保存 |
| 用户发送消息 | 用户消息、输入参数、引用资产 | 自动保存 |
| Agent run 开始 | run、Skill 路由、模型选择 | 自动保存 |
| Tool 调用开始 | Tool 调用记录、状态 | 自动保存 |
| 黑板更新 | 分镜、脚本、提示词、故事线节点、草稿态资产元素 | 自动保存 |
| 生成单个产物完成 | 产物引用、资产记录、最终资产元素 | 自动保存 |
| 积分确认扣减 | 扣费结果引用 | 自动保存 |
| 生成失败、取消或超时 | 已完成资产、失败状态、释放结果引用 | 自动保存 |
| SSE 断线重连 | 最新事件序号和快照 | 用于补偿和恢复 |

## 用户流程

### 生成资产保存

1. 用户在统一 Agent 会话中发起图片、音乐或视频生成。
2. 系统确认当前项目；如果没有项目，则先在当前空间下创建项目。
3. 系统完成模型选择、积分预估和用户确认。
4. 业务微服务冻结积分。
5. Agent 执行生成任务。
6. Agent 按 Skill 输出元素结构组织产物和资产元素。
7. 单个产物生成完成后，Agent 通过业务微服务创建用户可见资产和最终资产元素，并绑定当前项目。
8. 业务微服务返回资产引用和可展示资产元素摘要。
9. Agent 将资产引用和资产元素摘要写入会话、黑板和 AG-UI 事件。
10. 前端在当前项目的资产视图、产物历史和产物详情中展示该资产及其元素。
11. 业务微服务按已保存并可用的产物确认扣费。

### 创作过程保存

1. 用户进入统一 Agent 创作工作台。
2. 系统创建或恢复会话。
3. 用户消息、Agent 回复、Skill 路由、Tool 调用、黑板、草稿态资产元素、生成任务和资产引用持续保存。
4. 用户刷新页面或 SSE 断线重连时，前端根据 run 快照和事件补偿恢复创作过程。
5. 用户从会话历史进入原会话时，可以查看已保存的对话、资产视图和黑板。
6. 用户可以在原会话中引用已保存资产继续创作。

### 生成失败或取消

1. 生成任务失败、取消或超时。
2. 已完成且已保存成功的资产保留。
3. 已完成资产对应积分确认扣费。
4. 未完成或保存失败的部分不作为可用资产展示。
5. 未完成部分释放冻结积分。
6. 创作过程保存失败、取消或超时状态，供用户查看生成记录。

## 业务规则

- 资产保存和创作过程保存必须绑定当前登录身份和当前空间。
- 资产保存和创作过程保存必须绑定当前项目；会话和 run 必须能追溯到项目。
- 用户可见资产事实由业务微服务保存。
- 最终资产元素由业务微服务保存，属于用户可见资产事实的一部分。
- Agent Runtime 创作过程由 Agent 微服务保存。
- 草稿态资产元素由 Agent 微服务保存在黑板和创作过程中。
- Agent 只保存资产引用，不直接写业务资产数据库。
- Skill 只能使用平台内置固定资产元素类型声明输出结构，不允许自造不可渲染或越权元素类型。
- 前端按资产元素类型和展示提示渲染，不按具体场景硬编码固定字段。
- 用户有权限访问的资产都支持预览和下载。
- 资产文件存储在火山引擎 TOS，第一版不设置业务有效期。
- TOS 使用公共桶；前端上传使用后端签发的短期上传凭证，object key 由后端生成。
- 生成完成且资产保存成功后，该资产进入资产视图、产物历史和产物详情。
- 生成完成但资产保存失败时，不作为可用资产展示，不确认该资产对应扣费。
- 批量生成时，已保存成功的资产保留并扣费，未保存成功的部分释放冻结积分。
- 会话中的黑板、分镜、脚本、提示词和资产引用需要随创作过程自动保存。
- 产物详情第一版不提供独立“继续创作”快捷入口。
- 用户可以从会话历史进入原会话，并在会话中引用已保存资产继续创作。
- 企业拥有者不因企业积分管理权限获得成员资产或创作过程可见权限。
- 被移出企业后，用户失去企业空间资产和创作过程访问权限，个人空间资产不受影响。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| 资产保存失败 | 生成完成但业务微服务未能保存资产 | 资产保存失败 | 不展示为可用资产，不确认该资产扣费 |
| 创作过程保存失败 | 会话、黑板或事件保存失败 | 创作过程保存失败 | 提示重试或刷新，不影响已保存资产 |
| 引用资产无权限 | 用户引用不属于当前可见范围的资产 | 无权使用该资产 | 阻止引用 |
| 引用资产不存在 | 资产被删除、失效或保存失败 | 资产不可用 | 阻止引用 |
| 断线重连补偿失败 | 事件超过补偿窗口 | 已恢复到最新状态 | 使用 run 快照恢复可展示状态 |
| 批量部分保存成功 | 部分产物保存成功，部分保存失败 | 部分资产已保存 | 已保存资产可用并扣费，失败部分释放冻结积分 |
| 资产元素类型不支持 | Skill 声明了非平台内置的元素类型 | 输出结构不可用 | 阻止保存或提交审核 |
| 必填资产元素缺失 | 生成结果缺少 Skill 声明的必填元素 | 内容不完整 | 阻止转为最终资产或提示重新生成 |
| 资产元素渲染失败 | 前端无法识别元素类型或展示提示 | 内容展示异常 | 使用兜底文本摘要，并记录错误 |
| 上传格式不支持 | 用户上传了不支持的文件格式 | 文件格式不支持 | 拒绝上传，不创建资产 |
| 上传文件过大 | 用户上传文件超过大小限制 | 文件过大 | 拒绝上传，不创建资产 |
| 预览失败 | TOS 文件可访问但前端预览失败 | 预览失败，请重试 | 保留资产，允许下载或重试预览 |
| 下载失败 | 用户下载资产失败 | 下载失败，请重试 | 保留资产，允许重试下载 |

## 非目标

- 第一阶段不做资产跨空间转移。
- 第一阶段不做企业级资产库管理。
- 第一阶段不做企业拥有者查看全部成员资产和创作过程。
- 第一阶段不做资产删除、回收站和恢复流程。
- 第一阶段不在资产保存模块内做公开分享；公开分享由 [作品中心与精选作品 PRD](./prd/12-作品中心与精选作品PRD.md) 承接。
- 第一阶段不做协作编辑和多人同时编辑。
- 第一阶段不保存模型内部推理链路、系统提示词、密钥或供应商内部响应原文。
- 第一阶段不为每个创作场景硬编码资产字段。
- 第一阶段不允许 Skill 自定义任意前端组件或任意资产元素类型。
- 第一阶段不定义具体数据库字段和存储实现，后续由 API、RPC、Agent 数据模型和 SQL 脚本承接。

## 验收标准

- [ ] 生成图片、音乐、视频成功后自动保存为用户可见资产。
- [ ] 上传素材可保存为用户可见资产。
- [ ] 资产归属随当前空间确定。
- [ ] 个人空间资产仅本人可见。
- [ ] 企业空间资产第一版仅创建者本人可见，企业拥有者不额外可见。
- [ ] 资产可在同一用户可见范围内被会话引用。
- [ ] Skill 可以声明由平台内置固定资产元素类型组成的输出元素结构。
- [ ] 平台内置固定资产元素类型，不提供后台配置入口。
- [ ] 资产元素类型不散落硬编码在业务代码中。
- [ ] 草稿态资产元素保存在黑板和创作过程中。
- [ ] 最终资产元素随用户可见资产由业务微服务保存。
- [ ] 前端能按资产元素类型展示资产详情，不依赖具体场景固定字段。
- [ ] 生成完成且资产保存成功后才确认该资产对应扣费。
- [ ] 资产保存失败时不展示为可用资产，不确认该资产对应扣费。
- [ ] 批量生成部分成功时，已保存资产保留并扣费，失败部分释放冻结积分。
- [ ] 会话、消息、Agent run、Tool 调用、生成任务、黑板、分镜、脚本、提示词和资产引用会自动保存。
- [ ] 断线重连后可以恢复已保存创作过程。
- [ ] 用户可以从会话历史进入原会话查看对话、资产视图和黑板。
- [ ] 用户可以在原会话中引用已保存资产继续创作。
- [ ] 被移出企业后，用户不能访问企业空间资产和创作过程。
- [ ] Agent 微服务只保存资产引用，不直接写业务资产数据库。
- [ ] 上传素材在上传时校验格式、大小、MIME 类型和文件有效性。
- [ ] 有权限访问的资产都支持预览和下载。
- [ ] 资产文件存储在火山引擎 TOS，第一版不设置业务有效期。
