## 背景

本清单基于 `docs/product/prd/**`、当前 active 产品/设计文档、`docs/contracts/**`、`docs/standards/**` 和 `docs/test/**` 汇总第一版功能点，作为后续按功能模块拆票、排期、开发和验收的统一索引。第一阶段服务端历史设计仅作为追溯依据，不作为本文件的默认事实源。

`docs/product/prd/00-系统概要与功能大纲PRD.md` 已给出模块级大纲；本文件进一步拆到查询、新增、修改、启停、预览、确认、执行、取消、补偿、验证等动作粒度。

## 目标

- 按功能模块列出第一版所有用户可感知或端侧依赖的功能点。
- 标注每个功能点属于用户端、管理端，或用户端和管理端公用。
- 对 Agent、RPC、AG-UI、业务服务和测试类能力，用端侧依赖关系标明其服务边界。
- 为后续功能切片、契约冻结、开发、测试报告和 GitHub 任务拆分提供索引。

## 非目标

- 不替代各 PRD、当前契约或技术设计的字段级定义。
- 不新增业务范围，不把第一版非目标纳入开发清单。
- 不定义数据库字段、接口字段编号、OpenAPI schema 或 AG-UI payload 细节。
- 不包含前端实现任务、部署上线、CI/CD、告警、SLO 或生产运维手册。

## 边界检查

### 智能体微服务边界

- 负责统一 Agent、Eino Runtime、Skill 路由、TurnLoop、Tool 调用、AG-UI 事件、Agent API、Agent DB、黑板、Runtime snapshot、Memory 和 RPC client。
- 不直接访问或写入业务数据库。
- 不保存用户、企业、积分账户、资产事实、作品事实、业务权限或业务审计事实。

### 业务系统微服务边界

- 负责账户、企业、项目、模型、Tool 策略、Skill 生命周期、积分、兑换码、资产、作品、通知、审计、业务 HTTP API 和业务 RPC。
- 不负责 Eino 编排、Agent message/event/tool_call、黑板和 Agent 记忆。

### RPC 契约边界

- RPC 表达业务能力，不表达数据库表 CRUD。
- 写操作必须有幂等键；高风险写操作使用 preview / confirm 或等价确认语义。
- Agent 调业务能力必须走 RPC，不跨边界读取业务数据库。

### AG-UI 事件协议边界

- Agent 只生产文档化 canonical `type` 事件。
- 前端只消费协议定义的事件和 payload；unknown event 必须兼容忽略。
- AG-UI 不暴露 Eino 内部节点、系统 Prompt、模型推理链路、供应商原始响应、API Key 和内部成本。

### Agent 领域数据库边界

- Agent DB 保存 session、run、message、event、tool_call、task、interrupt、artifact、snapshot、memory 和 runtime config。
- 业务事实以业务服务和业务 DB 为准，Agent 侧只保存业务返回的引用和展示摘要。
