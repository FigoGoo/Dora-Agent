# 产品迭代文档入口

状态：active  
owner：产品与需求责任域  
更新时间：2026-06-28  
适用范围：产品变更、功能迭代、PRD 修订和验收口径变更记录

## 目标

本目录用于承接后续产品迭代，不再把新产品变更追加到第一版前置产品系统设计或历史归档文档中。

## 何时新建

- 新增产品能力、用户流程、页面范围、角色权限或验收标准。
- PRD 已 Done，但后续出现范围调整或优先级变化。
- 技术实现发现产品口径不完整，需要先补产品决策。
- 前端阶段需要把页面、状态、文案和异常流重新固化。

## 命名规则

推荐格式：`YYYY-MM-DD-功能名产品迭代.md`。

## 内容要求

使用 `docs/templates/产品迭代文档模板.md`。至少包含变更背景、目标用户、范围、非目标、用户流程、验收标准、影响 PRD、影响契约和测试入口。

## 与 PRD 的关系

- 小范围调整写入产品迭代文档，并在对应 PRD 的变更记录中引用。
- 影响核心流程、权限、计费、安全或跨服务契约的调整，必须同步更新对应 PRD。
- 产品迭代完成后，如被阶段总结承接，可在本目录保留 `archived` 状态或迁移到 `docs/releases/**`。
