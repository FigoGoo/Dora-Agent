## 背景

Skill 是统一 Agent 可选择和执行的可配置能力。产品已确认：Skill 不是一段 prompt，必须支持完整创建、测试、发布和生命周期管理。

## 目标

- 支持平台管理员创建系统 Skill。
- 支持企业拥有者创建企业 Skill。
- 支持普通用户创建个人 Skill。
- 保证所有 Skill 只能使用平台开放 Tool。
- 保证 Skill 不写入业务规则，不绕过业务微服务改变业务事实。

## 用户角色

| 角色 | 权限/特征 | 核心诉求 |
| --- | --- | --- |
| 平台管理员 | 创建、编辑、发布、废弃系统 Skill | 建立平台级默认能力 |
| 企业拥有者 | 创建、编辑、发布、废弃企业 Skill，管理企业 Skill 权限 | 为企业成员配置共用能力 |
| 普通个人用户 | 创建、编辑、发布、废弃个人 Skill | 为自己配置个性化能力 |
| 企业普通成员 | 使用企业 Skill，不能创建企业 Skill | 使用企业配置好的能力 |

## Skill 类型

- 系统 Skill：由平台管理员创建，平台范围可用。
- 企业 Skill：由企业拥有者创建，企业范围可用。
- 个人 Skill：由普通用户创建，仅本人可用。

## Skill 生命周期

- Draft：草稿，可编辑，不参与正式路由。
- PendingReview：已提交审核，等待平台管理员审核。
- Published：已发布，可被统一 Agent 路由使用。系统 Skill 可由平台管理员测试通过后直接发布；企业 Skill 和个人 Skill 需审核通过后发布。
- Rejected：审核未通过，需修改后重新提交审核。
- Deprecated：已废弃，不再被新任务使用，历史记录保留。

## Skill 内容

创建 Skill 时必须填写完整内容：

- 基础信息：名称、描述、类型、作用域、状态、版本。
- 适用意图：Agent 用于判断何时选择该 Skill。
- 行为指令：Skill 的执行目标、约束和禁止事项。
- 输入结构：用户需要提供或 Agent 可推断的输入。
- 输出结构：文本、图片、音乐、视频、文件、任务状态等输出约定。
- 输出元素结构：声明该 Skill 会产出的资产元素，且只能使用平台内置固定资产元素类型。
- 可用 Tool：只能从平台开放 Tool 中选择。
- 执行步骤：第一阶段采用表单化步骤列表，不做可视化拖拽编排。
- Memory 设置：是否启用记忆、记忆作用域和保留策略。
- 人工确认规则：涉及扣费、高风险 Tool、业务事实写入时必须确认。
- 安全策略：内容安全、Tool 权限、敏感信息限制。
- 测试样例：发布前至少需要 3 个测试样例，用于验证 Skill 是否可用。
- 发布与版本：草稿、发布、废弃状态和历史版本。

Skill Memory 默认开启，创建者可以在 Skill 配置中调整 Memory 设置。

Skill 默认展示最新版本。历史版本按版本号展示，支持按版本回滚。

## Skill 输出元素结构

Skill 输出元素结构用于声明该 Skill 在黑板和最终资产详情中会产出哪些可展示、可编辑、可引用的资产元素。

第一版规则：

- Skill 不能按场景自造任意字段，只能使用平台内置固定资产元素类型。
- Skill 可以组合多个资产元素类型，形成适合当前创作场景的输出结构。
- Skill 输出元素结构需要区分草稿态元素和最终资产元素。
- 草稿态元素展示在黑板，并随创作过程保存。
- 最终资产元素随用户可见资产保存，并进入资产详情、产物历史和后续会话引用。
- 输出元素结构需要声明元素名称、元素类型、是否必填、是否可编辑、是否可引用和展示位置。
- 输出元素结构需要纳入 Skill 测试样例，验证 Agent 是否能产出可保存、可展示的元素。
- 输出元素结构不能关闭或绕过 LLM 提示词安全评估。

示例，不作为固定字段：

| Skill 场景 | 输出元素结构示例 |
| --- | --- |
| 音乐创作 | 标题、歌词、风格标签、封面图片引用、音频引用、提示词 |
| MV 制作 | 故事线、分镜列表、歌词引用、音乐引用、视频引用 |
| 电商商品图 | 商品信息、卖点文案、参考图、提示词、生成图片引用 |
| 品牌 LOGO | 品牌名、品牌关键词、色板、字体建议、LOGO 图片引用 |

## 用户流程

### 平台管理员创建系统 Skill

1. 平台管理员进入系统 Skill 管理。
2. 新建系统 Skill。
3. 填写完整 Skill 内容。
4. 从平台开放 Tool 中选择可用 Tool。
5. 添加至少 3 个测试样例并验证。
6. 平台管理员直接发布系统 Skill。

### 企业拥有者创建企业 Skill

1. 企业拥有者进入企业空间。
2. 新建企业 Skill。
3. 填写完整 Skill 内容。
4. 从平台开放 Tool 中选择可用 Tool。
5. 添加至少 3 个测试样例并验证。
6. 提交平台管理员审核。
7. 平台管理员审核通过后发布企业 Skill，企业成员全部可见。
8. 审核通过或审核拒绝后，通过站内信通知创建者。

### 普通用户创建个人 Skill

1. 普通用户进入个人 Skill 管理。
2. 新建个人 Skill。
3. 填写完整 Skill 内容。
4. 从平台开放 Tool 中选择可用 Tool。
5. 添加至少 3 个测试样例并验证。
6. 提交平台管理员审核。
7. 平台管理员审核通过后发布个人 Skill，仅本人可使用。
8. 审核通过或审核拒绝后，通过站内信通知创建者。

## 业务规则

- 平台管理员才能创建系统 Skill。
- 企业 Skill 只能由企业拥有者创建。
- 企业普通成员不能创建企业 Skill。
- 普通用户只能创建个人 Skill。
- Skill 只能绑定平台开放 Tool。
- Skill 只能使用平台内置固定资产元素类型声明输出元素结构。
- Draft、PendingReview、Rejected 和 Deprecated Skill 不参与统一 Agent 路由。
- Skill 发布前必须至少配置 3 个测试样例。
- 平台管理员创建系统 Skill 不需要审核，测试通过后可直接发布。
- 企业 Skill 和个人 Skill 发布需要平台管理员审核通过。
- 平台管理员审核企业 Skill 和个人 Skill 时，审核意见非必填。
- 企业 Skill 和个人 Skill 审核通过或审核拒绝后，通过站内信通知创建者。
- Skill Memory 默认开启。
- Skill 默认展示最新版本，支持按历史版本回滚。
- 企业 Skill 不支持部门或成员级可见范围，企业成员全部可见。
- Skill 不能写入业务规则或业务事实。
- Skill 不能绕过业务微服务直接修改业务数据。
- Skill 不能暴露供应商密钥、内部模型配置或平台成本信息。
- Skill 输出元素结构不能要求前端实现未开放的自定义组件。
- Skill 不能关闭或绕过 LLM 提示词安全评估。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| 权限不足 | 企业普通成员创建企业 Skill | 无权创建企业 Skill | 拒绝操作 |
| Tool 不可用 | 选择了停用或无权限 Tool | 当前 Tool 不可用 | 阻止保存或发布 |
| 测试失败 | 发布前测试样例无法通过 | Skill 测试未通过 | 保持 Draft |
| 测试样例不足 | 发布前少于 3 个测试样例 | 请至少添加 3 个测试样例 | 阻止发布或提交审核 |
| 审核未通过 | 平台管理员拒绝发布 | Skill 审核未通过 | 状态变为 Rejected |
| 缺少必填内容 | Skill 内容不完整 | 请补全必填信息 | 阻止保存或发布 |
| 安全策略不通过 | 指令包含越权或敏感内容 | Skill 不符合安全策略 | 阻止发布 |
| 输出元素类型不支持 | Skill 使用非平台内置资产元素类型 | 输出元素类型不可用 | 阻止保存、测试或发布 |
| 输出元素测试失败 | 测试样例未产出必填资产元素 | 输出结构测试未通过 | 保持 Draft |

## 非目标

- 第一阶段不做可视化拖拽 Workflow / Graph 编排。
- 第一阶段不让普通用户、企业拥有者或企业成员创建 Tool。
- 第一阶段不让 Skill 直接配置模型供应商 API Key。
- 第一阶段不把业务规则写入 Skill。
- 第一阶段不支持企业 Skill 按部门或成员级可见。
- 第一阶段不允许 Skill 自定义任意资产元素类型或前端渲染组件。
