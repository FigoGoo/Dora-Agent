## 背景

产品已确认：模型供应商只由平台管理员配置，普通用户完全不能接入自己的 API Key。普通用户只能看到模型名称，不能看到供应商和内部配置。

## 目标

- 支持平台管理员配置模型供应商 API Key。
- 支持平台管理员维护模型名称、模型类型和启用状态。
- 支持平台管理员为生成类模型配置用户积分单价。
- 支持平台管理员配置和查看模型内部成本。
- 支持 API Key 连通性测试。
- 支持统一 Agent 和模型 Tool 使用平台配置的模型。
- 对普通用户隐藏供应商、密钥、endpoint、内部模型 ID、成本信息和完整价格表。

## 模型类型

第一阶段只支持以下模型类型：

| 模型类型 | 用途 |
| --- | --- |
| 文本模型 | 对话、意图识别、Skill 路由、文案生成、脚本生成 |
| 视觉理解模型 | 理解图片、分析封面、识别图像内容 |
| 图片生成模型 | 文生图、图生图、风格化 |
| 音乐生成模型 | 文生音乐、歌词生成歌曲、BGM |
| 视频生成模型 | 文生视频、图生视频、分镜转视频 |
| 语音模型 | TTS、ASR，后续再做 |

## 管理对象

供应商配置第一阶段只需要 API Key。

模型管理字段以产品语义为准：

- 模型名称：普通用户可见。
- 模型类型：文本、视觉理解、图片生成、音乐生成、视频生成、语音。
- 启用状态：启用或停用。
- 默认模型标记：同一生成类型可配置一个平台默认模型。
- 供应商关联：平台内部可见。
- API Key 引用：平台内部可见。
- 内部模型标识：平台内部可见。
- 内部成本：平台管理员可见，用于成本管理，普通用户不可见。
- 用户积分单价：生成类模型必填，平台内部管理，用于生成前积分预估。

模型配置第一阶段不做版本管理，但需要保留操作日志或审计日志。

## 用户流程

1. 平台管理员进入模型供应商管理。
2. 新增或编辑供应商 API Key。
3. 执行 API Key 连通性测试。
4. 进入模型管理。
5. 创建模型，填写模型名称、模型类型、供应商关联和内部模型标识。
6. 配置模型内部成本。
7. 如果是图片、音乐或视频生成模型，配置该模型的用户积分单价。
8. 如需作为默认模型，配置该生成类型的平台默认模型。
9. 启用模型。
10. 模型进入平台可用模型池。
11. 统一 Agent、模型 Tool 或模型选择功能按模型类型使用可用模型。

## 业务规则

- 只有平台管理员可以配置模型供应商和模型。
- API Key 需要支持连通性测试。
- 普通用户不能接入自己的 API Key。
- 普通用户不能看到供应商、API Key、endpoint、内部模型 ID、内部成本、调用参数等平台内部信息。
- 普通用户只能在生成前看到本次预计消耗积分，不看到完整模型价格表。
- 模型显示名称允许重名。
- 模型内部成本本期需要记录，并在平台管理员后台展示。
- 图片、音乐、视频生成模型必须配置用户积分单价。
- 不同生成类模型可以配置不同用户积分单价。
- 用户积分单价允许为 0，用于免费或活动模型。
- 图片、音乐、视频生成类型分别需要有平台默认模型，用于用户不主动选择模型时使用。
- 停用模型不能作为平台默认模型。
- 文本模型由平台默认配置，普通用户不能选择。
- 后台不配置默认图片生成参数、音乐生成参数、视频生成参数。
- 生成参数由用户本次生成输入传入，或使用 Tool/模型供应商默认值。
- 语音模型 TTS / ASR 后续再做。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| API Key 无效 | 平台管理员保存无效密钥 | API Key 校验失败 | 阻止启用或标记异常 |
| API Key 连通性失败 | 平台管理员执行连通性测试失败 | API Key 连通性测试失败 | 标记异常，不影响已启用模型的既有配置 |
| 模型停用 | 模型被平台管理员停用 | 该模型当前不可用 | 不出现在用户可选列表 |
| 默认模型停用 | 平台管理员停用当前默认模型 | 请先设置新的默认模型 | 阻止停用或要求先切换默认模型 |
| 供应商异常 | 供应商调用失败或超时 | 生成失败，可稍后重试 | 记录失败并返回明确错误 |
| 普通用户访问内部信息 | 普通用户请求供应商或密钥信息 | 无权访问 | 拒绝返回内部字段 |

## 非目标

- 第一阶段不允许普通用户、企业拥有者或企业成员接入自己的模型 API Key。
- 第一阶段不做模型配置版本管理。
- 第一阶段不做模型参数后台默认配置。
- 第一阶段不做语音模型 TTS / ASR。
- 第一阶段不做供应商异常后台告警、SLO 和可观测性运行手册；上线阶段再补充。
