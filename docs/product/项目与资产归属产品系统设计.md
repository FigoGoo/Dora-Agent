## 背景

用户在统一 Agent 对话中会持续产生图片、音乐、视频、脚本、分镜、提示词、上传素材和中间资产。如果只把这些内容挂在会话下，用户换会话、从首页继续创作或从资产创建作品时，会缺少一个稳定的创作目标容器。

第一版需要补充“项目 Project”概念。项目不是新的 Agent，也不是企业项目管理工具，而是用户可理解的创作容器：一次创作目标下的会话、run、黑板、资产和作品都可以聚合到同一个项目中。

## 目标

- 定义项目在产品中的定位和边界。
- 明确会话、run、资产、黑板和作品挂靠到项目的规则。
- 让首页、工作台、资产库和作品中心都能解释资产来源。
- 明确项目业务事实由业务微服务维护，Agent 微服务只保存 `project_id` 上下文和 Runtime 过程。
- 给后续 API、RPC、AG-UI、Agent 数据模型和页面设计提供输入。

## 非目标

- 第一版不做多人协作项目。
- 第一版不做项目文件夹、模板、标签体系或项目市场。
- 第一版不做企业项目管理，不允许企业拥有者查看成员项目内容。
- 第一版不做跨空间移动项目。
- 第一版不做项目删除回收站；可先支持归档。
- 项目不是公开展示对象，公开展示由作品和公开快照承担。

## 核心概念

| 概念 | 定义 | 主要 owner |
| --- | --- | --- |
| 项目 Project | 用户围绕一个创作目标形成的容器，聚合会话、资产、黑板和作品 | 业务微服务 |
| 会话 Session | 项目下的一段可恢复对话和上下文 | Agent 微服务 |
| Run | 会话中一次 Agent 执行，包含用户输入、模型选择、Tool 调用、确认和生成结果 | Agent 微服务 |
| 资产 Asset | 用户可见、可预览、可下载、可引用的生成结果或上传素材 | 业务微服务 |
| 黑板 Blackboard | 项目创作过程中的结构化草稿和最新工作区状态 | Agent 微服务 |
| 作品 Work | 用户从项目资产中整理出的可管理、可分享展示对象 | 业务微服务 |

## 归属关系

```text
Project
  -> Sessions
       -> Runs
            -> Messages / Events / Tool calls / Interrupts / Blackboard snapshots
  -> Assets
       -> source_session_id / source_run_id / source_artifact_id
  -> Blackboard
       -> latest project view + run snapshots
  -> Works
       -> created from project assets
```

归属规则：

- 一个项目必须归属于一个当前空间和一个创建者。
- 一个会话必须归属于一个项目。
- 一个 run 必须归属于一个会话，并通过会话归属于项目。
- 用户在会话中生成或上传的资产默认归属于当前项目。
- 资产必须保留来源会话、来源 run 或来源上传动作的引用摘要。
- 黑板以项目为用户可见聚合口径，同时保留 run 级快照用于恢复。
- 作品从项目资产中创建，作品可以引用一个项目下的多个资产。
- 项目本身不公开；作品分享后生成公开快照，公开快照不得暴露项目、会话、黑板、提示词、积分或模型成本。

## 用户流程

### 从首页开始创作

1. 用户在首页输入 Prompt，或选择快捷 Skill。
2. 系统在当前空间下创建项目，项目标题可由第一条 Prompt 摘要生成。
3. 系统创建项目下的第一条会话，并进入统一 Agent 工作台。
4. Agent run 生成的资产、黑板更新和过程快照都归属当前项目。

### 在已有项目继续创作

1. 用户从最近项目、项目列表或项目详情进入项目。
2. 用户可以打开历史会话恢复，也可以在同一项目下新建会话。
3. 新会话可以引用该项目已有资产。
4. 新生成资产继续归属于该项目，并保留来源会话和 run。

### 从项目资产创建作品

1. 用户进入项目详情或作品中心创建作品。
2. 用户从当前项目资产中选择封面、图片、音乐、视频或文件。
3. 系统创建作品并保存作品资产引用。
4. 用户分享作品后，业务微服务生成公开快照。

## 页面规则

| 页面 | 项目相关规则 |
| --- | --- |
| 首页 | “开始”默认创建项目；最近项目展示项目卡片而不是裸会话。 |
| 工作台 | 顶部显示当前项目；新建会话、恢复会话、资产和黑板都在当前项目上下文内。 |
| 项目列表 | 展示当前用户在当前空间下有权访问的项目。 |
| 项目详情 | 聚合项目会话、资产、黑板和作品，提供继续创作入口。 |
| 资产库 | 支持按项目筛选；资产详情展示所属项目和来源会话摘要。 |
| 作品中心 | 创建作品时优先从项目资产选择，作品详情可回到所属项目。 |

## 业务规则

- 项目创建必须绑定当前登录身份、当前空间和创建者。
- 个人空间项目只有创建者本人可见。
- 企业空间项目第一版仍只对创建者本人可见；企业拥有者不额外查看成员项目、资产、会话、黑板或生成记录。
- 当前空间切换后，项目列表、最近项目、资产和作品必须刷新为目标空间上下文。
- 归档项目后不再出现在默认最近项目中，但资产和作品不被删除。
- 项目标题、封面、最近更新时间可以由业务服务维护；自动标题可由 Agent 建议，但最终项目事实由业务服务保存。
- 会话恢复必须校验用户对项目和会话的访问权限。
- 引用资产时必须校验资产与当前项目、当前空间和当前用户的访问关系。

## 保存边界

业务微服务负责：

- 项目事实、状态、标题、封面、归档状态和列表。
- 项目和资产、作品的业务关联。
- 项目下资产、作品、空间、用户权限的最终校验。
- 项目相关写操作审计和幂等。

Agent 微服务负责：

- 会话、run、消息、事件、Tool 调用、中断、任务和黑板快照。
- 在 Agent Runtime 数据中保存 `project_id` 引用。
- 根据业务服务返回的项目上下文生产 AG-UI 事件和恢复快照。
- 通过 RPC 校验项目访问权限和资产引用权限。

Agent 微服务不能直接创建或修改业务项目数据库。需要创建项目、校验项目权限、绑定资产或读取项目摘要时，必须通过业务微服务 API / RPC。

## 契约输入

| 契约 | 需求 |
| --- | --- |
| API | 创建项目、项目列表、项目详情、归档项目、重命名项目、项目资产、项目会话、项目作品。 |
| RPC | `CreateProject`、`GetProject`、`ListProjects`、`UpdateProjectTitle`、`ArchiveProject`、`CheckProjectAccess`、`AttachAssetToProject`。 |
| Agent API | 创建会话和 run 时必须携带或解析项目上下文。 |
| AG-UI | 工作台事件公共字段建议包含 `project_id`，用于前端稳定归属和恢复。 |
| Agent DB | session、run、event、artifact 引用建议保存 `project_id` 普通字段，不把项目业务事实复制到 Agent DB。 |

## 验收标准

- [ ] 用户从首页开始创作时能自动形成项目。
- [ ] 用户能从最近项目或项目列表继续创作。
- [ ] 会话必须能解释自己属于哪个项目。
- [ ] 资产必须能解释所属项目和来源会话 / run。
- [ ] 黑板能按项目展示最新工作区，并能按 run 快照恢复。
- [ ] 作品能从项目资产创建，公开作品不暴露项目过程信息。
- [ ] 企业空间第一版不扩展成员项目可见性。
- [ ] Agent 微服务只保存 `project_id` 引用，不保存项目业务事实。
- [ ] 业务微服务拥有项目、资产、作品归属的最终解释权。
