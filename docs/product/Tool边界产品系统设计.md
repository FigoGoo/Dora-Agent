## 背景

产品已确认：所有 Skill 只能使用平台开放的 Tool。除平台管理员外，普通用户、企业拥有者和企业成员不能创建新的 Tool。

Tool 是 Agent 可执行能力的边界，也是 Agent 与业务事实、模型供应商和人工确认之间的受控通道。第一版内容安全治理由统一 Agent 使用 LLM 做提示词安全评估，不作为普通 Skill 可绑定 Tool。

第一版只允许平台内置 Tool 开关和白名单，不允许创建任意 HTTP Tool。

## 目标

- 明确 Tool 是平台提供的可执行能力。
- 限制 Skill 只能绑定平台开放 Tool。
- 避免 Agent 或 Skill 绕过业务微服务直接改变业务事实。
- 对高风险、扣费和业务写入 Tool 建立人工确认边界。
- 支持平台管理员配置 Tool 风险等级。
- 支持按企业、用户等级或套餐限制 Tool 可用范围。

## Tool 类型

| Tool 类型 | 描述 | 备注 |
| --- | --- | --- |
| 模型 Tool | 调用文本、视觉理解、图片生成、音乐生成、视频生成等模型 | 依赖模型供应商配置 |
| 文件 Tool | 读取或处理用户上传文件 | 需遵守权限和安全策略 |
| 产物 Tool | 保存、查询、展示生成产物 | 不直接替代业务资产规则 |
| 业务 RPC Tool | 通过业务微服务 RPC 改变或查询业务事实 | 业务事实唯一写入通道 |
| 人工确认 Tool | 在高风险或扣费操作前触发确认 | 与 AG-UI 交互关联 |
| 媒体处理 Tool | 转码、裁剪、拼接等媒体处理能力 | 后续扩展 |

## Tool 风险等级

- low：低风险读取或轻量处理，不需要人工确认。
- medium：可能影响用户体验、成本或产物状态，按场景要求确认。
- high：涉及扣费、业务事实写入、发布、删除或不可逆操作，必须人工确认。

Tool 风险等级由平台管理员配置。系统可以提供默认风险等级，但平台管理员拥有最终启停和风险等级配置权。

## Tool 可用范围

平台管理员可以按以下维度限制 Tool 可用范围：

- 平台全局启用或停用。
- 企业可用范围。
- 用户等级。
- 套餐。
- 当前空间：个人空间或企业空间。

Skill 绑定 Tool、Agent 通过 Skill 执行 Tool、Agent 按用户意图直接执行平台基础 Tool 时，都必须同时满足平台开放、当前空间、企业、用户等级和套餐限制。

## Tool 执行策略

第一版执行策略按 Tool 类型和风险等级采用保守默认值。第一版不继续按更细 Tool 子类型拆分默认超时。

| Tool 类别 | 默认超时 | 自动重试 | 取消策略 |
| --- | --- | --- | --- |
| low 风险只读 Tool | 10 秒 | 最多 1 次 | 可取消，取消后不产生业务影响 |
| medium 风险处理 Tool | 30 秒 | 仅幂等失败可重试 1 次 | 可取消，已完成步骤保留 |
| high 风险 Tool | 60 秒 | 默认不自动重试 | 取消前必须确认，已执行成功部分不回滚 |
| 模型生成 Tool | 按模型或任务类型配置 | 默认不自动重试 | 取消后按已完成产物结算，未完成部分释放冻结积分 |
| 业务 RPC 写入 Tool | 按 RPC 契约配置 | 默认不自动重试，除非契约明确幂等可重试 | 取消不撤销已确认成功的业务写入 |

超时规则：

- Tool 必须有超时配置。
- 平台管理员可以在平台允许范围内配置 Tool 超时。
- 超时后 Agent 输出明确失败或可重试状态。
- 长任务类 Tool 应返回任务状态，不应无限阻塞对话。

重试规则：

- 只有幂等 Tool 才允许自动重试。
- 业务写入、扣费、高风险、外部副作用 Tool 默认不自动重试。
- 业务 RPC 写入 Tool 如需重试，必须由 RPC 契约明确幂等键和可重试错误码。
- 模型生成 Tool 默认不自动重试，避免重复生成和重复扣费；如后续支持重试，必须复用同一任务和冻结记录。

取消规则：

- 用户取消后，不再发起新的 Tool 调用。
- 已经成功完成的 Tool 结果保留。
- 已完成生成产物按积分扣费规则确认扣费。
- 未完成生成部分释放冻结积分。
- 已确认成功的业务写入不因取消自动回滚；如需撤销，必须通过单独业务能力处理。

## 业务 RPC Tool 规则

业务 RPC Tool 是改变业务事实的唯一 Tool 类型，必须遵守：

- 通过业务微服务 RPC 调用，不直接访问业务数据库。
- 写操作必须支持幂等键。
- 高风险或扣费操作必须支持 preview / confirm。
- 必须带用户、企业、权限、审计上下文。
- 错误码和失败原因必须可被 Agent 和前端展示。
- RPC 方法表达业务能力，不表达数据库表操作。

## 用户流程

1. 平台管理员配置或启用平台开放 Tool。
2. 平台管理员配置 Tool 风险等级、超时、可用企业、用户等级和套餐限制。
3. 创建 Skill 时，创建者只能从平台开放 Tool 中选择。
4. 统一 Agent 路由到 Skill 后，只能调用该 Skill 绑定且当前用户有权限的 Tool。
5. 当没有匹配 Skill，但用户意图明确匹配平台开放基础 Tool 时，统一 Agent 可以按 Tool 边界直接调用该 Tool。
6. Tool 风险等级为 high 或涉及扣费、业务事实写入时，Agent 先触发人工确认。
7. 用户确认后，Agent 执行 Tool。
8. Tool 执行成功、失败、取消或超时后，Agent 输出明确状态。

## 业务规则

- 平台管理员控制平台开放 Tool。
- 第一版只允许平台内置 Tool 开关和白名单。
- 第一版不允许创建任意 HTTP Tool。
- 第一版不继续细分不同 Tool 子类型的默认超时，沿用当前按风险等级和任务类别的执行策略。
- 平台管理员配置 Tool 风险等级。
- Tool 可以按企业、用户等级或套餐限制可用范围。
- 除平台管理员外，普通用户、企业拥有者和企业成员不能创建新的 Tool。
- Skill 只能绑定平台开放 Tool。
- Skill 不能调用未开放 Tool。
- 当没有匹配 Skill 时，统一 Agent 可以基于用户意图直接调用平台开放基础 Tool，但必须满足当前空间、企业、用户等级、套餐、风险等级和人工确认规则。
- 业务事实变更必须通过业务 RPC Tool。
- 高风险 Tool 必须人工确认。
- 每个 Tool 必须配置超时。
- 只有幂等 Tool 才允许自动重试。
- 业务写入、扣费、高风险、外部副作用 Tool 默认不自动重试。
- 用户取消后，不再发起新的 Tool 调用；已完成结果保留，未完成生成部分释放冻结积分。
- Tool 不能暴露供应商密钥和内部系统配置。
- 内容安全治理第一版不作为 Skill 可绑定 Tool，由统一 Agent 在生成前固定执行 LLM 提示词安全评估。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| Tool 未开放 | Skill 引用未开放 Tool | 当前 Tool 不可用 | 阻止保存、发布或执行 |
| Tool 权限不足 | 当前用户无权使用 Tool | 无权使用该能力 | 拒绝调用 |
| 用户拒绝确认 | 高风险 Tool 执行前用户拒绝 | 已取消 | 不执行 Tool |
| RPC 调用失败 | 业务 RPC Tool 返回错误 | 操作失败 | 展示业务错误并记录 |
| Tool 超时 | Tool 执行超过限制 | 操作超时 | 中止，按重试规则判断是否可重试 |
| Tool 重试受限 | Tool 非幂等或高风险 | 当前操作不可自动重试 | 不自动重试，提示用户重新发起或等待人工处理 |
| Tool 取消 | 用户取消任务 | 已取消 | 不再发起新 Tool，按已完成和未完成部分处理 |

## 非目标

- 第一阶段不允许普通用户、企业拥有者或企业成员创建 Tool。
- 第一阶段不做任意 HTTP Tool 创建能力。
- 第一阶段不允许 Tool 直接访问业务数据库。
- 第一阶段不把内容安全治理做成 Skill 可绑定 Tool。
- 第一阶段不在产品文档中定义具体 RPC 字段，后续由契约文档承接。
