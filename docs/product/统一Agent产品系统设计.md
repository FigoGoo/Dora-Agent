## 背景

当前产品方向是不做多 Agent 体系，而是提供一个统一 Agent Runtime。统一 Agent 根据当前用户、企业上下文和用户意图，选择可用 Skill 完成音乐、图片、视频等 AIGC 创作任务。

## 目标

- 建立一个统一 Agent Runtime，而不是为系统、企业、个人分别创建不同 Agent。
- 让 Agent 配置能力不干涉任何业务规则。
- 让 Agent 根据用户意图选择不同 Skill；没有合适 Skill 时，使用文本模型继续对话。
- 让 Agent 只通过平台开放 Tool 或业务 RPC Tool 触发外部能力。
- 为 AG-UI 流式输出、任务进度、Tool 调用状态和人工确认预留产品边界。
- 覆盖音乐创作、MV 制作、短视频制作、电商商品图、电商广告视频、文旅宣传视频、品牌 LOGO 等典型创作场景。

## 用户角色

| 角色 | 权限/特征 | 核心诉求 |
| --- | --- | --- |
| 平台管理员 | 配置系统 Skill、模型、平台开放 Tool | 控制统一 Agent 可用能力 |
| 普通个人用户 | 使用系统 Skill 和个人 Skill | 通过一个 Agent 完成个人创作 |
| 企业拥有者 | 使用系统、企业、个人 Skill，管理企业成员、企业积分和企业 Skill | 通过一个 Agent 完成企业空间创作 |
| 企业普通成员 | 使用企业可见 Skill 和个人 Skill | 在权限范围内完成创作 |

## 核心规则

统一 Agent 必须基于当前空间加载 Skill 池。

当前空间由登录页面身份选择或用户在个人中心切换身份确定。用户在登录页面选择个人登录时，统一 Agent 使用个人空间；选择企业登录时，统一 Agent 使用企业空间。身份切换等价于退出当前登录身份并重新登录目标身份。

个人空间 Skill 池包括：

- 平台系统 Skill。
- 用户个人 Skill。

企业空间 Skill 池包括：

- 平台系统 Skill。
- 用户所属企业的企业 Skill。
- 用户个人 Skill。

企业空间可以使用用户个人 Skill，但生成产物归属企业，扣企业积分。如果个人 Skill 绑定的 Tool 不符合企业权限或平台开放规则，则该 Skill 在企业空间不可用。

统一 Agent 不承载业务规则最终解释权，不直接访问业务数据库，不直接写入业务事实。需要业务读写时，必须通过业务 RPC Tool 调用业务微服务。

## 用户流程

1. 用户通过登录页面选择个人登录或企业登录，或在个人中心切换当前身份，系统确定当前登录身份和当前空间。
2. 用户从首页新建创作或进入已有项目，系统确定当前项目。
3. 用户打开统一 Agent 对话框。
4. 用户输入意图，例如生成图片、生成音乐、生成视频或创作文案。
5. 系统加载当前用户可用 Skill 池。
6. Agent 使用文本模型识别意图并选择 Skill。
7. 如果没有合适 Skill，Agent 使用文本模型继续对话，不提示用户创建 Skill。
8. 如果用户意图匹配平台开放 Tool，Agent 可以按 Tool 边界、权限和人工确认规则调用 Tool。
9. Agent 根据 Skill 或意图匹配结果决定是否需要 Tool、模型选择、积分预估或人工确认。
10. 生成类任务在积分冻结和生成 Tool 执行前，使用 LLM 对用户提示词和生成前提示词做安全评估。
11. 提示词安全评估通过后，Agent 执行任务，并持续保存会话、事件、Tool 调用、黑板和资产引用。
12. 生成产物完成后，Agent 通过业务微服务保存用户可见资产，并绑定当前项目。
13. Agent 通过 AG-UI 输出流式回复、Tool 调用状态、任务进度和产物结果。
14. 任务成功、失败、取消或超时时，Agent 输出明确状态，并保存创作过程最终状态。

## Agent 能力清单

| 能力 | 输入 | 输出 | Tool/RPC 依赖 | 是否需要人工确认 |
| --- | --- | --- | --- | --- |
| 意图识别与 Skill 路由 | 用户消息、当前空间、可用 Skill 池 | 选中的 Skill、路由结果 | 文本模型 | 否 |
| 文本对话兜底 | 用户消息、当前空间、对话上下文 | 文本回复、可选 Tool 调用意图 | 文本模型、平台开放 Tool | 按 Tool 风险等级 |
| Skill 执行 | Skill 配置、用户输入、上下文 | 文本回复、任务状态、产物引用 | Skill 绑定 Tool | 按 Tool 风险等级 |
| 图片生成任务编排 | prompt、参考图、风格、数量、模型选择 | 图片产物、生成状态 | 图片生成模型 Tool、积分 RPC | 是 |
| 音乐生成任务编排 | prompt、歌词、风格、时长、模型选择 | 音乐产物、生成状态 | 音乐生成模型 Tool、积分 RPC | 是 |
| 视频生成任务编排 | prompt、图片、分镜、时长、模型选择 | 视频产物、生成状态 | 视频生成模型 Tool、积分 RPC | 是 |
| 视觉理解 | 图片、分析目标 | 图片分析结果 | 视觉理解模型 Tool | 否 |
| 提示词安全评估 | 用户提示词、生成前提示词 | 通过或拦截状态 | 文本模型 | 否 |

## 典型创作场景口径

第一版不单独拆分“场景化创作能力矩阵”产品子文档。音乐创作、MV 制作、短视频制作、电商商品图、电商广告视频、文旅宣传视频、品牌 LOGO 等是统一 Agent 必须覆盖的典型业务场景示例。

这些场景通过统一 Agent、Skill、平台开放 Tool、模型选择、积分扣费和 A2UI 外部工作区组合完成，不为每个场景创建独立 Agent 或独立业务系统。

## 业务规则

- 系统只提供一个统一 Agent Runtime。
- Agent 根据用户意图和可用 Skill 池选择 Skill。
- 没有合适 Skill 时，Agent 使用文本模型继续对话，不提示用户创建 Skill。
- 没有合适 Skill 但用户意图匹配平台开放 Tool 时，Agent 可以按 Tool 边界调用 Tool。
- Agent 每次执行必须绑定当前空间。
- Agent 每次执行必须绑定当前项目；项目由业务微服务维护，Agent 只保存 `project_id` 引用。
- 当前空间由登录页面身份选择或个人中心身份切换确定。
- 个人登录对应个人空间，企业登录对应企业空间。
- 身份切换等价于退出当前登录身份并重新登录目标身份。
- Agent 只读取当前登录身份上下文，不允许沿用切换前身份或仅依赖前端传入的旧空间上下文。
- Agent 不直接写业务数据库。
- Agent 不保存积分账户、订单、支付、业务权限等业务事实。
- Agent 保存创作过程和资产引用，业务微服务保存用户可见资产事实。
- 会话、run、资产引用和黑板快照必须能追溯到项目。
- Skill 声明输出元素结构，Agent 按结构生成资产元素草稿，业务微服务保存最终资产元素。
- 生成完成且资产保存成功后，资产进入资产视图、产物历史和产物详情。
- 生成类任务必须在积分冻结和生成 Tool 执行前完成 LLM 提示词安全评估。
- 提示词安全评估不通过时，不冻结积分、不执行生成、不保存可用资产。
- 扣费、业务事实写入、高风险 Tool 必须进入人工确认。
- 文本模型用于意图识别、Skill 路由、对话、文案生成和脚本生成，普通用户不能选择文本模型。
- Agent 任务历史、产物历史、会话历史在用户侧本人可见。
- 企业空间可见范围按 [账户体系产品系统设计](./账户体系产品系统设计.md) 执行；第一版企业拥有者不额外查看成员资产、产物、会话、任务、黑板或生成记录。
- AG-UI / A2UI 组件样式、组件作用、事件类型、顺序、payload 和交互契约单独设计。
- 用户可以在会话中引用已创作资产继续创作；第一版不提供脱离会话的产物详情“继续创作”快捷入口。
- 用户可以从会话历史进入原会话，查看已保存对话、黑板和资产视图，并在会话中继续引用资产创作。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| 无可用 Skill | 用户意图无法匹配系统、企业或个人 Skill | 按普通对话继续回复 | 使用文本模型对话兜底；如意图匹配平台开放 Tool，则按 Tool 边界执行 |
| Skill 不可用 | Skill 处于 Draft、Deprecated 或被停用 | 该能力当前不可用 | 不参与路由 |
| Tool 不可用 | Skill 绑定 Tool 停用或无权限 | 当前能力暂不可用 | 中止执行并记录原因 |
| 用户取消确认 | 用户拒绝扣费或高风险确认 | 已取消 | 不执行 Tool，不扣费 |
| 提示词安全评估不通过 | 用户提示词或生成前提示词被 LLM 评估为不安全 | 提示词不符合平台规则，请修改后重试 | 不冻结积分，不执行生成 |
| 提示词安全评估失败 | LLM 安全评估调用失败或超时 | 暂时无法完成安全评估，请稍后重试 | 不冻结积分，不执行生成 |
| 生成失败 | 模型调用失败、超时或供应商错误 | 生成失败，可稍后重试 | 输出失败状态，释放冻结积分由业务服务处理 |
| 资产保存失败 | 生成完成但资产未保存成功 | 资产保存失败 | 不展示为可用资产，不确认该资产扣费 |

## 非目标

- 第一阶段不做多 Agent 体系。
- 第一阶段不让用户创建自己的 Agent。
- 第一阶段不让 Agent 直接维护业务事实。
- 第一阶段不接入外部内容安全服务、不做人审队列、不审核已生成文件内容。
- 第一阶段不在本文档中定义完整 AG-UI / A2UI 事件协议，后续由单独契约文档承接。
