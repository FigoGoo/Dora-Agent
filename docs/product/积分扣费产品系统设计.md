## 背景

生成图片、音乐、视频会产生平台成本，因此需要设计积分扣费能力。产品已确认：积分扣费应作为独立业务模块处理，Agent 和 Tool 只触发预估、冻结、确认扣减或释放流程。

## 目标

- 图片按张计费，并按图片生成模型配置积分单价。
- 音乐按首计费，并按音乐生成模型配置积分单价。
- 视频按秒计费，并按视频生成模型配置积分单价。
- 生成前展示预计消耗积分和当前可用积分。
- 用户确认后再执行生成。
- 用户确认后先冻结积分，再执行生成。
- 提示词安全评估通过后，才进入积分预估、确认和冻结。
- 已完成且资产保存成功的产物确认扣减积分，失败、取消、超时或资产保存失败的未完成部分释放冻结积分。

## 积分账户

积分属于业务域，由业务微服务负责账户、冻结、扣减、释放和审计。

账户类型：

- 个人积分账户。
- 企业积分账户。

生成所在空间决定扣费账户：

- 个人空间生成扣个人积分。
- 企业空间生成扣企业积分。
- 企业积分不足时，不自动切换到个人积分，系统提示企业积分不足。

积分来源、兑换码和有效期规则见 [积分来源与有效期产品系统设计](./积分来源与有效期产品系统设计.md)。

## 计费单位

| 资源类型 | 计费单位 | 说明 |
| --- | --- | --- |
| 图片 | 张 | 按生成图片数量计费 |
| 音乐 | 首 | 按生成歌曲或 BGM 首数计费 |
| 视频 | 秒 | 按生成视频秒数计费 |
| 文本 | 不对用户单独展示 | 作为平台内部成本 |
| 视觉理解 | 不对用户单独展示 | 作为平台内部成本 |
| 语音 | 后续设计 | TTS、ASR 暂不进入第一阶段 |

## 模型级积分单价

平台管理员在模型管理中为每个生成类模型配置用户积分单价。

| 模型类型 | 计费单位 | 单价配置 |
| --- | --- | --- |
| 图片生成模型 | 张 | 每个图片模型配置多少积分/张 |
| 音乐生成模型 | 首 | 每个音乐模型配置多少积分/首 |
| 视频生成模型 | 秒 | 每个视频模型配置多少积分/秒 |

不同模型可以配置不同积分单价。平台内部供应商成本和用户积分价格是两套概念，普通用户不看到模型成本，也不看到完整价格表。

用户侧只展示：

- 本次预计消耗积分。
- 当前可用积分。
- 即将过期积分。
- 积分不足提示。

用户侧不展示：

- 模型供应商成本。
- 内部成本计算公式。
- 平台毛利或成本价。
- 模型供应商信息。

## 用户流程

1. 用户在统一 Agent 对话框发起图片、音乐或视频生成。
2. 用户选择本次生成模型并填写生成参数。
3. Agent 对用户提示词和生成前提示词做 LLM 安全评估。
4. 安全评估通过后，Agent 携带模型和生成参数请求业务微服务预估积分。
5. 前端展示预计消耗积分和当前可用积分。
6. 用户确认执行。
7. 业务微服务冻结积分。
8. Agent 执行生成任务。
9. 生成完成且资产保存成功后，业务微服务按已保存可用产物确认扣减。
10. 失败、取消、超时或资产保存失败后，业务微服务释放未完成或不可用部分对应的冻结积分。

## 业务规则

- 积分账户、冻结、扣减、释放、审计由业务微服务负责。
- 积分来源、兑换码、有效期、过期和批次由业务微服务负责。
- Agent 不保存积分账户或扣费事实。
- 扣费类生成任务执行前必须展示预估积分并要求用户确认。
- 提示词安全评估必须在积分预估、确认和冻结前完成。
- 提示词安全评估不通过时，不进入积分预估、确认和冻结。
- 用户确认后必须先冻结积分，再执行生成 Tool。
- 个人空间扣个人积分。
- 企业空间扣企业积分。
- 企业积分不足时不自动扣个人积分。
- 图片按张、音乐按首、视频按秒预估积分。
- 积分预估必须使用本次选择模型对应的用户积分单价。
- 不同生成类模型可以有不同积分单价。
- 第一版不做质量、分辨率、视频时长上限等额外倍率。
- 生成完成且资产保存成功后，才确认该资产对应扣费。
- 生成完成但资产保存失败时，不展示为可用资产，不确认该资产对应扣费。
- 批量生成或生成中取消时，已保存成功的资产确认扣费，未完成或保存失败部分释放冻结积分。
- 企业积分管理权限由企业拥有者承担。
- 企业拥有者可以查看企业积分账户、企业积分流水、账单和成员消耗明细。
- 企业积分权限不扩展为企业资产、产物、会话、任务、黑板或生成记录可见权限。
- 企业普通成员可以查看自己的积分消耗明细。
- 文本和视觉理解成本不对用户单独展示。
- 语音 TTS / ASR 后续再设计。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| 积分不足 | 预估积分大于可用积分 | 积分不足，无法生成 | 不执行生成，不冻结积分 |
| 提示词安全评估不通过 | LLM 评估提示词不安全 | 提示词不符合平台规则，请修改后重试 | 不进入积分预估，不冻结积分 |
| 提示词安全评估失败 | LLM 安全评估调用失败或超时 | 暂时无法完成安全评估，请稍后重试 | 不进入积分预估，不冻结积分 |
| 用户取消确认 | 用户拒绝扣费确认 | 已取消 | 不执行 Tool，不扣费 |
| 冻结失败 | 业务服务无法冻结积分 | 暂时无法生成 | 不执行生成 |
| 生成失败 | 模型调用失败、超时或供应商错误 | 生成失败，可稍后重试 | 释放冻结积分 |
| 资产保存失败 | 生成完成但资产保存失败 | 资产保存失败 | 不展示为可用资产，释放该资产对应冻结积分 |
| 生成取消且无可用资产 | 用户取消长任务，且没有保存成功资产 | 已取消生成 | 释放冻结积分 |
| 生成取消但已有可用资产 | 用户取消长任务，但已有部分资产保存成功 | 已取消生成，已保存内容将扣费 | 已保存资产确认扣费，未完成或保存失败部分释放冻结积分 |
| 企业积分不足 | 企业空间生成但企业积分不足 | 企业积分不足 | 不自动扣个人积分 |

## 非目标

- 第一阶段不做在线购买积分包、企业在线充值、支付、订单和发票。
- 第一阶段不向用户展示模型供应商成本。
- 第一阶段不做质量、分辨率、视频时长上限等积分倍率。
- 第一阶段不设计文本和视觉理解的用户侧扣费。
- 第一阶段不设计语音模型扣费。
- 第一阶段不定义具体 RPC 字段，后续由契约文档承接。
