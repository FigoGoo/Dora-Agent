## 背景

AIGC 智能体 SaaS 同时支持个人用户和企业用户。账户体系需要区分登录用户、个人空间、企业空间、企业成员身份、积分账户和生成资源归属，避免把身份权限、扣费账户和产物归属混在一起。

## 目标

- 用户注册后默认是普通个人用户，并拥有个人空间。
- 系统初始化时默认存在一个初始平台管理员账号。
- 初始平台管理员可以新增平台管理员账号。
- 第一版平台管理员不做 RBAC，所有平台管理员拥有相同后台权限。
- 企业通过专用企业注册页面创建，创建者默认成为企业拥有者。
- 普通用户只能通过企业邀请加入企业，并自动成为企业用户。
- 第一版一个用户同一时间最多属于一个企业。
- 登录页面支持选择个人登录或企业登录。
- 每次使用 Agent 时必须有当前空间。
- 用户通过个人中心切换身份，本质是退出当前登录身份并重新登录目标身份，同时切换当前空间。
- 积分账户跟空间绑定，不直接跟 Agent 绑定。
- 生成产物、Skill、历史记录和扣费账户按当前空间归属。
- 第一版企业空间本质是多账户共享企业积分，不做复杂企业业务管理后台。
- 第一版不做独立企业管理员角色，企业管理权限默认由企业拥有者承担。
- 企业拥有者不获得成员资产、会话、任务、黑板或生成记录的额外可见性。

## 核心概念

```text
用户 User
  -> 默认拥有个人空间 Personal Space
  -> 可通过邀请加入一个企业 Enterprise
  -> 在企业里有一个成员身份 Membership
  -> 当前操作必须落在一个空间里：个人空间 / 企业空间

积分账户 Credit Account
  -> 个人空间有个人积分账户
  -> 企业空间有企业积分账户
```

## 用户角色

| 角色 | 权限/特征 | 核心诉求 |
| --- | --- | --- |
| 平台管理员 | 管平台能力，不参与普通用户业务空间 | 配置系统 Skill、模型、Tool、积分单价 |
| 个人用户 | 拥有个人空间和个人积分账户 | 创建个人 Skill，生成个人产物 |
| 企业拥有者 | 企业最高管理角色 | 管企业成员、企业积分、企业 Skill |
| 企业普通成员 | 企业使用角色 | 使用企业能力生成内容，消耗企业积分 |

说明：平台管理员和普通用户仍是产品层面的两类顶层用户。企业拥有者、企业普通成员是普通用户在企业空间中的成员角色。第一版不做独立企业管理员角色。

## 平台管理员账号规则

- 平台管理员后台使用独立入口和独立登录态。
- 系统初始化时默认存在一个初始平台管理员账号。
- 初始平台管理员可以新增其他平台管理员账号。
- 第一版所有平台管理员拥有相同后台权限，不做 RBAC 和细分后台角色。
- 平台管理员登录态不等同普通用户登录态，不能直接作为普通用户进入 Agent 创作工作台。
- 平台管理员账号操作需要进入平台审计日志。

第一版用户身份状态：

- 普通个人用户：未加入企业。
- 企业用户：已通过邀请加入企业。
- 企业用户被移除企业后，自动恢复为普通个人用户。

## 空间与资源归属

| 资源 | 个人空间 | 企业空间 |
| --- | --- | --- |
| Skill | 个人 Skill 属于用户本人 | 企业 Skill 属于企业 |
| 生成图片、音乐、视频 | 属于个人用户 | 属于企业空间 |
| 积分扣费 | 扣个人积分账户 | 扣企业积分账户 |
| 历史记录 | 个人可见 | 企业成员只查看自己的，企业拥有者不额外查看成员记录 |
| 用户被移出企业 | 个人资源不受影响 | 企业资源留在企业 |

## 当前空间规则

用户每次使用 Agent 时必须处于一个当前空间。

当前空间通过登录页面身份选择或个人中心身份切换确定。第一版身份切换等价于退出当前登录身份，并重新登录目标身份。切换成功后，当前登录用户信息、权限上下文、当前空间、积分账户、Skill 池和产物归属都按目标身份刷新。

登录入口：

- 登录页面支持个人登录和企业登录。
- 选择个人登录后，系统登录为个人身份，当前登录用户信息为个人用户上下文，当前空间为个人空间。
- 选择企业登录后，系统登录为企业成员身份，当前登录用户信息为企业成员上下文，包含企业、成员角色和企业权限，当前空间为企业空间。
- 如果用户没有企业成员身份，不能通过企业登录进入企业空间。

身份切换入口：

- 普通个人用户只有个人身份，对应个人空间。
- 企业用户在个人中心可切换个人身份和企业成员身份。
- 切换为个人身份后，系统重新登录为个人身份，当前登录用户信息变为个人用户上下文，当前空间为个人空间。
- 切换为企业成员身份后，系统重新登录为企业身份，当前登录用户信息变为企业成员上下文，包含企业、成员角色和企业权限，当前空间为企业空间。
- 同一浏览器会话第一版只保留一个当前登录身份。
- 用户进入统一 Agent 创作工作台时，必须使用当前登录身份对应的当前空间。

个人空间：

- Agent 加载系统 Skill 和个人 Skill。
- 生成扣个人积分账户。
- 生成产物归属个人空间。
- 历史记录归属个人空间。

企业空间：

- Agent 加载系统 Skill、企业 Skill 和用户个人 Skill。
- 生成扣企业积分账户。
- 生成产物归属企业空间。
- 历史记录归属企业空间。
- 企业空间普通创作能力与普通用户功能一致，核心差异是多人共享企业积分。
- 企业成员和企业拥有者只查看自己的会话、任务、产物、黑板和生成记录。
- 用户个人 Skill 可以在企业空间使用，但生成产物属于企业，扣企业积分。
- 如果个人 Skill 绑定的 Tool 不符合企业权限或平台开放规则，则该 Skill 在企业空间不可用。

## Skill 权限规则

- 平台管理员可以创建系统 Skill。
- 企业拥有者可以创建企业 Skill。
- 企业普通成员不能创建企业 Skill。
- 普通用户可以创建个人 Skill。
- 个人 Skill 属于创建者本人。
- 企业 Skill 属于企业空间，不属于某个成员个人。
- 用户被移出企业后，个人 Skill 不受影响，企业 Skill 留在企业。

## 企业管理规则

- 企业只能通过专用企业注册页面创建。
- 企业创建者默认为企业拥有者。
- 第一版不做独立企业管理员角色，企业管理权限由企业拥有者承担。
- 企业拥有者可以转让。
- 企业拥有者只能转让给当前企业成员。
- 企业拥有者转让后，原企业拥有者转为企业普通成员。
- 企业成员只能通过企业邀请加入。
- 企业成员不能主动退出企业，只能由企业拥有者移除。
- 企业拥有者可以移除非企业拥有者成员。
- 未注册用户通过邀请注册后，成为企业用户。
- 已注册普通个人用户接受企业邀请后，自动成为企业用户。
- 企业用户被移除企业后，自动恢复为普通个人用户。
- 企业管理入口第一版只做成员管理和企业积分管理，不做企业资产、产物、会话、任务或生成记录管理。
- 企业拥有者不额外查看成员资产、产物、会话、任务、黑板或生成记录。
- 企业普通成员只能查看自己的产物和自己的积分消耗明细。
- 第一版不支持用户同时加入多个企业。

## 积分账户规则

- 个人空间绑定个人积分账户。
- 企业空间绑定企业积分账户。
- 个人空间生成扣个人积分。
- 企业空间生成扣企业积分。
- 企业积分不足时，不自动切换到个人积分。
- 企业积分权限由企业拥有者承担。
- 企业拥有者可以查看企业积分账户、企业积分流水、账单、成员消耗明细，并可在企业空间兑换企业兑换码。
- 企业积分权限不扩展为企业资产、产物、会话、任务、黑板或生成记录可见权限。
- 积分账户、冻结、扣减、释放和审计由业务微服务负责。
- Agent 不保存积分账户或扣费事实。

## 用户流程

### 个人登录

1. 用户进入登录页面。
2. 用户选择个人登录。
3. 用户完成登录。
4. 系统建立个人身份登录态。
5. 当前登录用户信息为个人用户上下文。
6. 当前空间为个人空间。

### 企业登录

1. 用户进入登录页面。
2. 用户选择企业登录。
3. 用户完成登录。
4. 系统校验用户是否拥有企业成员身份。
5. 校验通过后，系统建立企业身份登录态。
6. 当前登录用户信息为企业成员上下文，包含企业、成员角色和企业权限。
7. 当前空间为企业空间。

### 企业注册

1. 用户进入专用企业注册页面。
2. 用户填写企业注册信息并完成注册。
3. 系统创建企业空间。
4. 创建者自动成为企业拥有者。
5. 企业空间拥有企业积分账户、企业 Skill 和企业产物归属。

### 企业邀请加入

1. 企业拥有者发起成员邀请。
2. 被邀请人通过邀请链接或邀请入口进入注册或加入流程。
3. 未注册用户完成注册后，自动成为企业用户。
4. 已注册普通个人用户接受邀请后，自动成为企业用户。
5. 企业成员获得对应企业角色和权限。

### 移除企业成员

1. 企业拥有者选择移除非企业拥有者成员。
2. 系统移除该成员的企业访问权限。
3. 被移除成员自动恢复为普通个人用户。
4. 被移除成员的个人空间、个人 Skill、个人积分和个人产物不受影响。
5. 企业 Skill、企业产物、企业历史记录和企业积分仍归企业。

### 转让企业拥有者

1. 企业拥有者选择转让企业拥有者身份。
2. 系统只允许选择当前企业成员作为接收人。
3. 接收人成为新的企业拥有者。
4. 原企业拥有者转为企业普通成员。

### 个人空间生成

1. 用户在个人中心切换为个人身份，当前空间为个人空间。
2. 用户打开统一 Agent。
3. Agent 加载系统 Skill 和个人 Skill。
4. 用户发起生成任务。
5. 系统按用户选择模型和生成参数预估个人积分。
6. 用户确认后执行生成。
7. 产物和历史记录归属个人空间。

### 企业空间生成

1. 用户在个人中心切换为企业成员身份，当前空间为企业空间。
2. 用户打开统一 Agent。
3. Agent 加载系统 Skill、企业 Skill 和用户个人 Skill。
4. 用户发起生成任务。
5. 系统按用户选择模型和生成参数预估企业积分。
6. 用户确认后执行生成。
7. 产物和历史记录归属企业空间。

### 用户被移出企业

1. 企业拥有者移除企业成员。
2. 用户个人空间、个人 Skill、个人产物和个人积分不受影响。
3. 企业 Skill、企业产物、企业历史记录和企业积分仍归企业。
4. 用户不再访问该企业空间资源，除非后续重新加入并获得权限。

## 业务规则

- 用户注册后默认拥有个人空间，并默认为普通个人用户。
- 系统初始化时默认存在一个初始平台管理员账号。
- 平台管理员可以新增其他平台管理员账号。
- 第一版平台管理员不做 RBAC，所有平台管理员拥有相同后台权限。
- 企业通过专用企业注册页面创建。
- 企业创建者默认为企业拥有者。
- 企业成员只能通过邀请加入。
- 普通个人用户被邀请并接受后，自动成为企业用户。
- 企业用户被移除企业后，自动恢复为普通个人用户。
- 企业拥有者可以转让，且只能转让给当前企业成员。
- 企业拥有者转让后，原企业拥有者转为企业普通成员。
- 第一版不做独立企业管理员角色。
- 企业成员不能主动退出企业。
- 企业拥有者可以移除非企业拥有者成员。
- 第一版一个用户同一时间最多属于一个企业。
- 登录页面支持个人登录和企业登录。
- 个人登录后，当前登录身份为个人身份，当前空间为个人空间。
- 企业登录后，当前登录身份为企业成员身份，当前空间为企业空间。
- 没有企业成员身份的用户不能企业登录。
- 每次 Agent 操作必须绑定当前空间。
- 当前空间通过登录页面身份选择或个人中心身份切换确定。
- 身份切换等价于退出当前登录身份并重新登录目标身份。
- 身份切换成功后，当前登录用户信息、权限上下文、当前空间、积分账户、Skill 池和产物归属都按目标身份刷新。
- Agent、业务 API、RPC 和 AG-UI 事件只能使用当前登录身份上下文，不允许继续沿用切换前身份。
- 当前空间决定 Skill 池、积分账户、产物归属和历史记录归属。
- 企业空间本质是多账户共享企业积分，普通创作功能与普通用户一致。
- 企业空间可以使用用户个人 Skill，但必须受企业权限和平台开放 Tool 约束。
- 企业拥有者可以创建企业 Skill。
- 企业普通成员不能创建企业 Skill。
- 企业拥有者不额外查看成员资产、产物、会话、任务、黑板或生成记录。
- 企业积分管理权限由企业拥有者承担。
- 企业普通成员只能查看自己的产物和自己的积分消耗明细。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| 未选择当前空间 | 用户无法确定操作空间 | 请先选择个人或企业空间 | 阻止 Agent 操作 |
| 企业登录无企业身份 | 用户选择企业登录但没有企业成员身份 | 暂无企业身份 | 拒绝企业登录，可使用个人登录 |
| 身份切换失败 | 个人中心切换身份时目标身份不可用 | 身份不可用 | 保持原登录身份和原当前空间，不进入目标身份 |
| 重复加入企业 | 已是企业用户时接受其他企业邀请 | 当前已加入企业 | 阻止加入 |
| 企业权限不足 | 企业普通成员创建企业 Skill | 无权创建企业 Skill | 拒绝操作 |
| 移除企业拥有者 | 企业成员尝试移除企业拥有者 | 不能移除企业拥有者 | 阻止移除 |
| 转让对象非法 | 企业拥有者转让给非企业成员 | 只能转让给企业成员 | 阻止转让 |
| 企业积分不足 | 企业空间生成但企业积分不足 | 企业积分不足 | 不自动扣个人积分 |
| 个人 Skill 在企业不可用 | 个人 Skill 绑定 Tool 不符合企业权限 | 该个人 Skill 在当前企业不可用 | 不参与路由或拒绝执行 |
| 用户主动退出企业 | 企业成员尝试主动退出企业 | 企业成员暂不支持主动退出 | 阻止退出，需企业拥有者移除 |
| 用户被移出企业 | 用户被企业拥有者移出企业 | 已离开企业 | 移除企业资源访问权限 |

## 非目标

- 第一阶段不设计复杂组织架构、部门、多级角色和岗位体系。
- 第一阶段不增加企业更多角色或权限项，不做独立企业管理员角色。
- 第一阶段不支持一个用户同时加入多个企业。
- 第一阶段不设计 SSO、企业域名认证或外部身份源。
- 第一阶段不设计企业资源跨企业转移。
- 第一阶段不设计企业积分套餐和购买流程。
- 第一阶段不做企业资产、产物、会话、任务、黑板或生成记录管理。
- 第一阶段不定义具体数据库字段和 RPC 契约，后续由契约文档承接。
