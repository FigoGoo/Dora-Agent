## 背景

产品设计确认 Done 后，Go Eino 智能体微服务架构工程师、业务微服务后端工程师、前端开发工程师和测试工程师需要基于同一套输入进入契约设计、数据建模、页面设计和本地开发。

本文件定义产品侧交付给工程的输入清单。当前为建议稿，未确认前不能作为正式开发依据。

## 目标

- 明确产品 Done 后工程团队需要拿到哪些输入。
- 按功能点拆分工程交付范围，便于后续按功能点提交。
- 明确 Agent、业务后端、前端和测试的输入边界。
- 明确哪些内容必须先进入 RPC、API、AG-UI、Agent 数据模型和 SQL 脚本契约。
- 避免工程 subagent 在产品未 Done 时自行猜测字段、状态和流程。

## 交付前置条件

- 所有关联产品子文档 `product_status` 必须为 Done。
- 页面范围与交互状态必须确认。
- AG-UI / A2UI 组件范围、事件建议和交互规则必须确认。
- 工程交付输入清单必须确认。
- 产品体验设计师输出项目开发计划、功能点、owner、里程碑和依赖，项目计划统一放在 `docs/project/`。
- 工程 subagent 开发前必须输出需求映射矩阵。

## 创作场景覆盖原则

第一版不单独创建“场景化创作能力矩阵”产品子文档。音乐创作、MV 制作、短视频制作、电商商品图、电商广告视频、文旅宣传视频、品牌 LOGO 等是典型业务场景示例，不作为独立产品文档拆分。

统一 Agent、Skill、Tool、模型选择、积分扣费和 A2UI 工作区需要共同覆盖这些典型创作场景：

- 音乐创作。
- MV 制作。
- 短视频制作。
- 电商商品图。
- 电商广告视频。
- 文旅宣传视频。
- 品牌 LOGO。

后续工程拆分时，可以按功能点和基础能力提交，不按上述场景单独建一套平行业务系统。

## Go Eino 智能体微服务输入

Go Eino 智能体微服务架构工程师需要从产品文档获得：

- 统一 Agent 只使用一个 Agent Runtime 的口径。
- 当前空间决定 Skill 池：系统 Skill、企业 Skill、个人 Skill。
- 没有匹配 Skill 时，使用文本模型继续对话；如意图明确匹配平台开放基础 Tool，可按 Tool 边界直接调用 Tool。
- Skill 路由、Tool 调用、人工确认、模型选择和积分预估的产品流程。
- LLM 提示词安全评估流程，以及安全评估必须在积分冻结和生成执行前完成的规则。
- 模型选择只对当前对话生效，进入扣费确认后锁定。
- AG-UI / A2UI 组件范围、事件建议、事件顺序和公共 payload。
- 音乐创作、MV 制作、短视频制作、电商商品图、电商广告视频、文旅宣传视频、品牌 LOGO 等典型场景需要由统一 Agent 通过 Skill 和 Tool 组合覆盖。
- Agent Runtime 可持久化的数据范围：会话、运行、消息、事件、Tool 调用、任务、产物引用、资产引用、黑板、分镜、脚本、提示词、记忆和运行配置。
- Agent 微服务保存创作过程和资产引用，不保存用户可见资产事实。
- Agent 微服务按 Skill 输出元素结构保存黑板中的草稿态资产元素。
- 生成完成后需要通过业务微服务保存用户可见资产。
- 生成完成后需要通过业务微服务保存最终资产元素，Agent 只保存资产和元素引用。
- 不得保存积分账户、订单、支付、企业权限等业务事实。

- Agent 领域数据模型文档。
- AG-UI 事件协议文档。
- Agent API 契约文档。
- 与业务微服务的 RPC 调用需求清单。

## 业务微服务后端输入

业务微服务后端工程师需要从产品文档获得：

- 用户、个人空间、企业空间、企业成员和企业拥有者规则。
- 登录页面个人登录和企业登录规则。
- 个人中心身份切换规则：退出当前登录身份，重新登录个人身份或企业成员身份，并改变当前登录用户信息和当前空间。
- 企业创建、邀请加入、移除成员和拥有者转让流程。
- 企业拥有者转让后，原拥有者转为企业普通成员。
- 企业成员不能主动退出企业。
- 企业拥有者可以移除非企业拥有者成员。
- 第一版不做独立企业管理员角色，企业管理权限由企业拥有者承担。
- 第一版不增加企业更多角色或权限项。
- 企业空间本质是多账户共享企业积分，普通创作功能与普通用户一致。
- 企业拥有者不额外查看成员资产、产物、会话、任务、黑板或生成记录。
- 企业积分管理权限由企业拥有者承担。
- 企业普通成员可以查看自己的积分消耗明细。
- 平台后台用户管理规则：查询用户、查看基础信息、启用或禁用用户账号、记录原因和审计。
- 用户管理不得展示用户私有资产、会话、黑板、生成记录、提示词、私有素材或创作内容。
- 平台管理员不得代登录普通用户账号。
- 模型供应商 API Key、模型管理、默认模型、内部成本和用户积分单价规则。
- 积分账户、批次、有效期、兑换码、冻结、扣减、释放和审计规则。
- 用户可见资产事实、资产归属、资产权限、预览、下载、产物历史和产物详情规则。
- 最终资产元素、平台内置固定资产元素类型、资产元素权限和资产元素详情展示规则。
- 上传素材格式、大小、失败处理和上传时文本安全评估规则。
- 资产文件存储在火山引擎 TOS，且有权限访问的资产都支持预览和下载。
- 个人作品中心、作品、公开快照、分享、取消分享、点赞、分类、标签和精选作品公开访问规则。
- 公开作品下架和审计规则。
- Tool 白名单、风险等级、可用范围、超时、重试和取消策略。
- 企业 Skill、个人 Skill 的审核业务规则；系统 Skill 不需要审核。
- Skill 输出元素结构的审核规则，确保只能使用平台内置固定资产元素类型。
- 平台管理员账号初始化、新增平台管理员账号和不做 RBAC 的后台权限规则。
- 平台审计日志覆盖范围和敏感信息不可记录规则。

后续工程文档：

- RPC 契约文档。
- API 契约文档。
- 业务数据库 SQL 迭代脚本。
- 权限矩阵和业务错误码。

## 前端开发输入

前端开发工程师需要从产品文档获得：

- 页面范围与入口关系。
- 登录页面个人登录和企业登录入口。
- 个人中心身份切换、重新登录目标身份和当前空间切换规则。
- 普通用户端和企业空间共用同一个统一 Agent 创作工作台，通过当前空间切换改变上下文。
- 企业空间本质是多账户共享企业积分，普通创作功能与普通用户一致。
- 企业管理入口第一版只支持企业成员管理和企业积分管理，不做企业资产、产物、会话、任务、黑板或生成记录管理。
- 平台管理员后台采用独立入口、独立代码，与普通用户端和企业空间代码边界隔离。
- Agent 创作工作台主流程。
- A2UI 聊天区域组件、聊天外部组件、状态和交互规则。
- A2UI 组件样式必须符合站点主题样式，视觉 token、组件变体和状态样式由前端设计系统落地。
- A2UI 思考状态打印机输出规则，只展示可公开处理状态，不暴露模型内部推理链路。
- A2UI 平台能力 Tag 展示规则，支持 Skill、Tool 等平台内置能力 Tag，且不暴露内部配置。
- 模型选择在聊天输入框内完成，不作为独立 A2UI 组件。
- 聊天输入控件支持模型选择、单选、多选、输入框和素材选择。
- 聊天外部只有一个外部工作区组件，用于展示资产视图和黑板视图。
- 资产视图和黑板视图按平台内置固定资产元素类型渲染不同 Skill 的输出元素。
- 提示词安全评估不通过时展示 blocked 状态，并阻止进入积分冻结和生成执行。
- 资产保存中、资产保存成功、资产保存失败、创作过程保存和会话恢复状态。
- 积分预估、人工确认、生成进度、资产视图和黑板视图流程。
- Skill Builder 创建、测试、审核、版本和回滚页面状态。
- Skill 审核通过和审核拒绝的站内信通知。
- 个人积分、企业积分、兑换码、消耗明细页面状态。
- 平台后台用户管理列表、用户详情、启用/禁用确认和权限不足状态。
- 个人作品中心页面、作品创建/编辑/分享/取消分享交互。
- 精选作品中心和详情页免登录浏览、分类筛选、标签筛选、复制分享链接、登录后点赞交互。
- 平台管理员后台页面范围。
- 权限不足、积分不足、模型不可用、Tool 不可用、审核拒绝、断线重连等异常状态。

## 契约产出顺序建议

1. 产品体验设计师确认全部产品文档 Done。
2. Go Eino、业务后端、前端分别输出需求映射矩阵。
3. 主控 Codex 汇总跨边界冲突。
4. 先产出 RPC 契约，明确智能体微服务和业务微服务边界。
5. 再产出 AG-UI 事件协议，明确智能体微服务和前端实时事件边界。
6. 再产出 API 契约，明确前端查询、提交、确认、取消和历史接口。
7. 再产出 Agent 领域数据模型和业务 SQL 迭代脚本。
8. 最后按功能点进入本地开发、联调和测试。
