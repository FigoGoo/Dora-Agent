## 背景

AIGC 创作需要在生成前避免明显不安全的提示词进入模型生成链路。第一版内容安全治理不引入外部内容安全服务，不做人审队列，不审核已生成图片、音乐、视频文件内容，只使用平台文本模型对提示词进行安全评估。

## 目标

- 使用 LLM 对用户提示词和生成前提示词评估是否安全。
- 在扣费冻结和生成 Tool 执行前完成提示词安全评估。
- 对不安全提示词进行阻止，并提示用户修改。
- 避免把内容安全治理做成复杂审核系统。
- 不保存模型内部推理链路、安全策略细节或供应商原始响应。

## 评估范围

第一版评估以下文本：

- 用户输入的创作提示词。
- 聊天输入框控件中的文本参数。
- Skill 组装后的生成前提示词。
- 生成图片、音乐、视频前传给模型的最终提示词摘要。
- 上传素材时用户填写的文本说明、标题或标签。

第一版不评估以下内容：

- 已生成图片、音乐、视频文件本身。
- 上传图片、音频、视频文件本身。
- 模型内部推理链路。
- 系统提示词和平台安全策略细节。
- 供应商返回的内部原始响应。

## 用户流程

1. 用户在统一 Agent 创作工作台输入创作意图。
2. Agent 识别意图并选择 Skill。
3. Agent 根据 Skill 和用户输入组装生成前提示词。
4. 系统使用平台文本模型对提示词进行安全评估。
5. 如果评估通过，继续进入模型选择、积分预估和生成确认。
6. 如果评估不通过，阻止进入扣费冻结和生成 Tool 执行。
7. 前端展示可理解的提示，引导用户修改提示词。

## 业务规则

- 第一版内容安全治理只使用 LLM 做提示词安全评估。
- 普通用户不能选择或配置内容安全评估模型。
- 提示词安全评估使用平台默认文本模型或平台指定文本模型。
- 提示词安全评估必须发生在积分冻结和生成 Tool 执行前。
- 提示词评估不通过时，不冻结积分，不执行生成，不保存可用资产。
- 提示词评估不通过时，可以保存会话中的失败状态和用户可见提示。
- 内容安全评估结果只保存必要状态、时间、评估对象摘要和用户可见原因。
- 不保存 LLM 内部推理链路、安全策略细节、系统提示词或供应商原始响应。
- Skill 不能关闭或绕过提示词安全评估。
- 平台管理员第一版不配置复杂内容安全规则，后续如需扩展再单独设计。

## 异常场景

| 场景 | 触发条件 | 用户提示 | 系统行为 |
| --- | --- | --- | --- |
| 提示词不安全 | LLM 评估为不安全 | 提示词不符合平台规则，请修改后重试 | 阻止生成，不冻结积分 |
| 评估失败 | LLM 安全评估调用失败或超时 | 暂时无法完成安全评估，请稍后重试 | 阻止生成，不冻结积分 |
| 提示词为空 | 用户未提供必要文本 | 请补充创作要求 | 不进入安全评估和扣费确认 |
| 生成前提示词缺失 | Skill 未能组装有效提示词 | 当前 Skill 输出不完整 | 阻止生成，记录失败状态 |

## 非目标

- 第一阶段不接入外部内容安全服务。
- 第一阶段不做人审队列或人工复核。
- 第一阶段不审核已生成图片、音乐、视频文件内容。
- 第一阶段不对上传图片、音频、视频文件做多模态安全审核。
- 第一阶段不向普通用户展示具体安全策略、规则命中项或内部评估细节。
- 第一阶段不把内容安全做成普通用户或企业用户可配置能力。

## 验收标准

- [ ] 用户提示词在生成前会经过 LLM 安全评估。
- [ ] Skill 组装后的生成前提示词会经过 LLM 安全评估。
- [ ] 提示词评估不通过时，不进入积分冻结。
- [ ] 提示词评估不通过时，不执行图片、音乐或视频生成。
- [ ] 提示词评估失败或超时时，不执行生成。
- [ ] 用户能看到可理解的修改提示。
- [ ] 内容安全评估不暴露模型内部推理链路、系统提示词或安全策略细节。
- [ ] Skill 不能关闭或绕过提示词安全评估。
