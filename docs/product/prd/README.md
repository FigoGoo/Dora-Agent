## 背景

本目录用于承接已经确认过的产品系统设计讨论，并将其转化为后续页面设计、契约设计、后端开发、Agent 开发和测试验收可直接引用的 PRD。

当前 PRD 套件已由用户确认进入 `product_status：Done`，工程责任域可以按开发流程先输出需求映射矩阵，再进入契约设计和代码开发。

## 使用方式

1. 产品与需求责任域按模块与用户继续确认 PRD。
2. 确认完成后，将对应 PRD 的 `product_status` 从 `Draft` 更新为 `Done`。
3. 工程责任域进入开发前，先基于 Done PRD 输出需求映射矩阵。
4. 涉及跨服务边界时，先补 RPC、API、AG-UI、Agent 数据模型和 SQL 脚本文档。
5. 正式开发按功能点提交，不按大而全的系统批量提交。

## 当前约束

- 第一版只有一个统一 Agent，不做多 Agent 体系。
- 第一版平台管理员后台独立入口、独立代码。
- 第一版企业空间本质是多账户共享企业积分，不做企业资产和会话管理后台。
- 第一版内容安全治理只使用 LLM 做提示词安全评估。
- 第一版不做在线购买积分包、支付、订单、发票。
- 第一版不做可观测性、告警、SLO 和线上运行手册。
- 第一版前端设计系统在产品功能确认后再落地。
