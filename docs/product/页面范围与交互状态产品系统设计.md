## 背景

当前产品规则已覆盖 Agent、Skill、模型、Tool、积分和账户。为了让前端开发工程师后续能够按产品文档落地，需要先明确第一版页面范围、入口关系和关键交互状态。

已确认：第一版企业空间本质是多账户共享企业积分，不做企业资产、产物、会话、任务、黑板或生成记录管理。

本文件是产品侧页面范围建议稿，待确认后再进入前端设计系统和页面实现。

已确认：普通用户端和企业空间共用同一个统一 Agent 创作工作台，通过当前空间切换改变上下文；平台管理员后台是独立入口、独立代码。

## 目标

- 定义第一版需要哪些页面。
- 定义每个页面服务的角色和主要操作。
- 定义通用交互状态，避免前端自行发明状态。
- 明确哪些页面承接 Agent 创作、Skill Builder、模型、Tool、积分和企业管理。
- 为后续前端设计系统、API 契约和 AG-UI 契约提供输入。

## 平台管理员后台页面

平台管理员后台为独立入口、独立代码，不与普通用户端或企业空间共用同一套前端入口。

第一版平台后台只有一个平台管理员角色，不做 RBAC；平台管理员能力按平台管理员账号、用户管理、系统 Skill、Skill 审核、模型供应商、模型单价、Tool 白名单、积分发放、兑换码、精选作品管理和审计日志模块组织。

| 页面 | 主要操作 | 关键状态 |
| --- | --- | --- |
| 平台后台首页 | 查看平台能力配置入口 | loading、error |
| 平台管理员账号 | 基于初始管理员新增、停用平台管理员账号 | loading、enabled、disabled、confirming |
| 用户管理 | 查询用户、查看用户基础信息、启用或禁用用户账号 | loading、enabled、disabled、confirming、permission_denied |
| 系统 Skill 管理 | 创建、编辑、测试、直接发布、废弃系统 Skill | draft、testing、published、deprecated |
| Skill 审核队列 | 审核企业 Skill 和个人 Skill，审核意见非必填 | pending_review、approved、rejected、empty |
| 模型供应商管理 | 配置供应商 API Key，执行连通性测试 | untested、testing、connected、failed |
| 模型管理 | 创建模型、启停模型、配置默认模型、内部成本、用户积分单价 | enabled、disabled、default、price_missing |
| Tool 管理 | 启停平台内置 Tool、配置白名单、风险等级、超时、可用范围 | enabled、disabled、risk_low、risk_medium、risk_high |
| 积分发放 | 手动给个人或企业发放积分，设置有效期 | draft、confirming、success、failed |
| 兑换码管理 | 创建兑换码、批量导出兑换码、绑定用户/企业/渠道、查看兑换状态 | active、redeemed、expired、disabled |
| 精选作品管理 | 查询公开作品、下架风险公开作品 | shared、taken_down、confirming |
| 平台审计日志 | 查看管理员操作日志 | loading、empty、filtered_empty |

平台管理员账号规则：

- 平台后台为独立入口、独立代码、独立登录态。
- 系统初始化时默认存在一个初始平台管理员账号。
- 初始平台管理员可以新增其他平台管理员账号。
- 第一版所有平台管理员拥有相同后台权限，不做 RBAC 和细分后台角色。
- 平台管理员登录态不等同普通用户登录态，不能直接作为普通用户进入 Agent 创作工作台。

平台审计日志第一版覆盖：

- 平台管理员登录、退出、登录失败。
- 新增、停用平台管理员账号。
- 用户账号启用、禁用、状态变更原因。
- 系统 Skill 创建、编辑、测试、发布、废弃。
- 企业 Skill 和个人 Skill 审核通过、审核拒绝。
- 模型供应商 API Key 新增、编辑、连通性测试、停用。
- 模型新增、编辑、启用、停用、默认模型切换、内部成本修改、用户积分单价修改。
- Tool 启用、停用、白名单修改、风险等级修改、超时修改、可用范围修改。
- 积分手动发放。
- 兑换码创建、停用、批量导出。
- 精选作品下架。
- 平台内置资产元素类型配置发布或变更。

审计日志不得记录 API Key 明文、完整兑换码列表、用户隐私内容、模型内部推理链路或平台安全策略细节。

## 通用交互状态

| 状态 | 触发场景 | 展示要求 |
| --- | --- | --- |
| loading | 页面或列表加载中 | 展示骨架或加载状态，避免空白 |
| empty | 数据为空 | 展示明确空状态和可执行入口 |
| filtered_empty | 筛选后无结果 | 支持清除筛选 |
| error | 普通错误 | 展示可理解错误和重试入口 |
| permission_denied | 权限不足 | 说明无权限，不展示越权操作 |
| streaming | Agent 文本输出中 | 支持停止生成 |
| thinking | Agent 正在处理用户意图 | 使用思考状态打印机输出可公开处理状态，不展示内部推理链路 |
| model_selecting | 需要选择生成模型 | 在聊天输入框内按图片、音乐、视频分组展示模型名称 |
| confirming | 扣费、高风险 Tool 或业务写入待确认 | 展示影响范围、预计积分和确认动作 |
| generating | 生成任务执行中 | 展示进度、当前状态和取消入口 |
| partial_completed | 批量或长任务部分完成 | 展示已完成产物和未完成状态 |
| cancelled | 用户取消任务 | 说明已完成部分是否扣费，未完成部分释放冻结积分 |
| timeout | Tool 或模型超时 | 展示超时原因和可重试建议 |
| insufficient | 积分不足 | 阻止生成，展示可用积分和所需积分 |
| unavailable | 模型、Tool 或 Skill 不可用 | 引导重新选择或稍后再试 |
| pending_review | Skill 等待审核 | 展示当前审核状态 |
| rejected | Skill 审核未通过 | 展示审核结果，审核意见可能为空 |
| blocked | 提示词安全评估不通过 | 阻止生成，引导用户修改提示词 |
| reconnecting | AG-UI 流断线重连中 | 保留当前内容并尝试补偿事件 |
| saving | 资产或创作过程保存中 | 展示保存中状态，避免用户误以为已完成 |
| save_failed | 资产或创作过程保存失败 | 展示可理解错误和重试入口 |

## 关键交互规则

- 用户端以 Agent 创作工作台作为第一版核心入口。
- 登录页面支持选择个人登录或企业登录。
- 个人登录成功后，当前登录用户信息为个人用户上下文，当前空间为个人空间。
- 企业登录成功后，当前登录用户信息为企业成员上下文，当前空间为企业空间。
- 没有企业成员身份的用户不能企业登录。
- 同一用户每次操作必须处于个人空间或企业空间。
- 用户通过个人中心切换身份来切换当前空间。
- 身份切换等价于退出当前登录身份并重新登录目标身份。
- 身份切换成功后，前端必须刷新当前登录用户信息、权限上下文、当前空间、积分账户和可用能力。
- 普通用户端和企业空间共用同一个统一 Agent 创作工作台，通过当前空间切换改变 Skill 池、积分账户、产物归属和历史可见范围。
- 个人作品中心展示当前登录用户自己的作品集；个人身份下展示个人空间作品，企业身份下展示当前用户本人企业空间作品。
- 用户可以从已保存资产创建个人作品，编辑作品标题、封面、简介、分类和标签。
- 用户分享作品后，作品进入精选作品中心。
- 用户取消分享后，作品不再出现在精选作品中心，公开链接不可访问。
- 精选作品中心和精选作品详情支持免登录查看。
- 精选作品点赞需要登录；未登录点击点赞时引导登录。
- 精选作品支持复制分享链接、分类筛选、标签筛选和资源类型筛选。
- 公开作品展示不得暴露源会话、黑板、提示词、积分、模型成本、内部用户 ID、手机号、邮箱或私有素材。
- 平台管理员后台采用独立入口、独立代码，与普通用户端和企业空间代码边界隔离。
- 平台管理员后台第一版只有一个平台管理员角色，不做 RBAC 和多后台角色拆分。
- 平台管理员后台默认初始化一个初始平台管理员账号，可新增其他平台管理员账号。
- 平台管理员后台按平台管理员账号、用户管理、系统 Skill、Skill 审核、模型供应商、模型单价、Tool 白名单、积分发放、兑换码、精选作品管理和审计日志组织权限范围。
- 用户管理支持查询用户、查看用户基础信息、启用或禁用用户账号。
- 用户管理不展示用户私有资产、会话、黑板、生成记录、提示词、上传私有素材或创作内容。
- 平台管理员不能通过用户管理代登录普通用户账号。
- 企业空间本质是多账户共享企业积分，普通创作功能与普通用户一致。
- 企业管理入口第一版只支持企业成员管理和企业积分管理，不做企业资产、产物、会话、任务、黑板或生成记录管理。
- 第一版不做独立企业管理员角色，企业管理权限由企业拥有者承担。
- 企业拥有者不额外查看成员资产、产物、会话、任务、黑板或生成记录。
- 企业拥有者可以移除非企业拥有者成员。
- 第一版不增加企业更多角色或权限项。
- 企业空间中的个人 Skill 可用于企业创作，但受企业权限、套餐和 Tool 边界限制。
- 模型选择只对当前对话生效，在聊天输入框内完成，进入扣费确认后锁定。
- A2UI 组件分为聊天区域组件和聊天外部组件。
- A2UI 组件样式必须符合站点主题样式，不单独创建独立视觉风格。
- 聊天区域支持消息流、思考状态打印机、平台能力 Tag、积分预估、人工确认、Tool 状态、生成进度、错误提示和聊天输入控件。
- 思考状态打印机输出只展示可公开处理状态和进度摘要，不展示模型内部推理链路。
- 平台能力 Tag 用于展示 Skill、Tool 等平台内置能力，只展示可公开名称、类型、状态和风险提示。
- 平台能力 Tag 不展示内部配置、供应商、密钥、endpoint、内部成本或平台内部标识。
- 提示词安全评估不通过时，Agent 创作工作台展示 blocked 状态，不进入积分冻结和生成执行。
- 聊天输入控件支持模型选择、单选、多选、输入框和素材选择。
- 聊天外部只有一个外部工作区组件，用于展示资产视图和黑板视图。
- 资产视图展示音乐、图片、视频等素材。
- 黑板视图展示 Skill 产出的资产元素草稿、素材、提示词、脚本和故事线；视频类 Skill 按分镜展示图片、脚本、提示词和生成状态。
- 资产视图和黑板视图按平台内置固定资产元素类型渲染，不按具体创作场景硬编码字段。
- 第一版产物详情不提供独立“继续创作”快捷动作；用户只能在统一 Agent 会话中引用已创作资产继续创作。
- 生成完成且资产保存成功后，资产进入资产视图、产物历史和产物详情。
- 会话历史支持进入原会话，查看已保存对话、黑板和资产视图。
- 创作过程保存包括会话、消息、Agent run、Tool 调用、生成任务、黑板、分镜、脚本、提示词和资产引用。
- 平台管理员创建系统 Skill 不需要审核；企业 Skill 和个人 Skill 需要平台管理员审核。
- Skill 审核通过或审核拒绝后，通过站内信通知创建者。
- 企业成员不能主动退出企业。
- 企业拥有者转让后，原拥有者转为企业普通成员。
- 企业普通成员可以查看自己的企业空间积分消耗明细。
- 前端页面字段和状态必须来自产品文档、API 契约和 AG-UI 契约，不允许自行发明后端字段。

## 非目标

- 第一阶段不定义最终视觉稿。
- 第一阶段不定义前端技术选型和组件实现。
- 第一阶段不做复杂组织架构、部门、岗位和多企业切换。
- 第一阶段不做在线购买积分包、支付、订单和发票页面。
- 第一阶段不做移动端独立交互规范，先保证响应式可用。
- 第一阶段不把平台管理员后台混入普通用户端或企业空间入口。

## 验收标准

- [ ] 第一版普通用户端页面范围已确认。
- [ ] 第一版企业空间页面范围已确认。
- [ ] 登录页面支持选择个人登录或企业登录。
- [ ] 企业登录时没有企业成员身份会被拒绝。
