## 测试边界

本文件只验证用户端触发的服务端能力，包括公开读、登录态私有读写、企业 owner 能力、企业 member 能力、用户 Skill、积分、资产、作品和通知。页面跳转、弹窗、组件展示和浏览器状态不在本文件测试范围内；当设计文档提到页面状态时，本文件只验证 API 错误码、响应 DTO 和服务端副作用。

## 测试用例

### 账户、身份和空间

| ID | 功能点 | 测试入口 | 前置数据 | 服务端断言 | 证据 |
| --- | --- | --- | --- | --- | --- |
| U-ACCOUNT-001 | 个人注册创建默认空间 | `POST /api/auth/register` | 新手机号或邮箱，带 `Idempotency-Key` | 同事务创建 `business_users`、personal `business_spaces`、personal `credit_accounts`；响应含 `AuthSessionDTO`、当前 personal identity；重复同 key 同 hash 返回同一用户和空间 | HTTP contract、业务 DB、幂等记录 |
| U-ACCOUNT-002 | 注册幂等冲突 | `POST /api/auth/register` | 同一 `Idempotency-Key`，修改邮箱或手机号 | 返回 `409 IDEMPOTENCY_CONFLICT`；不创建第二个用户、空间或积分账户 | HTTP contract、业务 DB |
| U-ACCOUNT-003 | 个人登录成功 | `POST /api/auth/login` | active 用户和 personal space | 创建 active `auth_sessions`，token 只存 hash；响应含 current_space、available_identities；不返回 password hash、手机号/邮箱明文 | HTTP contract、业务 DB、脱敏断言 |
| U-ACCOUNT-004 | 禁用用户登录拒绝 | `POST /api/auth/login` | `business_users.status=disabled` | 返回 `403 PERMISSION_DENIED` 或稳定禁用错误；不创建 active session | HTTP contract、业务 DB |
| U-ACCOUNT-005 | 企业身份登录 | `POST /api/auth/login` | active enterprise member | session 的 `login_identity_type=enterprise_member`，`space_id` 为企业空间，`enterprise_role` 正确 | HTTP contract、业务 DB |
| U-ACCOUNT-006 | 被移除企业成员登录拒绝 | `POST /api/auth/login` | `enterprise_members.status=removed` | 返回 `PERMISSION_DENIED`；不返回企业空间 token | HTTP contract、业务 DB |
| U-ACCOUNT-007 | 当前空间解析 | `GET /api/account/current-space`、RPC `ResolveCurrentSpaceContext` | personal 与 enterprise session 各一套 | 响应 `space_type`、`credit_account_scope`、`skill_scope_keys`、`permission_summary` 与当前身份一致 | HTTP/RPC contract、业务 DB |
| U-ACCOUNT-008 | 错空间拒绝 | RPC `ResolveCurrentSpaceContext(expected_space_id)` | token 属于 A 空间，请求 expected B 空间 | 返回 `CROSS_SPACE_DENIED`，不信任客户端空间头 | RPC contract、日志 |
| U-ACCOUNT-009 | 身份切换成功 | `POST /api/account/switch-identity` | 用户有 personal 和 enterprise active identity | 写入新 session 或更新身份上下文；响应 current identity 变更；写身份切换审计 | HTTP contract、业务 DB、审计 |
| U-ACCOUNT-010 | 身份切换到无权限企业 | `POST /api/account/switch-identity` | 用户已被移出企业 | 返回 `PERMISSION_DENIED`；原 session 不被破坏 | HTTP contract、业务 DB |
| U-ACCOUNT-011 | 登出 | `POST /api/auth/logout` | active session | session 标记 `revoked` 或失效；后续私有 API 返回 `UNAUTHENTICATED` | HTTP contract、业务 DB |

## 风险和待确认

- 公开分类、标签和公开 Skill 摘要的最终枚举需以 OpenAPI/字典契约落地后补充字段级 contract fixture。
- 项目恢复是否作为第一版正式能力需实现阶段确认；若未实现，用例 U-PROJ-010 应改为稳定不支持错误。
- 企业成员是否可查看企业积分摘要的具体字段需以最终权限矩阵为准，但不得扩展到成员私有资产、会话和黑板。

## 验收标准

- 用户端所有 P0 功能都有 HTTP contract 用例。
- 每个写接口覆盖成功、缺幂等键、幂等重放、幂等冲突和权限拒绝。
- 公开接口全部覆盖匿名可读范围和私有字段过滤。
- 企业空间覆盖 owner/member/removed 三类权限。
- 用户端测试不包含页面视觉、组件渲染或浏览器点击步骤。
