## 契约计划

- RPC / Thrift：业务微服务拥有业务语义、权限、事务、错误码和幂等键；智能体微服务提出调用需求，主控汇总到 `docs/contracts/`。
- HTTP API：前端只消费契约字段；API 适配 Agent run、项目、资产、作品、积分、后台和企业空间。
- AG-UI：智能体微服务生产事件，前端消费事件；新增事件先更新 AG-UI 协议。
- Agent 数据模型：只保存 Agent Runtime 数据，不复制业务主数据；业务事实通过 RPC 产生。
- SQL：业务库和 Agent 领域库分目录准备迭代脚本，不添加数据库级外键约束。

## 风险与阻塞

| 风险 | owner | 处理方式 | 状态 |
| --- | --- | --- | --- |
| PRD 扩展事件与 AG-UI 标准事件命名存在差异 | 主控 Codex、Go Eino、前端 | M1 映射矩阵中列出差异，M2 统一协议命名 | open |
| RPC 能力可能被误设计为数据库表操作 | 主控 Codex、业务后端 | 契约按业务能力建模，写操作必须有幂等键和审计 | open |
| Agent DB 与业务 DB 边界容易混淆 | 主控 Codex、Go Eino、业务后端 | Agent 只保存 Runtime，业务事实只走业务微服务 RPC | open |
| 前端 PNG 草图含视觉示意文本，不能直接当字段契约 | 前端开发工程师 | 页面实现以 API/AG-UI 契约字段为准 | open |
| 内容安全证据字段需要实现一致性 | 产品体验设计师、Go Eino、业务后端、测试 | 契约已定 M2 基线，开发和测试按 `safety_evidence` 摘要对象验证 | open |
| TOS 公共桶下架后的缓存风险 | 产品体验设计师、业务后端、前端、测试 | 第一版不做 CDN 缓存失效；下架后业务列表和详情不可访问，重新公开必须换 object key，已缓存 URL 不承诺即时失效 | open |
