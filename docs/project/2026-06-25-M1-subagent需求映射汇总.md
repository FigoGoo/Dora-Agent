## 任务结论

四个 subagent 已完成 M1 需求映射：

- Go Eino 智能体微服务架构工程师：完成智能体侧需求映射、Eino 能力选型复核、RPC/AG-UI/Agent DB 需求。
- 业务微服务后端工程师：完成业务侧需求映射、RPC server 能力清单、业务模型、SQL、权限、错误码、幂等和审计建议。
- 前端开发工程师：完成页面、路由、组件、状态、LoginModal、API 依赖和 AG-UI 消费矩阵。
- 浏览器、RPC 与数据库测试工程师：完成浏览器、RPC contract、AG-UI、Agent DB 和业务 DB 分边界测试计划。

M1 结论：可以进入 M2 契约先行阶段。暂不直接进入跨服务代码实现；先补齐 RPC、API、AG-UI、Agent 领域数据模型和 SQL 计划。

## 关键需求映射

| 产品条目 | Eino 侧 | 业务后端侧 | 前端侧 | 测试侧 |
| --- | --- | --- | --- | --- |
| 账户与空间 | run 携带 user/space/project 上下文 | 空间解析、权限、企业身份 | Auth/Space 状态和身份切换 | 权限 E2E、跨空间拒绝 |
| 项目容器 | session/run/artifact 保存 `project_id` 引用 | ProjectService 和项目权限 | 项目列表、详情、进入工作台 | 项目归属、项目资产绑定 |
| Skill 路由 | Published 且当前空间可用 Skill Retriever | SkillCatalog / SkillReview | Skill Builder、Tag 展示 | Skill 生命周期、路由池过滤 |
| Tool 边界 | Tool Runtime 和风险确认 | ToolCapability 策略 | ToolStatus、确认卡 | 白名单、风险、超时、取消 |
| 模型选择 | 当前对话生效，确认后锁定快照 | 模型目录、默认模型、积分单价 | Composer 内 ModelPicker | 模型锁定、不可用状态 |
| 内容安全 | 固定前置 Workflow，阻断不进入积分 | 可校验安全证据 | blocked 状态 | 不安全不冻结、不生成 |
| 积分闭环 | 调用预估/冻结/释放/扣费 RPC | CreditService 保存事实 | CreditEstimate、ConfirmPanel | 幂等、并发冻结、释放 |
| 资产保存 | 保存过程引用，最终保存走 RPC | AssetService 保存资产事实 | AssetCard、MediaAssetsCard | Agent DB / 业务 DB 分边界 |
| AG-UI | 生产 SSE 事件、补偿和快照 | 不生产 AG-UI | reducer、去重、断线恢复 | 顺序、payload、未知事件 |
| 作品公开 | 可复用安全评估能力 | 作品、公开快照、点赞 | 作品中心、公开精选 | 匿名访问、隐私字段 |
| 平台后台 | 不负责 | 独立管理员、审计、审核 | AdminShell、DataTable | 后台脱敏、审计 |

## M2 裁决与剩余待细化

| 问题 | 影响 | 建议裁决 |
| --- | --- | --- |
| AG-UI 使用 `type` 还是 `event_type` | 前端消费和测试夹具不一致 | M2 统一为 `type`，兼容接收 `event_type` 一版 |
| `agent.started` 还是 `agent.run.started` | 事件命名不一致 | M2 统一为 `agent.run.*`，保留标准基础事件映射表 |
| 内容安全 owner | 安全配置、证据字段和阻断链路可能分散 | Agent 执行安全评估，业务写入要求 `safety_evidence.result=passed` |
| confirmation 后用户修改参数 | run/interrupt 状态机影响 | 修改关键参数时取消当前 interrupt，重新预估并创建新确认 |
| AG-UI 产品口径 | 仅工程事件不足以指导用户可见反馈 | 产品体验设计师已确认用户可见组件和敏感字段禁显规则 |
| 内容安全证据字段 | Agent 产出、业务校验和前端展示边界需统一 | M2 统一为 `safety_evidence` 摘要对象；禁止外泄策略细节、完整 Prompt 和供应商原始响应 |
| 公开媒体访问方式 | TOS 公共桶、上传签名、公开快照和 CDN 访问策略需统一 | M2 统一为后端签发上传凭证、前端直传 TOS、公开快照引用 TOS 公共 URL；CDN 域名为 `https://tos.doraigc.com`，不做缓存失效 |
| 项目归档后继续创作 | run、资产保存和扣费可能写入只读项目 | M2 明确禁止继续创作；只允许查看历史、资产、黑板和作品 |

## 第一批协作开发切片

1. 契约骨架：创建 `docs/contracts/**`，确定 RPC / API / AG-UI / Agent DB 的 owner 和状态。
2. Project 主路径：Auth/Space + Project + Agent session/run 上下文。
3. 工作台事件主路径：SSE mock、AG-UI reducer、event_id 去重、sequence 合并、snapshot fallback。
4. 最小生成闭环：安全通过、积分预估、确认、冻结、Tool mock、资产保存并扣费。
5. 双库边界：Agent 保存过程引用，业务保存资产事实和积分事实。
6. 公开访问闭环：首页/精选作品免登录，点赞和创作触发 LoginModal。
7. 后台审计闭环：用户禁用、模型/Tool/Skill/积分高风险操作补审计和脱敏。

## 边界检查

- Go Eino 智能体微服务不直接访问业务数据库。
- 业务微服务不承担 Eino 编排、AG-UI 生产和 Agent Runtime 状态。
- 前端不发明 API、RPC 或 AG-UI 字段，不保存业务事实。
- 测试必须分别验证 Agent DB 和业务 DB，不混用事实源。
