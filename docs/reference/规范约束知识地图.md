## 二、三条接缝咬合（Agent 平台骨架，迁移必须成套搬，拆散即断）

同一个「人工确认」语义在三层各有落点，靠同一套 ID 串起：

```
          RPC 契约              AG-UI 事件             TurnLoop 状态机
人工确认   preview/confirm  ↔   interrupt.required  ↔   interrupt / resume(四要素)
幂等去重   idempotency_key  ↔   event_id 去重        ↔   resume 校验幂等键
公共标识   权限/审计上下文    ↔   session_id/run_id   ↔   session/run/turn 三层
```

### AG-UI 事件信封（七字段，缺一不可，AG-UI事件规范.md）
`event_id`(全局唯一·重放不变·前端去重 :46-48,68) + `type` + `session_id`(聚合历史 :52) + `run_id`(运行状态 :54) + `sequence`(同 run 单调递增·合并依据 :63) + `timestamp`(服务端时间·仅展示·不作排序 :58-59) + `payload`(不泄露引擎内部实现 :42)。
- 基础事件 12 个：`agent.started/message.started/message.delta/message.completed/tool.call/tool.result/graph.node.started/graph.node.completed/interrupt.required/resume.accepted/agent.completed/agent.failed`。
- `message.delta` 增量渲染、`message.completed` 固化（:37-38）。
- **未知事件记录并忽略不崩溃**（:83）；服务端新增事件保持旧前端可降级（:84）。
- **断线双轨重连**：`last_event_id` 或 `run_id+sequence` 续传（:73）；无法续传回退 HTTP 查 run 状态（:74）。
- 错误事件 payload 必含 `error_code/message/recoverable/trace_id`（:79）。

### TurnLoop（TurnLoop执行规范.md）
- session/run/turn 三层，每轮必关联 session_id+run_id（:12）。
- **全状态持久化、不靠进程内存作唯一来源**（:47-48）——"会写循环"升级成"可运维 Agent"的分水岭。
- 用户输入先校验权限/会话状态/内容格式（:16）；追加输入先存为消息再推进（:17）。
- interrupt 存四要素：原因/待确认动作/过期时间/恢复上下文（:28）。
- resume 校验四要素：run/interrupt/用户权限/幂等键（:32），成功后发 `resume.accepted` 再继续（:33）。
- preempt 保存原因并发状态事件（:38）；长任务支持查询/取消/失败恢复/取结果（:42-43）。
- 不可恢复失败发 `agent.failed` + 保留 trace_id/run_id/错误分类（:58）。

### RPC 契约（RPC契约规范.md）
- 方法表达业务能力不表达表 CRUD（:69）；DTO 五要素：类型/含义/必填性/默认值/兼容规则（:17），不暴露业务库内部结构（:18）。
- 长任务走提交/查询模式，禁同步 RPC 无限等待（:36）。
- 只对幂等读或带幂等键的写重试，限次数+退避（:40-41）。
- 错误码六类：业务/权限/参数/幂等冲突/系统/超时（:30）。
- 权限上下文随请求传，**最终授权在业务服务**（:50-51）；写操作记审计字段(操作者/租户/来源/trace_id/动作/结果 :55)。
- 破坏性变更(删字段/改含义/改错误码)走新版本（:60）；需人工确认走 preview/confirm（:70）。
- **Agent 不绕过业务服务直写业务库**（:71）。

## 三、分域硬约束速查

### 流程·协作·文档 `[G]`
- DoR（开发流程:48-56）：产品 Done + 功能点 owner + 需求映射完成 + 契约 owner + DB 边界确认 + .env 足够。
- DoD（开发流程:62-67）：变更只覆盖该功能点 · 契约/数据模型/配置/SQL/文档同步 · 本地验证或说明原因+风险 · 按功能点拆分提交 · PR 含映射/契约/SQL/验证/风险。
- 分支 `feature|fix|docs|chore/<slug>`；提交 `<type>(<scope>): <summary>`，type∈feat/fix/docs/test/refactor/chore（GitHub:16-48）。
- PR 强制字段（Done状态/需求映射/契约影响/SQL/验证结果/风险）+ 用 PR 模板（GitHub:60-62）；按变更类型路由评审人（:68-72）。
- 文档三态：draft(不作实现依据)/active(当前事实源)/deprecated(不删·指替代)（文档:38-40,68）；必填元信息(owner/更新时间/适用范围/相关代码路径/相关契约 :44-48)；内容 7 段(背景/目标/非目标/约束/方案/风险/可验证验收 :52-58)。
- 冲突即停回报主控（交接:132-139）：产品自相矛盾/缺关键字段/需写业务事实但无 RPC/需展示未定义事件/违反业务规则/需越界。

### 后端栈 `[D]`（迁移换等价物）
Go 1.26.3(固定路径 `/Users/figo/sdk/go1.26.3`) · Kitex(RPC，业务=server/智能体=client) · Thrift(IDL，`api/thrift/{common,business,agent}/`) · Gin(仅 agent 的 API/SSE/WS/AG-UI；业务默认不暴露 HTTP) · PostgreSQL(业务库与 Agent 库隔离) · GORM(不承载业务规则，查询必传 context) · golang-migrate(**禁运行时 AutoMigrate** 改生产 schema) · etcd(服务发现+非敏感配置，key 带命名空间 `/dora/<env>/<service>/<key>`，密钥不进普通路径) · 火山引擎日志 · Go testing+testify。
- Gin handler 只做 transport 层；业务逻辑进 application/domain，DB 访问进 repository（栈:55-56）。
- 事务边界在 application 层明确；外部 RPC/模型调用一般不进 DB 事务（Go:58-60）。
- 生成代码不得手改，生成命令写进 PR，IDL 变更补 contract test（CW:68-71）。

### 数据·配置 `[G]`
- Agent 库 10 张白名单表：`agent_sessions/agent_runs/agent_messages/agent_events/agent_tool_calls/agent_tasks/agent_interrupts/agent_artifacts/agent_memories/agent_runtime_configs`（数据建模:18-27）。
- 上游标识(session_id/run_id/user_id/tenant_id)只作普通字段/过滤/审计，不作 schema 关联（:31）。
- 审计字段集：id/tenant_id/user_id/trace_id/created_at/updated_at/created_by/updated_by/deleted_at（:61）。
- 状态机不许任意跳转、必须可测试（:47）；删除优先软删/归档（:39）。
- 状态机取值：session{active/archived/expired} · run{pending/running/waiting_confirmation/resuming/completed/failed/cancelled} · task{pending/running/completed/failed/cancelled} · interrupt{required/accepted/rejected/expired/resolved}（:43-46）。
- 建议索引：runs(session_id,created_at)/events(run_id,sequence)/tasks(status,updated_at)/interrupts(run_id,status)/tool_calls(run_id,tool_name)（:51-55）。
- **配置化**（智能体配置化规范.md）：策略/提示词/模型参/Tool 白名单/Skill 白名单/Retriever/Memory/Workflow/Interrupt 策略/AG-UI 开关/限流并发，全进 `agent_runtime_configs`（:29-39,43）；配置必带版本/状态/owner/更新时间/审计且可回滚（:44-45）；密钥不进配置表只存引用 key（:46）；代码只做解释器/执行器不写死策略（:51）；每版本有最小验证集、映射产品验收（:56-58）。
- **迭代 SQL**（迭代SQL脚本规范.md）：`db/migrations/iterations/<日期-迭代名>/{agent,business}/` 双库分离 + README；命名 `NNNN_<feature_slug>.up|down.sql`（:34-37）；down 必备或写明不可回滚原因（:42）；禁词扫描 `FOREIGN KEY`/`REFERENCES`（:49）；PR 必含 SQL 或说明无需 + 列本地执行结果（:57-59）；本地 golang-migrate，线上 CentOS8 手动迁移 `[D]`。

### 存储·本地·安全 `[G]`
- **对象存储**（TOS对象存储规范.md）：后端签短期凭证(5-15min :142)+前端直传+confirm 两段，AK/SK 永不下发前端；object key 后端生成(:46)、字符集 ASCII 小写/数字/-/_///(:47)、禁含 PII/标题/原始文件名/提示词(:48)、含环境前缀+不可猜测 ID(:49-50)；新版本不覆盖旧 key(:52)；**下架重新公开必换 key**(:53)；公开作品走独立 `public/works` 快照前缀、详情只返 `public_media_url` 不返源 key(:97-103,154)；archived 项目禁签凭证/禁确认资产(:146)；临时对象 `tmp` 前缀 24h 内清理(:115-121)；CDN 第一版不支持缓存失效 `[D]`。
- **本地环境**（本地开发与配置规范.md）：macOS+Docker；依赖 PostgreSQL/etcd/mock 服务；端口 业务 Kitex 19001 / 智能体 Gin 18080 / 前端 3000 / PG 5432 / etcd 2379 `[D]`。**四层配置覆盖链**：默认→`.env.example`(占位)→`.env.local`(本机)→etcd(非敏感)→密钥系统(敏感)（:32-38）；新增配置项必须同步 .env.example + 规范（:28,80）；业务库 `dora_business` / Agent 库 `dora_agent` 分离 `[D]`。
- **安全**（安全规范.md）：认证上下文必备(:15)，业务授权只在业务层最终裁决、Agent 层只传上下文(:16-17)，前端不绕服务端权限(:18)；强制 tenant_id 隔离含测试数据、禁跨租户(:21-23)；**Tool 安全四件套**=输入/输出 schema+权限要求+错误分类(:34) + 改业务事实的 Tool 走 RPC 带幂等键(:35) + 高风险 Tool 人工确认(:36)；**禁 Agent 任意系统命令/任意 SQL/任意外部 HTTP**(:37)；不信任用户输入/网页/文件/检索中的指令(:40)、**Tool 调用权限不由模型输出决定**(:41)、系统提示词/密钥/内部配置不暴露给用户和模型生成内容(:43)；外部 URL 下载白名单+超时+大小(:50)；日志脱敏黑名单(完整Token/密码/AK/SK/支付凭证/未脱敏PII :60)；富文本展示 XSS 防护(:74)、SSE/WS 连接校验认证(:75)。
- **密钥纪律横切**：`.env.example` 只占位 → 真实值仅 `.env.local`/密钥系统 → 不进库/日志/前端/Agent DB（TOS:27,41-42 · 本地:66-71 · 安全:27,60,72）。

### 前端·测试
- 前端栈 `[D]`：React 19.2 + react-router-dom 7.18 + 纯 JS/JSX(无 TS) + Vite 8 + EventSource(SSE) 消费 AG-UI + adapter 边界 mock + 自写纯函数 reducer(`reduceAguiEvents`)。图标统一 `lucide-react`(:16)。
- 前端机制 `[G]`：**不发明后端字段**(契约为 SSOT :15,26,31) · 七态(loading/empty/error/success/streaming/interrupt/resume :21) · run 态与 UI 态分离(:22) · AG-UI 按 event_id 去重/sequence 合并/未知不崩/断线重放(:32-33,39) · 仅图标按钮必有 aria-label+tooltip(:17) · 错误态给可理解信息+重试/返回/支持/详情入口(:43-44)。
- 测试 `[G]`（测试规范.md）：单元覆盖错误分支+边界(:11) · 集成用隔离数据禁污染(:16) · 契约验 DTO/分页默认10/错误码/权限/超时/重试/幂等/版本兼容(:18-22) · AG-UI 验顺序/幂等/未知兼容(:22) · 浏览器留截图+日志+网络证据(:27) · Agent 库与业务库分别独立验证(:37-46) · 修复必复现原问题+回归相关路径(:48-51) · 缺陷报告九要素(级别/复现/期望/实际/浏览器/AG-UI/API·RPC/DB证据/定位/owner :53-55)。

## 四、改造取舍线
