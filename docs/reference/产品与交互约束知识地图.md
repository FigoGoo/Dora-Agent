## 二、统一 Agent 创作闭环（产品核心流程）

- **单一统一 Runtime，不做多 Agent**：一个 Agent 按上下文+意图在"当前空间 Skill 池"内选 Skill；意图识别本身不需人工确认（统一Agent:104,47,87）。
- **二级兜底（流程图 06PRD:71-75）**：无匹配 Skill → 先判意图是否命中"平台开放 Tool"，命中则按 Tool 边界直接调；否则落"文本对话兜底"。**兜底不打断、不推销建 Skill**（统一Agent:74,106）。
- **Skill 路由前提**：仅 Published + 当前空间可见 + 绑定 Tool 在该空间可用（企业空间用个人 Skill 时其 Tool 须符企业权限，否则不可用）（统一Agent:62,135）。
- **Tool 三类必须人工确认**：扣费 / 业务事实写入 / 高风险 Tool；取消后不发起新 Tool、不扣费（统一Agent:122,137）。Tool 风险三级 low/medium/high，确认要求挂在风险等级上（Tool边界:46-48）。
- **工作台必备区域**：一聊天区(消息流/思考/输入控件/确认/进度/错误) + **一个外部工作区**(资产视图/黑板视图切换，第一版不拆多组件)（06PRD:94-95,123-124）。
- **Skill 治理**：三类(系统/企业/个人)按创建者身份分级；系统 Skill 免审、企业/个人必审；测试通过 ≠ 审核通过；审核结果发站内信；版本可回滚形成新审计版本（SkillBuilder:40-51,141-146）。

## 三、交互承重墙（AG-UI / A2UI 契约，`[G]` 为主）

- **三件套边界**：AG-UI = agent→前端事件协议(SSE)；A2UI = 事件落到哪些前端组件/长什么样/什么状态；**HTTP API = 用户反向动作**(确认/取消/重试/补偿查询/媒体授权 URL，不走事件流)（交互契约设计）。
- **组件契约**：聊天区 **9 个组件**(message.stream / thinking.typewriter / platform.tags / credit.estimate / confirmation.panel / tool.status / generation.progress / error.notice / chat.input.controls) + 聊天外 **仅 1 个** `workspace.panel`(资产/黑板视图切换)。设计:99-103。
- **事件→组件 = 多对多"建议消费组件"映射**，一个事件可驱动多个组件（如 tool.call.started → tool.status + platform.tags）；组件须能**从标准事件降级渲染**，不强依赖扩展事件（矩阵:62-75）。
- **可靠性底线**：每事件可幂等识别(event_id 去重) + 同 run 顺序游标(sequence)合并；**未知事件忽略+记日志不崩溃**；**三级断线重连**(最后已消费事件标识 → run+游标 → 超窗口取运行快照)；缺口时暂停增量渲染进 reconnecting（设计:244-263）。
- **确认门状态机**：credits.estimated → confirmation.required 出确认卡 → Composer 进 locked(chat.controls.locked) → HTTP 确认 → credits.frozen → 才进生成；**积分不足不进确认**；改模型须先取消确认再重新预估；只有生成完成才 credits.charged，取消时已完成部分扣费、未完成 released（设计:122-126）。
- **思考与脱敏**：思考状态打印机流式 + 完成可折叠，只展示可公开处理状态；**敏感信息前后端双端脱敏**(不露推理链/Prompt/密钥/成本/供应商原文)（设计:116-119,197-202）。
- **错误分类**：user/permission/tool/model/rpc/system + blocked + archived_readonly，统一 error.notice，只给可理解原因+重试建议。
- **模型选择不是独立 A2UI 组件**，必须在 Composer 内以 capsule/Chip 完成，当前对话生效（设计:42 · 视觉:182）`[D]`。
- **页面分两类**：Agent 实时页(消费 AG-UI，仅工作台及子页) vs 非实时页(只走 API/RPC)（矩阵:21,131）。

## 四、UI/UE 体验与设计系统

### 信息架构与体验 `[G]`（结构）/ `[D]`（具体页/区名）
- **左右双层 Shell**：左 `AppSideNav`(Logo / **新建创作置顶** / 一级导航 / 底部主题登录) + 右 `PageContent`(ContextHeader 放当前空间/积分/通知/用户 + PageBody)；**主导航绝不放顶部横向**；未登录/登录两套导航同一 Shell 切换（IA:61-84,121-124）。
- **统一 Agent 工作台三栏**：顶 ProjectToolbar + 左 StoryboardPanel(故事板/分镜) + 中 PreviewStage(预览) + 右 ChatPanel(对话/资产/输入框)；沉浸模式隐藏全局 AppSideNav（工作台体验:29-35 · 视觉:42-59）。
- **核心交互九状态机**：loading→streaming→thinking→confirming→generating→partial_completed→blocked→reconnecting→save_failed（工作台体验:136-147）。
- **预览默认分流**：图/乐普通生成默认开资产视图；视频/MV/短视频默认开黑板(分镜)视图；用户切换后会话内记忆（视觉:341-344）。
- **登录弹窗承接**贯穿全站：未登录可浏览/可输入 Prompt，触发写动作(点赞/创作/上传/查个人内容)弹 LoginModal 并**保留页面+意图**，登录后继续（首页:78-79,127-137 · 12PRD:202）。
- **项目=创作容器心智**：首页创作自然建项目；工作台 ContextHeader 永显当前项目并按项目作用域加载会话/资产/黑板；项目详情五标签页(概览/会话/资产/黑板/作品)（项目体验:13-19,86-93）。
- **归档项目=只读流**：历史可看，Composer/继续创作/新建会话/上传/保存/确认扣费/建作品全禁用（项目体验:77-83）。
- **全局状态规范**(每页验收 checklist)：loading(Skeleton)/empty(说原因+入口)/error(可理解+重试)/success + 权限态 permission_denied/login_required/blocked/insufficient + 工作台态 streaming/interrupt/resume/reconnecting/archived_readonly（页面路由:113-126 · 页面范围与交互状态:127-152）。
- **首页轻量不 Dashboard 化**：只解决"看能力/确认身份/快速起步"；右侧四块=Prompt入口/快捷开始(3场景)/公开作品/最近项目（首页:18-36,140-153）。

### 设计系统方法论 `[G]`（值多为 `[D]`）
- **设计 Token 单一真源 + 语义命名**：`--bg/--surface/--surface-subtle/--line/--text/--muted/--accent` 按角色命名(非 `--gray-100`)；**日夜双主题共用同一套语义 Token，只换值不换名**，主题切换零页面级硬编码（视觉08:116-135,682-700）。
- **透明度/层级作为独立设计轴**：面板深浅靠 alpha 递进(0.05→0.10→0.15)，边框分三级(line-minimal/line/line-strong)独立成轴；玻璃=多层渐变叠极低 alpha 非实色（视觉08:120-126,161-204）。
- **三套字体栈**(--font-ui/--font-cn/--font-display) + 字号-字重-行高三元组按场景枚举；圆角/阴影/动效 easing 按组件语义分档；**图标单一来源 lucide-react**(尺寸 16/18/20/24)（视觉08:210-323,548-566）。
- **组件矩阵法**：每组件先定尺寸档(高度+padding+字号成组)×变体×**强制全状态**(default/hover/focus-visible/active/disabled/loading)；**loading/hover/长文本一律不改布局尺寸**(贯穿铁律)（公共组件09:36-56,104-119）。
- **Tag/Badge 同形不同义用类型区分**(SkillTag/ToolTag/ModelTag/StatusBadge/RiskBadge/AssetTypeTag/WorkCategoryTag)；**风险/状态绝不只靠颜色**(必配图标，色盲可达)（公共组件09:474-494）。
- **App Shell + 8 类页面模板枚举 + 页面状态矩阵 + 截图验收**(1440/1920/2560/768)：把"页面"约束成"模板实例 + 状态机"（页面模板10:39-93,443-479）。
- **后台与 C 端彻底隔离**：`admin_frontend` 独立入口/独立代码/独立登录态，默认日间高密度表格化，不用 C 端玻璃视觉，**不消费 AG-UI**；敏感信息成体系脱敏（后台06:13,58,261,284-287）。
