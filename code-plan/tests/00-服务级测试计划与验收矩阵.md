# 00-服务级测试计划与验收矩阵

状态：production-design-ready  
owner：浏览器、RPC 与数据库测试工程师  
更新时间：2026-06-27  
适用范围：非前端、非部署范围内的 Agent API、AG-UI 事件生产、业务 RPC、业务 HTTP API、Agent DB、业务 DB、跨服务主链路和错误路径验收  
相关代码路径：`tests/agent/**`、`tests/business/**`、`tests/contract/**`、`tests/e2e/**`、`api/thrift/**`、`api/openapi/**`、`api/agui/**`、`db/migrations/iterations/**`  
相关设计契约：`code-plan/agent/**`、`code-plan/business/**`、`docs/product/**`、`docs/standards/**`

## 目标

建立本次开发的服务级测试事实源，保证 Agent 微服务和业务微服务在没有前端开发、没有部署上线文档的范围内，仍能完成可验证的契约、数据、事件和业务主链路闭环。

## 非目标

- 不测试前端页面、路由、组件、状态管理、A2UI reducer 或浏览器 UI 自动化。
- 不编写部署上线、CI/CD、告警、SLO、生产运行手册或发布回滚手册。
- 不替代 Agent、业务或契约 owner 修改设计文档。

## 主链路验收矩阵

| 主链路 | 必测点 | 阻断标准 |
| --- | --- | --- |
| 创建 session/run | 空间解析、项目权限、Agent DB session/run、trace_id | 无法区分未登录、无权限和项目归档。 |
| 安全评估通过后预估 | `SafetyEvidenceDTO` 场景、摘要、过期时间，未通过不调用积分预估 | 安全未通过仍冻结或生成。 |
| 用户确认后冻结 | `confirmation_id`、`freeze_id`、幂等键、重复确认 | 同一确认重复冻结或幂等冲突不稳定。 |
| 生成成功保存资产 | 业务签发 object key、Agent 上传后 commit、业务 DB final asset/element、Agent DB asset ref | Agent 自造 object key 或业务资产未入库却扣费。 |
| 保存成功扣费 | `CommitGeneratedAssetAndCharge` 原子事务、ledger、released_points | 保存失败扣费，或扣费成功资产不存在。 |
| 失败/取消/归档释放 | `ReleaseFrozenCredits`、terminal run state、AG-UI failed/cancelled/blocked | 冻结未释放或 run 无终态。 |
| 独立 Tool 扣费 | `EstimateToolCredits`、`ChargeToolUsageCredits`、actual quantity、防重复明细 | Tool 重复扣费或超预估未阻断。 |
| SSE replay | sequence 连续、Last-Event-ID、gap 补偿、snapshot fallback | 缺口后继续合并造成状态错乱。 |
| 作品公开 | 公开快照、匿名访问、点赞、后台下架、隐私字段过滤 | 公开接口泄露源会话、提示词、成本或私有素材。 |
| 通知 | Skill 审核通知、未读数、已读、跳转权限 | 通知跳转到无权限资源。 |

## M3 / M4 / M6 阶段验收阻断

| 阶段 | 必须新增或强化的测试 | 阻断标准 |
| --- | --- | --- |
| M3 配置能力 | Agent API route parity 与 Authorization token 鉴权测试；真实 SSE stream / heartbeat / replay 测试；Skill / Tool / Model / PlatformDictionary RPC contract test；M3 业务 RPC server `NOT_IMPLEMENTED` 扫描；TurnLoop 安全评估、Skill 路由、Tool 策略和模型快照测试；Agent/业务依赖方向测试。 | 任一 M3 RPC 仍无 client、无业务 handler、无 fixture 或返回 `NOT_IMPLEMENTED`；测试脚本只做字符串门禁但报告写通过。 |
| M4 积分资产闭环 | Credit / Asset / AssetCreditCommit RPC contract test；积分预估、确认、冻结、扣费、释放、幂等冲突、重复 `estimate_item_id`、保存失败释放、归档释放、跨空间资产拒绝；Agent DB 只保存业务引用、业务 DB 不保存 Agent Runtime 的边界测试。 | 保存失败扣费、Agent 自造 object key、重复扣费、冻结未释放、跨库边界污染或 M4 方法返回 `NOT_IMPLEMENTED`。 |
| M6 服务级验收 | M2/M3/M4 全量 HTTP/RPC/OpenAPI/AG-UI/DB contract；业务 03/04/05/14 的完整行为测试；架构依赖方向测试；全量 route parity；全量 fixture 场景覆盖正常、权限、业务错误、幂等冲突、超时、版本兼容；测试报告真实性检查。 | M2 遗留项未关闭却进入 M6；fixture 只验证形状不验证语义；报告把未执行项、mock-only 项或缺 owner 项写成通过。 |

## 数据库验证边界

| 数据库 | 允许验证 | 不允许混淆 |
| --- | --- | --- |
| Agent DB | `agent_sessions`、`agent_runs`、`agent_messages`、`agent_events`、`agent_tool_calls`、`agent_tasks`、`agent_interrupts`、`agent_artifacts`、`agent_memories`、runtime config | 不把资产、积分、作品、用户、企业、项目事实当作 Agent DB 真相。 |
| 业务 DB | 用户、空间、企业、项目、模型、Tool 策略、Skill 生命周期、积分、资产、作品、通知、审计 | 不保存 Agent message、event、tool call、blackboard、memory。 |

## 执行与报告规则

- 实现完成后，涉及 Go 代码必须运行 `gofmt` 和对应 `go test`。
- 涉及契约必须运行 schema/fixture 校验和 contract test。
- 涉及 SQL 必须静态检查无数据库级外键或引用约束关键字，并验证 up/down 可执行。
- 涉及 AG-UI 必须校验 schema、payload、sequence、replay、unknown event 和 snapshot fallback。
- 未执行项必须写明：未执行命令、原因、影响、替代验证和 owner。
