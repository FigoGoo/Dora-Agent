| P0-2 | Agent DB、Repository、状态机 | 表字段和迁移计划确认 | CRUD、幂等、分页、状态迁移测试通过 |

| P0-5 | Eino TurnLoop、Skill 路由、Tool Runtime | Eino 版本和示例确认 | 文本兜底、Skill 路由、Tool 调用、事件输出可测 |
| P0-6 | 内容安全、积分确认、生成任务、资产保存闭环 | 安全证据和积分资产 RPC 确认 | 安全阻断、确认、冻结、保存、扣费/释放主链路通过 |
| P0-7 | 黑板、快照、会话恢复、Memory | snapshot schema 和黑板元素确认 | 断线补偿、snapshot fallback、历史恢复通过 |

## 可开发级需求映射矩阵

每条产品验收标准必须映射到 Agent 模块、函数、RPC、AG-UI、Agent DB 和测试。当前矩阵作为开发和评审的主索引：

| 产品验收 / 业务规则 | Agent 模块 | 目标函数或接口 | 【业务开发】需要提供的能力与参数 | AG-UI | Agent DB | 测试 |
| --- | --- | --- | --- | --- | --- | --- |
| 当前空间决定 Skill 池、积分账户、资产归属 | `application/run`、`runtime/skill` | `ResolveRunContext(ctx, req)`、`LoadRoutableSkills(ctx, context)` | `AccountSpaceService.ResolveCurrentSpaceContext(auth_context, request_meta)` 返回 `space_id`、`space_type`、`enterprise_role`、`credit_account_type`；`SkillCatalogService.ListRoutableSkills(auth_context, page_size=10)` 返回 Published Skill 摘要 | `platform.tags.updated` | `agent_runs.space_id`、`agent_runs.skill_selection` | 空间切换、企业成员、个人空间路由测试 |
| 项目必须绑定 session/run，归档后只读 | `application/session`、`application/run` | `CreateSession`、`CreateRun`、`ResumeRun`、`ConfirmInterrupt` | `ProjectService.CheckProjectAccess(auth_context, project_id, access_purpose)` 返回 `project_status`、`creative_allowed`、`allowed_actions`；归档返回 `PROJECT_ARCHIVED` | `project.archived.blocked`、`agent.run.cancelled` | `agent_sessions.project_id`、`agent_runs.project_id`、`agent_events.project_id` | 创建、恢复、确认、运行中归档测试 |
| 安全评估必须在预估、冻结、生成前 | `runtime/safety`、`runtime/turnloop` | `EvaluateSafety(ctx, req)`、`BuildSafetyEvidence(ctx, result)` | 如业务侧需校验证据，提供 `SafetyEvidenceDTO` 校验规则；写入 RPC 接收 `safety_evidence` | `safety.prompt.evaluating`、`safety.prompt.evaluated`、`safety.prompt.blocked` | `agent_safety_evaluations` | 通过、阻断、失败、超时、不冻结测试 |
| 积分预估、确认、冻结、扣费、释放 | `application/run`、`runtime/turnloop` | `EstimateCredits`、`CreateCreditInterrupt`、`FreezeCredits`、`CommitOrReleaseCredits` | `CreditService.EstimateGenerationCredits`、`FreezeCredits`、`ReleaseFrozenCredits`；参数含 `resource_type`、`model_pricing_snapshot_id`、`quantity`、`duration_seconds`、`idempotency_key` | `credits.estimated`、`confirmation.required`、`credits.frozen`、`credits.charged`、`credits.released` | `agent_interrupts`、`agent_tool_calls`、`agent_events` | 余额不足、确认取消、冻结失败、释放测试 |
| 生成完成且资产保存成功后才扣费 | `runtime/tool`、`infra/rpc` | `SaveGeneratedAssetAndCharge(ctx, req)` | `AssetCreditCommitService.CommitGeneratedAssetAndCharge(auth_context, request_meta, project_id, run_id, artifacts[], safety_evidence)` 原子保存资产和扣费，返回 `asset_refs[]`、`charged_points`、`released_points` | `asset.save.completed`、`credits.charged` | `agent_artifacts.business_ref_id` | 保存成功扣费、保存失败释放、部分完成测试 |
| 资产引用必须校验权限 | `runtime/retriever`、`infra/rpc` | `BatchCheckAssetAccess(ctx, req)` | `AssetService.BatchCheckAssetAccess(auth_context, asset_ids[], project_id, purpose)` 返回每个资产 `allowed`、`reason`、`asset_summary` | `error.notice`、`workspace.assets.updated` | `agent_artifacts` | 批量校验、跨空间拒绝、被移出企业测试 |
| AG-UI 事件可排序、幂等、补偿 | `events/agui`、`events/stream` | `PublishEvent`、`OpenRunStream`、`ReplayEvents` | 业务错误返回需包含稳定 `error_code`、`user_message`、`retryable`、`trace_id`，供事件 payload 映射 | 全部事件 | `agent_events(event_id, run_id, sequence)` | 重放、重复、缺口、未知事件测试 |

| 普通用户不能看到供应商、密钥、内部成本 | `runtime/modeltool`、`events/agui` | `ResolveModelSnapshot`、`BuildModelTagPayload` | `ModelConfigService` 返回用户展示字段和内部执行引用，Agent 不把内部字段写入事件 | `platform.tags.updated` | `agent_runs.model_selection_snapshot` | payload 脱敏测试 |
| Tool 停用、权限不足、高风险必须受控 | `runtime/tool`、`runtime/turnloop` | `CheckToolPolicy`、`CreateRiskInterrupt` | `ToolCapabilityService.CheckToolExecutionPolicy` 返回 `allowed`、`risk_level`、`requires_confirmation`、`timeout_ms`、`retry_policy` | `tool.call.failed`、`confirmation.required` | `agent_tool_calls`、`agent_interrupts` | Tool 停用、风险确认、非幂等不重试测试 |
| 会话历史可恢复对话、资产和黑板 | `application/session`、`events/stream`、`runtime/memory` | `BuildSessionSnapshot`、`ReplayEvents` | `ProjectService.CheckProjectAccess(view)`、`AssetService.BatchCheckAssetAccess` | `process.snapshot.saved`、`workspace.*` | `agent_messages`、`agent_events`、`agent_artifacts` | Last-Event-ID、snapshot fallback 测试 |

## 契约先行顺序

1. `ProjectService.CheckProjectAccess`、`AccountSpaceService.ResolveCurrentSpaceContext`、`AssetService.BatchCheckAssetAccess` 先确认，因为它们影响 session/run 创建和权限闭环。
2. `SkillCatalogService`、`ToolCapabilityService`、`ModelConfigService` 再确认，因为它们影响 Skill 路由、Tool 策略和模型选择。
3. `CreditService`、`AssetCreditCommitService` 再确认，因为它们影响扣费事务和失败补偿。
4. AG-UI 事件 payload 和 Agent API DTO 随上述 RPC 确认同步冻结。
5. Agent DB migration 在 API/RPC/AG-UI 字段冻结后生成，避免字段反复漂移。
6. 执行 [14](./14-生产级闭眼开发门禁与任务切片验收设计.md) 的门禁检查；未通过的切片不能进入编码。

## 业务开发对齐点

- 哪些业务能力必须先落 RPC 契约。
- 哪些字段属于业务事实，Agent 只能保存引用。
- 项目、积分、资产、作品和权限的最终解释权归属。
- Agent 侧是否需要额外业务批量查询能力来避免逐条 RPC。
- 所有业务 RPC 是否统一接收 `auth_context`、`request_meta` 和稳定错误结构。
- 写操作是否均支持 `idempotency_key`，并在参数不一致时返回 `IDEMPOTENCY_CONFLICT`。
- 安全证据 `safety_evidence` 是否由业务服务保存摘要并校验过期时间、场景和内容摘要。

## 【业务开发】需要提供的能力与参数

业务开发需要先按矩阵确认以下能力，Agent 侧才能进入对应批次实现：

| 能力组 | 必要方法 | 关键参数 |
| --- | --- | --- |
| 空间和项目 | `ResolveCurrentSpaceContext`、`CheckProjectAccess` | `auth_context`、`project_id`、`access_purpose`、`request_meta.trace_id`。 |
| 资产权限和摘要 | `BatchCheckAssetAccess` | `auth_context`、`project_id`、`asset_ids[]`、`purpose`、分页参数。 |
| Skill / Tool / Model | `ListRoutableSkills`、`GetPublishedSkillSpec`、`CheckToolExecutionPolicy`、`ResolveDefaultModel` | `skill_id`、`version`、`tool_name`、`resource_type`、`pricing_snapshot_id`。 |
| 积分和资产提交 | `EstimateGenerationCredits`、`FreezeCredits`、`CommitGeneratedAssetAndCharge`、`ReleaseFrozenCredits` | `run_id`、`estimate_id`、`freeze_id`、`artifacts[]`、`final_elements[]`、`safety_evidence`、`idempotency_key`。 |
| Skill 测试 | `GetReviewCandidateSkillSpec`、`SaveSkillTestResult`、`ListAssetElementTypes` | `skill_id`、`version`、`test_case_results[]`、`output_validation`。 |
| 安全证据校验 | 业务写入侧校验证据或同等安全服务 | `scene`、`target_type`、`target_ref_id`、`evaluated_object_digest`、`expires_at`、`policy_version`。 |
