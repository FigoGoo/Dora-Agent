## 文档目标

- 定义智能体微服务的架构边界和职责。
- 设计 Go package 分层和依赖方向。
- 说明每层使用的设计模式和职责。
- 明确 Agent 服务和业务服务、前端、数据库、Eino Runtime 的边界。

## 架构结论

智能体微服务采用分层架构，HTTP/SSE 层只负责协议适配，Application 层负责编排用例，Runtime 层负责 Eino、TurnLoop、Tool、Skill、Memory，Domain 层定义状态和事件，Infra 层封装 Agent DB、RPC、配置、日志和外部适配器。

```text
frontend
  -> api/http Handler
  -> application Service
  -> runtime TurnLoop / Eino / Tool / Skill
  -> domain Model / State / Event
  -> infra Repository / RPC / Config / Logger
  -> business rpc / agent db / sse stream
```

依赖方向只能自上而下：`api` 依赖 `application`，`application` 依赖 `runtime`、`domain` 和接口，`runtime` 依赖 `domain` 和端口接口，`infra` 实现端口接口。`domain` 不依赖任何基础设施。

## 包职责与设计模式

| 包 | 职责 | 设计模式 |
| --- | --- | --- |
| `internal/api/http` | Gin handler、请求解析、响应映射、SSE 入口 | Controller / Adapter |
| `internal/application/session` | 创建会话、查询快照、会话恢复 | Application Service |
| `internal/application/run` | 创建 run、追加输入、取消、snapshot | Application Service / Unit of Work |
| `internal/application/interrupt` | confirm、reject、resume 中断 | Application Service / State Machine |
| `internal/runtime/turnloop` | 多轮执行、状态推进、长任务和补偿 | Orchestrator / State Machine |
| `internal/runtime/eino` | Eino Agent、Graph、Workflow 构建和执行 | Runtime Adapter |
| `internal/runtime/tool` | Tool registry、执行、重试、取消、错误分类 | Strategy / Adapter |
| `internal/runtime/skill` | Skill spec、路由、输出元素结构 | Strategy / Retriever |
| `internal/runtime/memory` | 会话摘要、授权偏好、检索 | Repository + Policy |
| `internal/domain/model` | Agent model、DTO、值对象 | Domain Model |
| `internal/domain/state` | session/run/task/interrupt 状态机 | State Machine |
| `internal/domain/event` | 内部事件定义 | Domain Event |
| `internal/infra/repository` | Agent DB CRUD、事务、分页、幂等 | Repository |
| `internal/infra/rpc` | Kitex client、DTO 映射、错误映射 | Gateway / Adapter |
| `internal/events/agui` | 内部事件到 AG-UI payload 映射 | Mapper / Publisher |
| `internal/events/stream` | SSE 连接、重放、补偿 | Publisher |
| `internal/observability` | log、trace、metrics、callback | Middleware |

## 核心接口

```go
// RunApplicationService 负责一次 Agent run 的创建、推进、取消和快照。
type RunApplicationService interface {
    CreateRun(ctx context.Context, req CreateRunCommand) (CreateRunResult, error)
    AppendUserInput(ctx context.Context, req AppendUserInputCommand) (RunSnapshot, error)
    CancelRun(ctx context.Context, req CancelRunCommand) (CancelRunResult, error)
    GetSnapshot(ctx context.Context, req GetSnapshotQuery) (RunSnapshot, error)
}

// TurnLoop 负责推进多轮执行，不直接操作业务事实。
type TurnLoop interface {
    Start(ctx context.Context, input TurnInput) (TurnResult, error)
    Resume(ctx context.Context, input ResumeInput) (TurnResult, error)
    Preempt(ctx context.Context, input PreemptInput) (PreemptResult, error)
}

// BusinessGateway 聚合业务 RPC 能力，具体实现位于 infra/rpc。
type BusinessGateway interface {
    ResolveCurrentSpace(ctx context.Context, req ResolveCurrentSpaceRequest) (SpaceContext, error)
    CheckProjectAccess(ctx context.Context, req CheckProjectAccessRequest) (ProjectAccessResult, error)
    BatchCheckAssetAccess(ctx context.Context, req BatchCheckAssetAccessRequest) (BatchAssetAccessResult, error)
}
```

## 启动流程

1. 读取 `.env` 和配置文件，解析 Agent Runtime 配置。
2. 初始化 logger、trace、metrics。
3. 初始化 Agent DB 连接和 repository。
4. 初始化 Kitex RPC client。
5. 初始化 Eino runtime factory、Tool registry、Skill runtime、TurnLoop。
6. 初始化 HTTP router 和 SSE stream manager。
7. 启动健康检查、优雅退出和后台清理任务。

## 禁止依赖

- `runtime` 不直接依赖 GORM 或 Kitex 具体实现，只依赖端口接口。
- `domain` 不依赖 `infra`、`api`、`runtime`。
- `api/http` 不直接访问 repository 和 RPC client。
- `events/agui` 不读取业务数据库，不自行补业务事实字段。
- 任何包都不得保存或输出 API Key、系统 Prompt、完整供应商响应、内部成本。

## 架构级测试

| 测试 | 验证点 |
| --- | --- |
| 依赖方向测试 | 使用静态检查或包导入检查，防止 `domain -> infra`。 |
| Handler 测试 | HTTP 状态码、错误映射、幂等头缺失。 |
| Application 测试 | 创建 run 时必须先校验项目和资产权限。 |
| Runtime 测试 | TurnLoop 状态变化必须输出事件并持久化。 |
| Infra 测试夹具测试 | RPC 错误、超时、幂等冲突能映射到领域错误。 |

## 业务开发对齐点

- Agent 侧 `infra/rpc` 只能调用业务服务能力，不直接访问业务数据库。
- 业务服务需要提供哪些批量 RPC 和权限校验 RPC。
- Agent 侧保存 `project_id`、`asset_id` 等引用不代表业务事实来源。

## 【业务开发】需要提供的能力与参数

| 架构依赖 | 业务能力 | 关键参数 |
| --- | --- | --- |
| 权限上下文 | 统一 `AuthContext` DTO | `actor_user_id`、`login_identity_type`、`space_id`、`enterprise_id`、`enterprise_role`。 |
| 请求元信息 | 统一 `RequestMeta` DTO | `request_id`、`trace_id`、`idempotency_key`、`source=agent`。 |
| 项目上下文 | 项目创建、读取、权限校验 | `project_id`、`access_purpose`、`project_status`、`creative_allowed`。 |
| 业务展示摘要 | 项目、资产、模型、Skill、Tool 摘要查询 | 只返回前端/Agent 可展示摘要，不返回业务内部表结构。 |
| 稳定错误结构 | 所有 RPC 统一错误响应 | `error_code`、`user_message`、`retryable`、`trace_id`、`details`。 |
