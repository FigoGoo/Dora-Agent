## 文档目标

- 定义 Agent 微服务的日志、trace、指标和脱敏规则。
- 定义错误分类和用户可见错误映射。
- 定义 Agent Runtime 配置化边界。
- 定义功能点验收测试清单。

## 测试矩阵必须覆盖

| 测试类别 | 用例 |
| --- | --- |
| 函数级单元测试 | DTO 校验、状态机迁移、错误映射、幂等键生成、安全摘要哈希。 |
| TurnLoop 测试 | 正常生成、确认取消、安全阻断、Tool 失败、项目归档、部分完成、恢复后继续。 |
| RPC 测试夹具测试 | `CheckProjectAccess`、`BatchCheckAssetAccess`、`ResolveGenerationModelSnapshot`、`EstimateGenerationCredits`、`EstimateToolCredits`、`FreezeCredits`、`ChargeToolUsageCredits`、`PrepareGeneratedAssetObjects`、`CommitGeneratedAssetAndCharge`、`ReleaseFrozenCredits`、`GetReviewCandidateSkillSpec`、`SaveSkillTestResult`、`ListAssetElementTypes` 正常和错误路径。 |
| Contract test | 请求/响应字段、分页默认 10、错误码、超时、重试、幂等冲突、版本兼容。 |
| AG-UI 重放测试 | 正常序列、重复 `event_id`、sequence 缺口、unknown event、Last-Event-ID、snapshot fallback。 |
| Agent DB 测试 | CRUD、索引查询、无外键约束、无业务事实字段、事件补偿、软删除/归档。 |
| 安全证据测试 | scene/target_type/digest 匹配、过期证据、阻断、失败、超时、不泄露原文。 |
| 模型 Tool 测试 | 提交、查询、取消、超时、供应商错误、部分完成、供应商产物下载/解码、TOS 上传、object key 不匹配。 |
| E2E 主链路 | 注册登录、项目创建、run、SSE、确认、生成、资产保存、扣费、恢复、项目归档阻断。 |

## 测试夹具数据必须覆盖

- 个人空间和企业空间各一套 `auth_context`。
- active / archived 项目。
- Published / ReviewCandidate / Deprecated Skill。
- 可用 / 停用 / 高风险 Tool。
- 默认模型 / 停用模型 / 价格缺失模型。
- 余额足够 / 余额不足 / 冻结失败积分账户。
- Tool 用量预估 / 独立 Tool 扣费 / 生成资产扣费 / 重复 `estimate_item_id` 幂等。
- 生成产物对象准备 / 上传授权过期 / object key 不匹配 / 部分 artifact 上传成功。
- 可访问 / 跨空间 / 被移出企业后的资产。
- Skill 审核候选 spec / 少于 3 个测试样例 / 输出元素类型非法 / Skill 测试结果重复回传。
- passed / blocked / failed / expired 安全证据。

## package 设计

```text
services/agent/internal/observability/
  logger.go
  trace.go
  metrics.go
services/agent/internal/apperror/
  error.go
  mapper.go
services/agent/internal/infra/config/
  config.go
  runtime_config.go
```

日志使用结构化 JSON；错误通过 `apperror.Code` 统一映射 HTTP、AG-UI 和日志字段；配置读取失败时服务启动失败，外部供应商未配置时对应 Tool 返回明确 `PROVIDER_CONFIG_MISSING`，不返回虚假成功。

## 错误分类表

| Agent 错误码 | 用户可见 | 来源 | HTTP / AG-UI |
| --- | --- | --- | --- |
| `AUTH_REQUIRED` | 是 | 鉴权缺失 | HTTP 401 |
| `PROJECT_ARCHIVED` | 是 | 业务 RPC | HTTP 409 / `project.archived.blocked` |
| `ASSET_PERMISSION_DENIED` | 是 | 业务 RPC | HTTP 403 / `workspace.assets.updated` |
| `SAFETY_BLOCKED` | 是 | Safety Evaluator | `safety.prompt.blocked` |
| `CREDIT_INSUFFICIENT` | 是 | 业务 RPC | `credits.insufficient` |
| `CONFIRMATION_EXPIRED` | 是 | Interrupt | `confirmation.expired` |
| `PROVIDER_CONFIG_MISSING` | 是 | Tool adapter | `tool.call.failed` |
| `PROVIDER_TIMEOUT` | 是 | Tool adapter | `tool.call.failed` |
| `RPC_TIMEOUT` | 是 | RPC client | `agent.run.failed` |
| `INTERNAL_ERROR` | 是，泛化文案 | 代码异常 | HTTP 500 / `agent.run.failed` |

## 配置键

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| `APP_ENV` | `local` | 运行环境。 |
| `APP_NAME` | `dora-agent` | 服务名。 |
| `LOG_LEVEL` | `debug` | 日志级别。 |
| `AGENT_HTTP_ADDR` | `0.0.0.0:18080` | Agent API 监听地址，对齐 `.env.example`。 |
| `AGENT_DATABASE_URL` | 无默认 | Agent Runtime 数据库连接，对齐 `.env.example`。 |
| `BUSINESS_SERVICE_NAME` | `dora.business` | 业务 Kitex 服务名。 |
| `KITEX_REGISTRY` | `etcd` | Kitex 注册中心类型。 |
| `ETCD_ENDPOINTS` | `http://127.0.0.1:2379` | 本地服务发现地址。 |
| `KITEX_TIMEOUT_MS` | `3000` | Kitex 默认超时。 |
| `AGENT_SSE_ENABLED` | `true` | SSE 开关。 |
| `AGENT_WS_ENABLED` | `true` | WebSocket 预留开关，生产级实现不默认作为实时通道。 |
| `AGENT_CONFIG_SOURCE` | `postgres` | Agent runtime config 来源。 |
| `AGENT_DEFAULT_CONFIG_VERSION` | `local-dev` | 默认运行配置版本。 |
| `AGENT_TOOL_ALLOWLIST` | 空 | Tool 白名单，逗号分隔。 |
| `AGENT_MEMORY_ENABLED` | `true` | 全局 Memory 开关；默认启用 session summary，`user_preference` 和 `space_preference` 必须同时满足 Skill `memory_policy` 与 Agent 授权。 |
| `AGENT_SSE_HEARTBEAT_SECONDS` | `15` | SSE heartbeat 间隔。 |
| `AGENT_EVENT_REPLAY_PAGE_SIZE` | `10` | 补偿默认分页。 |
| `AGENT_EVENT_REPLAY_MAX_PAGE_SIZE` | `100` | 补偿分页上限。 |
| `AGENT_TOOL_DEFAULT_TIMEOUT_MS` | `120000` | Tool 默认超时。 |
| `AGENT_SAFETY_POLICY_VERSION` | `local-v1` | 安全策略版本。 |
| `MODEL_PROVIDER`、`IMAGE_PROVIDER`、`VIDEO_PROVIDER`、`MUSIC_PROVIDER` | 空 | 外部模型供应商配置；未配置时返回 `PROVIDER_CONFIG_MISSING`。 |

`.env.example` 当前缺少 `AGENT_SSE_HEARTBEAT_SECONDS`、`AGENT_EVENT_REPLAY_PAGE_SIZE`、`AGENT_EVENT_REPLAY_MAX_PAGE_SIZE`、`AGENT_TOOL_DEFAULT_TIMEOUT_MS`、`AGENT_SAFETY_POLICY_VERSION`。实现 P0-1 时必须补齐模板，或在配置加载器中提供上述默认值并在启动日志输出最终值。

## 日志和指标

| 类型 | 名称 | 标签 |
| --- | --- | --- |
| counter | `agent_run_total` | `status`、`space_type`、`skill_scope`。 |
| counter | `agent_rpc_error_total` | `rpc_method`、`error_code`。 |
| histogram | `agent_turn_duration_ms` | `status`、`resource_type`。 |
| histogram | `agent_tool_duration_ms` | `tool_name`、`provider`、`status`。 |
| gauge | `agent_sse_connections` | `service_instance`。 |
| counter | `agent_safety_evaluation_total` | `scene`、`result`、`policy_version`。 |

所有日志必须带 `trace_id`、`run_id`、`session_id`、`project_id`、`actor_user_id`，敏感文本输出摘要哈希和长度，不输出原文。

## 本地验证命令

| 范围 | 命令 |
| --- | --- |
| Go 格式化 | `gofmt ./services/agent/...` |
| Go 单测 | `go test ./services/agent/...` |
| RPC contract | `go test ./tests/contract/...` |
| Agent DB | `go test ./tests/agent/db/...` |
| AG-UI replay | `go test ./tests/agent/agui/...` |

当工程骨架尚未创建对应目录时，验收报告必须记录“未执行：目录不存在”，不能把缺失测试报告为通过。

## 验收报告要求

每个功能点完成后输出：修改文件、中文功能逻辑说明、边界说明、执行命令、测试结果、未覆盖风险、需要业务开发提供的 RPC 或 fixture。外部模型、对象存储、短信、支付等未配置时必须返回明确配置错误，并在测试中断言错误码。

## 业务开发对齐点

- RPC 错误码和 trace_id 是否贯通。
- 业务服务是否返回可展示错误摘要。
- 业务审计和 Agent trace 如何关联。
- contract test 数据由谁维护。

## 【业务开发】需要提供的能力与参数

| 测试输入 | 业务开发需要提供 |
| --- | --- |
| RPC 测试夹具 schema | 每个 RPC 的正常响应、业务错误、权限错误、幂等冲突和超时样例。 |
| contract test fixture | Thrift DTO 样例、错误码枚举、分页默认值、request hash 规则。 |
| Tool 和 Skill fixture | `ResolveGenerationModelSnapshot`、`EstimateToolCredits`、`ChargeToolUsageCredits`、`PrepareGeneratedAssetObjects`、`GetReviewCandidateSkillSpec`、`SaveSkillTestResult`、`ListAssetElementTypes` 的正常、失败、幂等和权限样例。 |
| 业务错误映射 | `error_code`、`user_message`、`retryable`、`trace_id` 字段稳定输出。 |
| 审计关联 | 写 RPC 返回或记录 `audit_ref` / `ledger_ref` / `trace_id`，供 Agent 日志关联。 |
| 安全证据校验样例 | passed、expired、digest mismatch、scene mismatch 四类样例。 |
