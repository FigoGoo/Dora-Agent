## 文档目标

- 定义“全量闭眼开发到生产”的进入门禁。
- 把 Agent 开发拆成可直接提交、可测试、可验收的功能切片。
- 明确哪些缺失项必须阻断开发，禁止带着口头约定进入编码。
- 定义每个切片的产物、命令、测试证据和失败恢复策略。

## 闭眼开发定义

满足以下条件时，开发者可以按文档直接开发，不需要再猜业务语义、字段、状态、错误码和测试标准：

| 维度 | 必须满足 |
| --- | --- |
| 契约 | Thrift IDL、OpenAPI、AG-UI JSON Schema、Agent DB migration 字段冻结。 |
| 代码落点 | 每个功能点有明确 package、接口、DTO、Repository、RPC client 和测试路径。 |
| 状态闭环 | session、run、task、interrupt、event、snapshot 的状态迁移和失败补偿完整。 |
| 业务边界 | 所有业务事实读写都有 RPC 能力、参数、错误码和幂等规则。 |
| 产品一致性 | 统一 Agent、无 Skill 文本兜底、直接平台 Tool、当前 run 模型锁定、AG-UI/A2UI 事件、积分资产闭环全部能映射到 `docs/product` 中 Done 状态 PRD。 |
| 安全边界 | 安全证据字段、摘要哈希、过期、场景、失败行为和业务校验规则明确。 |
| 配置 | `.env.example` 与配置加载器字段一致；缺配置返回明确错误。 |
| 测试 | 每个切片有单元、集成、contract、AG-UI replay 或 DB 测试至少一类证据。 |
| 验收 | 验收报告必须写明执行命令、结果、未执行原因和风险。 |

当前 `code-plan/agent/**` 已完成文档设计层冻结，允许按功能切片进入正式开发。上表中的 Thrift IDL、OpenAPI、AG-UI JSON Schema、Agent DB migration、fixture 和配置文件是后续实现落点；若现有临时代码与 `code-plan` 不一致，以 `code-plan` 为准，在对应切片内先补契约再写实现。

## 实现与联调阻断门禁

以下任一情况存在时，不允许合并、联调或验收对应功能点；但不阻断以 `code-plan` 为事实源启动该功能点开发：

| 阻断项 | 阻断范围 | 处理方式 |
| --- | --- | --- |
| Thrift 字段编号未冻结 | RPC client、业务写入、contract test | 按 `code-plan/agent/07` 与业务 15 先更新 `api/thrift/business_agent_service.thrift`。 |
| OpenAPI schema 缺失 | Agent API、前端联调 | 按 `code-plan/agent/04` 先更新 `api/openapi/agent-workbench.yaml`。 |
| AG-UI schema 或 fixture 缺失 | SSE、前端 reducer、事件补偿 | 按 `code-plan/agent/06` 先更新 `api/agui/agent-workbench-events.schema.json` 和 fixture。 |
| migration 字段类型缺失 | Agent DB、Repository | 按 `code-plan/agent/03` 先补 `db/migrations/iterations/<date>_agent_runtime/agent/*.sql`。 |
| `.env.example` 缺配置 | 服务启动、外部 Tool、SSE | 先补配置模板或在加载器提供默认值。 |
| 业务 RPC 测试夹具缺错误路径 | TurnLoop、积分、资产、权限 | 按 `code-plan/agent/07` 与业务 15 先补 contract fixture。 |
| 外部供应商未配置 | 模型 Tool 真实调用 | 返回 `PROVIDER_CONFIG_MISSING`；生产运行不得使用隔离测试 adapter。 |

## 产品一致性 Done Gate

| 产品要求 | Agent 验收点 |
| --- | --- |
| 一个统一 Agent 承接全部创作场景 | 不按音乐、MV、电商图、旅游视频、Logo 等场景拆 Agent；路由只选择 Skill、Tool、模型和控件。 |
| 无匹配 Skill 时文本模型兜底 | `agent.skill.missing` 后进入文本模型回复，不调用未授权 Tool，不伪造 Skill。 |
| 直接 Tool 意图只能使用平台开放 Tool | 调用 `ToolCapabilityService.CheckToolExecutionPolicy`，非开放、停用、高风险不满足确认条件时拒绝。 |
| 模型选择只影响当前会话 run，确认后锁定 | `chat.controls.requested` 提供模型控件；进入 `confirmation.required` 后发送 `chat.controls.locked`，后续修改必须新开 run。 |
| 安全评估先于积分预估、冻结和生成 | `safety.prompt.evaluated` 未通过时不出现 `credits.estimated`、`credits.frozen`、`tool.call.started`。 |
| 生成成功保存资产后再扣费 | Agent 必须先通过 `PrepareGeneratedAssetObjects` 上传到业务签发 TOS object key，再调用 `CommitGeneratedAssetAndCharge`；`asset.save.completed` 之前不发送 `credits.charged`，保存失败必须释放冻结。 |
| Tool 用量可独立计费 | 直接 Tool 使用 `EstimateToolCredits` + `ChargeToolUsageCredits`，生成链路 Tool 用量通过 `EstimateGenerationCredits.tool_usage_items[]` 和 `CommitGeneratedAssetAndCharge.artifacts[].estimate_item_id` 结算。 |
| AG-UI/A2UI 组件事件完整 | fixture 覆盖 `agent.thinking.*`、`platform.tags.updated`、`chat.controls.*`、`confirmation.*`、`tool.call.*`、`generation.*`、`asset.save.*`、`workspace.*`。 |
| Agent 不保存业务可见资产事实 | Agent DB 只保存 artifact / asset ref / snapshot；最终资产、元素、积分流水以业务 RPC 返回为准。 |

## 生产验收矩阵

| 主链路 | 必测点 |
| --- | --- |
| 注册登录后创建项目并创建 session | 权限、空间上下文、session 落库。 |
| 创建 run 并打开 SSE | run 并发、事件 sequence、Last-Event-ID。 |
| 安全评估通过后积分确认 | safety evidence、estimate、confirmation payload 锁定。 |
| 用户确认后冻结并生成 | 幂等确认、freeze_id、Tool task 状态。 |
| 生成完成保存资产并扣费 | `ResolveGenerationModelSnapshot`、`PrepareGeneratedAssetObjects`、`CommitGeneratedAssetAndCharge` 顺序，object key 校验、asset_ref、charged/released points。 |
| 直接平台 Tool 调用成功 | `EstimateToolCredits`、`FreezeCredits`、`ChargeToolUsageCredits` 顺序、幂等和 `charged_line_items[]`。 |
| 保存失败或取消 | 释放冻结积分、run terminal state、错误 AG-UI。 |
| 项目运行中归档 | 停新 Tool、释放未结算冻结、`project.archived.blocked`。 |
| SSE 断线恢复 | event replay、sequence 缺口、snapshot fallback。 |
| Skill 发布前测试 | 至少 3 个样例、安全证据、输出元素校验、测试结果回传。 |
| 模型供应商错误 | 鉴权错误、限流、超时、部分完成、取消。 |

## 提交和验收要求

- 每个切片独立提交，提交说明必须包含中文功能逻辑说明、边界说明和测试证据。
- 生产环境没有真实外部服务配置时必须返回明确配置错误；测试环境可使用隔离测试 adapter，但测试报告必须标注。
- 任何跨服务字段变更必须先改契约，再改调用方和服务方。
- 任何新增 AG-UI 事件必须先改 schema 和 fixture，再改 producer 和 reducer。
- 任何新增 Agent 表字段必须先改 migration 和 model，再改 repository。
