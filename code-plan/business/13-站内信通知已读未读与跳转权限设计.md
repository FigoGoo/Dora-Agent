## 目标

实现站内信通知。Skill 审核通过或拒绝后通知创建者；通知写入失败不回滚审核事实，但必须有补偿或失败审计。

## 非目标

- 不创建 Agent run 完成通知；Agent 工作台状态通过 AG-UI 展示。
- 不把通知作为业务事实来源，跳转前必须重新校验目标资源权限。
- 不发送短信、邮件或外部推送。

## 业务规则

- 通知不是业务事实来源，跳转时必须重新校验关联资源权限。
- Skill 审核通知创建者，审核事实不因通知失败回滚。
- 通知正文不得包含系统 Prompt、内部审核策略、密钥、供应商配置。
- 通知列表只返回当前登录用户自己的通知。
- 未读数按 `read_at IS NULL` 计算。

## 事务设计

| 事务 | 原子写入 | 回滚条件 |
| --- | --- | --- |
| 创建通知 | `notifications`、幂等记录 | 接收用户不存在、幂等冲突 |
| 创建通知失败补偿 | `notification_create_failures`、失败摘要 | 原业务事实不回滚；补偿记录写入失败时写 error 日志 |
| 单条已读 | `notifications.read_at`、幂等记录 | 通知不属于当前用户 |
| 全部已读 | 当前用户匹配通知批量更新 `read_at`、幂等记录 | 用户未登录 |
| 跳转权限校验 | 只读校验目标资源，不写业务状态 | 目标资源不可见返回 `PERMISSION_DENIED` |

## 【Agent开发】需要提供的能力与参数

| 场景 | 业务服务行为 | 【Agent开发】要求 |
| --- | --- | --- |
| Skill 审核通知 | 业务服务创建通知 | Agent 不生产站内信 |
| Agent run 完成通知 | 不创建站内信，通过 Agent 工作台状态和 AG-UI 展示 | Agent 不调用通知 RPC |
| 通知跳转到 Skill 或作品 | 前端调用业务 API 重新校验权限 | Agent 无需参与 |
| trace 关联 | 若通知来自 Agent 触发的业务流程，可记录 `trace_id` | Agent 调业务写 RPC 时必须传 trace |

## 测试

- 通知创建幂等。
- 审核通知失败不回滚审核事实，并记录补偿。
- 用户不能读取他人通知。
- MarkAll 只影响当前用户。
- 跳转资源无权限时前端二次查询返回权限错误。
