## 目标

建立业务服务身份和空间事实源。当前空间决定 Skill 池、积分账户、资产归属、项目和历史可见范围。业务服务负责最终权限判断，Agent 和前端不能自行推断业务权限。

## 非目标

- 企业角色范围固定为 `owner`、`member`；新增企业管理员角色必须先更新权限矩阵、RPC 契约、审计动作和测试用例。
- 企业成员不能主动退出企业。
- 企业拥有者不额外查看成员资产、会话、任务、黑板或生成记录。
- 不保存 Agent 会话或 run。

## 需求映射矩阵

| 产品条目 | 业务解释 | 业务产出 | 【Agent开发】依赖 |
| --- | --- | --- | --- |
| 个人登录对应个人空间 | 用户注册时创建个人空间和个人积分账户 | `RegisterPersonalAccount` 事务 | Agent 调用空间解析获得 `space_type=personal` |
| 企业登录对应企业空间 | 当前登录身份必须绑定 active 企业成员 | `ResolveCurrentSpaceContext` | Agent run 必须携带当前 `space_id`，业务返回企业角色 |
| 企业拥有者管理成员和企业积分 | owner 权限在业务服务校验 | `EnterpriseService` | Agent 不因企业 owner 身份扩展成员资产可见范围 |
| 当前空间决定 Skill/积分/资产 | 空间上下文是所有业务 RPC 输入 | `SpaceContextDTO` | Agent 每次创建 run、加载 Skill、预估积分前解析或校验空间 |
| 被移出企业后失去访问 | `enterprise_members.status=removed` 后拒绝企业空间 | 权限校验 | Agent 恢复会话和引用资产时需处理 `PERMISSION_DENIED` |
| 企业空间可使用个人 Skill | 当前用户的个人 Skill 可在企业空间运行，但运行产物和扣费归企业空间 | `ResolveCurrentSpaceContext`、`skill_scope_keys[]` | Agent 不自行拼接 scope，只使用业务返回的 Skill 池和 `credit_account_scope` |

## 权限规则

- `personal` 身份只能访问自己的个人空间资源。
- `enterprise_member` 身份必须对应 active 企业成员。
- 企业 owner 可以邀请、移除非 owner 成员、转让 owner、管理企业积分。
- 企业 member 可以在企业空间创作和使用企业 Skill，但不能管理成员或企业 Skill。
- 企业成员在企业空间可以使用自己的个人 Skill；生成资产、作品草稿、积分预估和扣费均归企业空间，且必须遵守企业 Tool allowlist 和企业积分账户。
- 企业 owner 不能查看成员私有项目、资产、会话、黑板和生成记录。
- 用户 `disabled` 时拒绝所有登录态私有操作。

## 事务设计

| 事务 | 原子写入 | 回滚条件 |
| --- | --- | --- |
| 个人注册 | user、personal space、credit account、idempotency | 任一写入失败 |
| 企业创建 | enterprise、enterprise space、owner member、enterprise credit account、audit | 任一写入失败 |
| 接受邀请 | invite status、member、audit | 已有 active 企业身份、邀请过期 |
| 移除成员 | member removed、audit | 目标为 owner、非 owner 操作 |
| 转让 owner | old owner -> member、new member -> owner、enterprise owner_user_id、audit | 目标非 active 成员 |

## 【Agent开发】需要提供的能力与参数

| 【Agent开发】场景 | 业务 RPC | Agent 必传参数 | Agent 处理返回 |
| --- | --- | --- | --- |
| 创建 run 前解析上下文 | `ResolveCurrentSpaceContext` | `actor_user_id`、`login_identity_type`、`space_id`、`trace_id` | 使用返回的 `space_id`、`enterprise_role`、`credit_account_scope` 作为后续 RPC 上下文 |
| 加载 Skill 池 | 先 `ResolveCurrentSpaceContext`，后 `ListRoutableSkills` | `skill_scope_keys[]` 或 `space_id` | 不自行拼接企业/个人 Skill 范围 |
| 企业成员被移除后恢复会话 | `ResolveCurrentSpaceContext` 或资源 RPC 返回 `PERMISSION_DENIED` | `session_id`、`project_id`、当前身份 | 停止恢复企业空间私有创作过程，AG-UI 展示权限不足 |
| 企业空间积分预估 | `ResolveCurrentSpaceContext` 后 `EstimateGenerationCredits` | `credit_account_scope=enterprise` | 企业积分不足时不得切换个人积分 |
| 企业空间使用个人 Skill | `ResolveCurrentSpaceContext` 返回企业 `space_id` 和 `credit_account_scope=enterprise`，`ListRoutableSkills` 返回可运行 Skill | 当前用户身份、企业空间 | 输出、资产和扣费都使用企业空间，不回写个人空间 |

## 测试

- 注册默认个人空间和积分账户同事务。
- 企业创建默认 owner 成员同事务。
- 企业普通成员不能创建企业 Skill 或管理成员。
- owner 不能移除 owner，转让后原 owner 降级。
- 被移出企业后企业空间 RPC 返回 `PERMISSION_DENIED`。
- `ResolveCurrentSpaceContext` 对错空间返回 `CROSS_SPACE_DENIED`。
- 企业空间使用个人 Skill 时业务返回 `credit_account_scope=enterprise`；Agent 预估和资产保存不得落到个人空间。
- 列表分页默认 10，最大 50。
