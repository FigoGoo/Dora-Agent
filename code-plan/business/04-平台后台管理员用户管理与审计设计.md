## 目标

实现平台后台基础能力：平台管理员账号、用户基础信息查询、用户启用禁用、后台操作审计和敏感信息脱敏。后台不能查看或下载用户私有资产、会话、黑板、提示词、私有素材和生成记录。

## 非目标

- 后台权限范围固定为同权平台管理员；新增 RBAC 必须先更新后台权限矩阵、审计动作、API 契约和测试用例。
- 平台管理员不得代登录普通用户。
- 用户管理不展示用户私有创作内容。
- 不管理 Agent Runtime。

## 需求映射矩阵

| 产品条目 | 业务解释 | 业务产出 | 【Agent开发】依赖 |

## 详细数据库表设计

### `platform_admins`

| 字段 | 类型 | 必填 | 默认值 | 索引/约束 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `admin_id` | varchar(64) | 是 | 生成 | pk/unique | 平台管理员 ID |
| `account` | varchar(120) | 是 |  | unique | 登录账号 |
| `display_name` | varchar(80) | 是 | 空字符串 | idx | 后台展示名称 |
| `password_hash` | varchar(255) | 是 |  |  | 密码 hash |
| `must_rotate_password` | boolean | 是 | false | idx | 初始管理员首次登录后必须修改密码 |
| `status` | varchar(32) | 是 | `active` | idx | `active`、`disabled` |
| `created_by` | varchar(64) | 否 | null | idx | 创建该管理员的 admin_id |
| `disabled_by` | varchar(64) | 否 | null | idx | 停用操作人 |
| `disabled_reason` | varchar(512) | 否 | null |  | 停用原因摘要 |
| `last_login_at` | timestamptz | 否 | null | idx | 最近登录时间 |
| `created_at` | timestamptz | 是 | now() | idx | 创建时间 |
| `updated_at` | timestamptz | 是 | now() |  | 更新时间 |

所有 `active` 管理员权限相同。停用当前登录管理员必须由 application 阻断；引入细分后台角色前不得在代码中预留未定义权限分支。

### `platform_admin_bootstraps`

| 字段 | 类型 | 必填 | 默认值 | 索引/约束 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `bootstrap_id` | varchar(64) | 是 | 生成 | pk/unique | 初始化记录 ID |
| `admin_id` | varchar(64) | 是 |  | unique | 初始平台管理员 ID |
| `account` | varchar(120) | 是 |  | unique | 初始化登录账号 |
| `credential_secret_ref` | varchar(255) | 是 |  |  | 初始化凭证来源引用，不保存明文密码 |
| `status` | varchar(32) | 是 | `initialized` | idx | `initialized`、`rotated`、`disabled` |
| `initialized_by` | varchar(64) | 是 | `system_seed` | idx | 初始化来源 |
| `trace_id` | varchar(128) | 是 |  | idx | 初始化链路 |
| `created_at` | timestamptz | 是 | now() | idx | 初始化时间 |
| `rotated_at` | timestamptz | 否 | null | idx | 首次密码轮换时间 |
| `updated_at` | timestamptz | 是 | now() |  | 更新时间 |

初始化规则：migration/seed 启动时先查询 `platform_admins` 是否存在 active 管理员；不存在时读取 `ADMIN_BOOTSTRAP_ACCOUNT` 和 `ADMIN_BOOTSTRAP_PASSWORD_HASH` 或安全配置引用创建初始管理员，并写 `platform_admin_bootstraps`。已存在 active 管理员时 seed 返回幂等成功，不创建重复账号。初始管理员首次登录成功后，后台必须返回 `must_rotate_password=true`，完成密码轮换后将该字段置为 false，并把 bootstrap 状态更新为 `rotated`。

## 权限规则

- 后台 RPC 必须 `login_identity_type=admin` 且 `admin_id` active。
- 普通用户 token 不可调用后台 RPC。
- 初始管理员只能由 seed 创建，HTTP/RPC 不暴露初始化入口。
- 初始管理员首次登录后必须完成密码轮换；`must_rotate_password=true` 时仅允许访问密码轮换和退出接口。
- admin 不可通过后台读取用户私有资产、会话、黑板、提示词、私有素材和生成记录。
- 禁用用户后，后续私有业务 RPC 返回 `PERMISSION_DENIED` 或 `RESOURCE_UNAVAILABLE`。
- 禁用用户不删除其公开作品；公开展示由作品状态控制。

## 事务设计

| 事务 | 原子写入 | 回滚条件 |
| --- | --- | --- |
| 初始管理员初始化 | `platform_admins`、`platform_admin_bootstraps`、审计 | 已存在 active 管理员时幂等返回；配置缺失、账号冲突、审计失败 |
| 管理员密码轮换 | `platform_admins.password_hash`、`must_rotate_password=false`、`platform_admin_bootstraps.rotated_at`、审计、幂等记录 | 当前密码错误、新密码强度不满足、审计失败 |
| 创建管理员 | `platform_admins`、审计、幂等记录 | 账号重复、审计失败 |
| 禁用/启用用户 | `business_users.status`、审计、幂等记录 | 用户不存在、preview 失效、审计失败 |

| 动作 | `business_action` | 审计内容 |
| --- | --- | --- |

## 【Agent开发】需要处理的影响

| 场景 | 业务服务行为 | 【Agent开发】处理 |
| --- | --- | --- |
| 用户被禁用后创建 run | `ResolveCurrentSpaceContext` 或 `CheckProjectAccess` 返回 `PERMISSION_DENIED` | 不创建 run，AG-UI 展示账号不可用 |
| 用户被禁用时已有 run | 后续业务 RPC 返回权限错误 | TurnLoop 停止新 Tool；如有冻结，调用释放冻结 |
| 后台查看用户 | 业务服务不返回 Agent session/run/黑板 | Agent 不需要提供后台私有内容查询能力 |
| 审计关联 Agent trace | 业务写 RPC 带 `trace_id`、`source=agent_service` | Agent 所有写 RPC 必须传 trace，便于后台审计排障 |

## 测试

- 普通用户调用后台 RPC 返回 `PERMISSION_DENIED`。
- 空库 seed 创建一个 active 初始管理员，重复 seed 不创建重复账号。
- 已存在 active 管理员时 seed 不覆盖密码、不改变已有管理员状态。
- 初始管理员首次登录返回 `must_rotate_password=true`，未轮换前访问后台业务接口返回 `STATE_CONFLICT`。
- 管理员密码轮换成功后 `must_rotate_password=false`，初始管理员 bootstrap 状态更新为 `rotated`。
- 后台用户列表不包含私有资产、会话、黑板、提示词字段。
- 禁用用户后登录和 Agent 私有业务 RPC 失败。
- 禁用用户公开作品不自动删除。
- 启禁用用户必须写审计，审计失败回滚状态变更。
- 幂等重复禁用返回同一结果，同 key 不同状态返回 `IDEMPOTENCY_CONFLICT`。
