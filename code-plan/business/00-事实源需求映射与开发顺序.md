## 产品到业务服务映射

| 产品条目 | 业务服务解释 | 业务文档 | 核心产出 | 【Agent开发】依赖 |
| --- | --- | --- | --- | --- |
| 当前空间决定 Skill、积分、资产、历史 | 空间是权限、积分账户、项目和资源隔离入口 | 03 | `AccountSpaceService.ResolveAuthContextFromToken`、`ResolveCurrentSpaceContext` | Agent HTTP 先用 Authorization token 解析 `auth_context` 和 `space_context`；创建 run 前消费 `space_context` |
| 统一 Agent 不写业务库 | 所有业务事实写入由业务 RPC 承接 | 全部 | Kitex RPC server、DTO、幂等、审计 | 业务写入 Tool 必须调用 RPC，不直接访问业务 DB |
| 初始平台管理员账号 | 空库初始化必须创建一个 active 管理员并强制首次密码轮换 | 04、14、15 | `BootstrapInitialAdmin`、`platform_admin_bootstraps`、`ADMIN_BOOTSTRAP_*` 配置 | 无 |
| 项目承载会话、资产、作品 | 项目是业务容器，Agent 只保存 `project_id` | 05 | `ProjectService`、`ProjectAssetService` | session/run/resume/commit 前按目的调用 `CheckProjectAccess` |
| 模型选择当前对话生效 | 业务服务提供可选模型、默认模型、价格快照和非敏感运行快照 | 06 | `ModelConfigService`、`ResolveGenerationModelSnapshot` | 安全评估后、预估前传 `model_id`、`resource_type`、`pricing_snapshot_id`；执行前解析运行快照 |
| Tool 边界 | 业务服务维护 Tool 策略，Agent 负责执行前校验 | 07 | `ToolCapabilityService.CheckToolExecutionPolicy` | 执行 Tool 前传 `tool_name`、`tool_type`、`project_id`、`risk_context` |
| Skill 只 Published 路由，Memory 默认开启且作用域可配置 | 业务服务保存 Skill 生命周期和 runtime spec，未传 `memory_policy` 时默认 session summary 策略；user/space 偏好必须保留授权要求；`confirmation_policy` 只能加强确认 | 08 | `SkillCatalogService`、`SkillReviewService`、`PublishedSkillSpecDTO.memory_policy`、`PublishedSkillSpecDTO.confirmation_policy` | 路由前分页加载 Skill 池，执行时使用 `published_skill_spec`，Agent 用策略和授权取交集，并合并确认策略 |

| 积分预估、冻结、扣费、释放 | 积分账户、批次和流水是业务事实；预估必须在 passed 安全证据后执行 | 09、11 | `CreditService`、`AssetCreditCommitService` | 生成或 Tool 执行前传 `safety_evidence` 预估，确认后冻结，独立 Tool 成功后扣费，失败/取消释放，资产保存成功提交扣费 |
| 兑换码绑定指定用户、企业或渠道 | 业务服务在兑换事务前校验绑定目标并保持兑换码不被误消耗 | 09、02、14、15 | `bind_target_type`、`bind_target_id`、`redeem_channel`、`REDEEM_CODE_TARGET_MISMATCH` | 无 |
| 资产保存和元素 | 最终资产和最终元素由业务服务保存；生成产物 object key 由业务服务签发 | 10、11 | `AssetService.PrepareGeneratedAssetObjects`、`AssetCreditCommitService` | 生成完成先准备对象槽并上传，再传产物、元素、`storage_object_ref`、`safety_evidence` |

| 管理端后台能力 | 每个业务领域提供对应 Admin HTTP API 和审计，不单独维护跨领域对接文档 | 04、06、07、08、09、12、13 | `/api/admin/**` 场景 DTO | 后台配置变更影响 Agent 能力池时需要 Agent 缓存失效或短 TTL |

## M3 / M4 / M6 Agent RPC 承接

业务服务必须按 Agent 07 的阶段拆分交付 RPC server、Application、Repository、migration、fixture 和测试。缺对应方法时，不得进入该阶段联调或验收。

| 全局阶段 | 业务文档和切片 | 业务必须交付给 Agent 的 RPC | 额外阻断项 |
| --- | --- | --- | --- |
| M3 配置能力 | 03 的 token 解析子集，06、07、08，另前置 10 的元素类型只读字典子集 | `AccountSpaceService.ResolveAuthContextFromToken`；`SkillCatalogService.ListRoutableSkills`、`GetPublishedSkillSpec`、`GetReviewCandidateSkillSpec`、`SaveSkillTestResult`；`ToolCapabilityService.CheckToolExecutionPolicy`；`ModelConfigService.ListAvailableGenerationModels`、`ResolveDefaultModel`、`ResolveGenerationModelSnapshot`；`PlatformDictionaryService.ListAssetElementTypes` | 业务 01/02 的分层、BusinessError DTO、公共字段基线和 request hash 必须能支撑 M3 RPC；M3 方法不得继续返回 `NOT_IMPLEMENTED`。 |
| M4 资金与资产 | 09、10、11，含 05 的项目资产权限用法 | `CreditService.EstimateGenerationCredits`、`EstimateToolCredits`、`FreezeCredits`、`ChargeToolUsageCredits`、`ReleaseFrozenCredits`；`AssetService.BatchCheckAssetAccess`、`PrepareGeneratedAssetObjects`；`AssetCreditCommitService.CommitGeneratedAssetAndCharge`；`ProjectService.CheckProjectAccess` 的 `attach_asset`、`commit_asset`、`create_work` 用法 | 项目资产绑定、资产访问校验、对象槽签发、资产元素入库和保存扣费事务必须完成；保存失败不得扣费，失败/取消/归档必须释放冻结。 |
| M6 服务级验收 | 03、04、05、14、15 以及所有 M3/M4 切片 | M2 遗留的 owner transfer、管理员后台 RPC、项目创建/更新/资产绑定等业务 03/04/05 契约按产品范围收口；全量 Agent-facing RPC 做 contract test 和 `NOT_IMPLEMENTED` 扫描。 | 业务 14 不能只做 fixture 形状校验；必须有真实 RPC/HTTP contract、repository、migration、DB 边界和服务级主链路证据，未执行项写原因、影响和 owner。 |

## 跨域契约优先级

1. 通用 Thrift：`AuthContext`、`RequestMeta`、`AuditContext`、`Pagination`、`BusinessError`、`SafetyEvidenceDTO`。
2. Agent 必需 RPC：`ResolveAuthContextFromToken`、`ResolveCurrentSpaceContext`、`CheckProjectAccess`、`ListRoutableSkills`、`CheckToolExecutionPolicy`、`ListAvailableGenerationModels`、`ResolveGenerationModelSnapshot`、`EstimateGenerationCredits`、`EstimateToolCredits`、`FreezeCredits`、`ChargeToolUsageCredits`、`ReleaseFrozenCredits`、`PrepareGeneratedAssetObjects`、`CommitGeneratedAssetAndCharge`、`BatchCheckAssetAccess`。
3. 前端 API 适配层：账户、企业、项目、后台、模型、Tool、Skill、积分、资产、作品、通知必须写回对应领域文档，不能另建脱离领域状态机的“对接设计”。
4. 公开读 RPC：公开作品、精选作品、公开 Skill 摘要。
5. 后台运营 RPC：管理员、用户状态、模型配置、Tool 策略、Skill 审核、积分发放、下架、审计查询。

## Agent 交付参数总表

| 【Agent开发】能力 | 业务服务方法 | Agent 必传参数 | 业务服务返回 | 错误处理 |
| --- | --- | --- | --- | --- |
| 解析登录 token | `ResolveAuthContextFromToken` | `authorization`、`trace_id`、`expected_space_id` 可选 | `auth_context`、`space_context`、`session_id` | `UNAUTHENTICATED` 终止请求；`CROSS_SPACE_DENIED` 不信任客户端空间头 |
| 解析当前空间 | `ResolveCurrentSpaceContext` | `auth_context`、`trace_id` | `space_id`、`space_type`、`credit_account_id`、`enterprise_role`、`permission_summary` | `UNAUTHENTICATED` 终止 run；`PERMISSION_DENIED` 发可理解错误 |
| 校验项目访问 | `CheckProjectAccess` | `project_id`、`access_purpose`、`auth_context` | `allowed`、`project_status`、`allowed_actions` | `PROJECT_ARCHIVED` 停止新 Tool，必要时释放冻结 |
| 加载 Skill 池 | `ListRoutableSkills` | `space_id`、`actor_user_id`、`page_size=10`、`resource_type` 可选 | `skills[]`、`next_page_token` | 无可用 Skill 时文本兜底 |
| 获取 Skill runtime spec | `GetPublishedSkillSpec` | `skill_id`、`version_id` 可选、`space_id` | `skill_spec`、`output_element_schema[]`、`tool_bindings[]`、`memory_policy`、`confirmation_policy`、`execution_policy_summary` | Skill 不可用则重新路由或失败 |
| Tool 执行策略 | `CheckToolExecutionPolicy` | `tool_name`、`tool_type`、`project_id`、`risk_context`、`auth_context` | `allowed`、`risk_level`、`timeout_ms`、`retry_policy`、`requires_confirmation` | 停用/无权限不执行 Tool |
| 模型选择 | `ListAvailableGenerationModels` / `ResolveDefaultModel` / `ResolveGenerationModelSnapshot` | `resource_type`、`space_id`、`model_id`、`pricing_snapshot_id` | `models[]`、`default_model`、`pricing_snapshot_id`、非敏感 `model_snapshot` | 无可用模型或确认后停用时阻断生成并释放冻结 |
| 积分预估 | `EstimateGenerationCredits` | `project_id`、`resource_type`、`model_id`、`quantity`、`duration_seconds`、`pricing_snapshot_id`、`tool_usage_items[]` 非必填、`safety_evidence` | `estimate_id`、`line_items[]`、`estimate_points`、`available_points`、`expires_soon_points`、`credit_account_scope`、`insufficient` | 积分不足不进入确认；安全证据无效不创建预估 |
| Tool 积分预估 | `EstimateToolCredits` | `project_id`、`tool_usage_items[].tool_name`、`tool_type`、`billing_unit`、`quantity`、`metadata_summary`、`safety_evidence` | `estimate_id`、`line_items[].estimate_item_id`、`pricing_policy_id`、`estimate_points`、`credit_account_scope` | Agent 不传单价，只展示业务返回 |
| 冻结积分 | `FreezeCredits` | `estimate_id`、`points`、`run_id`、`idempotency_key`、`auth_context`；`confirmation_id` 可选 | `freeze_id`、`frozen_points`、`expires_at` | 冻结失败不执行生成 |
| 独立 Tool 成功扣费 | `ChargeToolUsageCredits` | `estimate_id`、`freeze_id`、`session_id`、`run_id`、`charge_items[].estimate_item_id`、`tool_call_id`、`actual_quantity`、`idempotency_key` | `charged_line_items[]`、`charged_points`、`freeze_status` | 只对不生成资产且独立扣费 Tool 调用 |
| 释放冻结 | `ReleaseFrozenCredits` | `freeze_id`、`release_points`、`reason`、`run_id`、`idempotency_key` | `released_points`、`release_status` | 可幂等重试 |
| 准备生成产物对象 | `PrepareGeneratedAssetObjects` | `project_id`、`session_id`、`run_id`、`artifacts[]`、`idempotency_key` | `upload_slots[]`，含 `object_key`、`upload_url`、`upload_headers` | Agent 上传到指定 object key，不能自造 key 或保存供应商临时 URL |
| 保存资产并扣费 | `CommitGeneratedAssetAndCharge` | `freeze_id`、`project_id`、`session_id`、`run_id`、`artifacts[].storage_object_ref`、`final_elements[]`、`safety_evidence`、`idempotency_key`；`estimate_id`、`artifacts[].estimate_item_id` 新 Agent 必传 | `asset_refs[]`、`charged_line_items[]`、`charged_points`、`released_points` | 保存失败不扣对应积分；已结算 `estimate_item_id` 不重复扣 |
| 批量校验资产引用 | `BatchCheckAssetAccess` | `asset_ids[]`、`project_id`、`access_purpose` | `results[]`、`denied_asset_ids[]` | 有拒绝则阻止引用或提示移除 |
