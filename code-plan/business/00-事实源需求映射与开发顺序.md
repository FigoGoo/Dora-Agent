## 产品到业务服务映射

| 产品条目 | 业务服务解释 | 业务文档 | 核心产出 | 【Agent开发】依赖 |
| --- | --- | --- | --- | --- |
| 当前空间决定 Skill、积分、资产、历史 | 空间是权限、积分账户、项目和资源隔离入口 | 03 | `AccountSpaceService.ResolveCurrentSpaceContext` | 创建 run 前传 `actor_user_id`、`space_id`、`login_identity_type` 并消费 `space_context` |
| 统一 Agent 不写业务库 | 所有业务事实写入由业务 RPC 承接 | 全部 | Kitex RPC server、DTO、幂等、审计 | 业务写入 Tool 必须调用 RPC，不直接访问业务 DB |
| 初始平台管理员账号 | 空库初始化必须创建一个 active 管理员并强制首次密码轮换 | 04、14、15 | `BootstrapInitialAdmin`、`platform_admin_bootstraps`、`ADMIN_BOOTSTRAP_*` 配置 | 无 |
| 项目承载会话、资产、作品 | 项目是业务容器，Agent 只保存 `project_id` | 05 | `ProjectService`、`ProjectAssetService` | session/run/resume/commit 前按目的调用 `CheckProjectAccess` |
| 模型选择当前对话生效 | 业务服务提供可选模型、默认模型、价格快照 | 06 | `ModelConfigService` | 安全评估后、预估前传 `model_id`、`resource_type` |
| Tool 边界 | 业务服务维护 Tool 策略，Agent 负责执行前校验 | 07 | `ToolCapabilityService.CheckToolExecutionPolicy` | 执行 Tool 前传 `tool_name`、`tool_type`、`project_id`、`risk_context` |
| Skill 只 Published 路由，Memory 默认开启 | 业务服务保存 Skill 生命周期和 runtime spec，未传 `memory_policy` 时默认 `enabled=true` | 08 | `SkillCatalogService`、`SkillReviewService`、`PublishedSkillSpecDTO.memory_policy` | 路由前分页加载 Skill 池，执行时使用 `published_skill_spec` |
| 提示词安全评估前置 | 业务写入接收脱敏安全证据，不做模型推理链路保存 | 10、11、12 | `SafetyEvidenceDTO` 校验规则 | 提供 `safety_evidence`，含摘要、场景、过期时间和策略版本 |
| 积分预估、冻结、扣费、释放 | 积分账户、批次和流水是业务事实 | 09、11 | `CreditService`、`AssetCreditCommitService` | 生成或 Tool 执行前预估，确认后冻结，独立 Tool 成功后扣费，失败/取消释放，资产保存成功提交扣费 |
| 兑换码绑定指定用户、企业或渠道 | 业务服务在兑换事务前校验绑定目标并保持兑换码不被误消耗 | 09、02、14、15 | `bind_target_type`、`bind_target_id`、`redeem_channel`、`REDEEM_CODE_TARGET_MISMATCH` | 无 |
| 资产保存和元素 | 最终资产和最终元素由业务服务保存 | 10、11 | `AssetService`、`AssetCreditCommitService` | 生成完成传产物、元素、TOS 引用、`safety_evidence` |
| 上传素材 | 业务服务签发上传意图并确认资产 | 10 | `CreateUploadIntent`、`ConfirmUploadedAsset` | 若由 Agent 发起上传元数据安全评估，传 `asset_upload_metadata` 证据 |
| 资产元素类型发布或变更审计 | 内置元素类型通过同步机制维护，变更必须入记录和后台审计 | 10、14、15 | `SyncAssetElementTypes`、`asset_element_type_change_records` | Agent 通过 `ListAssetElementTypes` 读取 active 类型，不参与变更 |
| 作品分享与公开快照 | 公开状态和快照由业务服务维护，公开列表、详情和分享结果返回可复制 `share_url` | 12 | `WorkShareService`、`PublicContentService`、`PUBLIC_WEB_BASE_URL` | 分享前如复用 Agent 安全评估，传 `work_share` 证据 |
| Skill 审核通知 | 通知是业务触达，不是 Agent Runtime | 08、13 | `NotificationService.CreateNotification` | 无需 Agent 生产通知，仅可传 trace 关联 |
| 用户端页面能力 | 每个业务领域提供对应 User/Public HTTP API，不单独维护跨领域对接文档 | 03～13 | `/api/**`、`/api/public/**` 场景 DTO | Agent 工作台实时流仍由 Agent API/AG-UI 负责 |
| 管理端后台能力 | 每个业务领域提供对应 Admin HTTP API 和审计，不单独维护跨领域对接文档 | 04、06、07、08、09、12、13 | `/api/admin/**` 场景 DTO | 后台配置变更影响 Agent 能力池时需要 Agent 缓存失效或短 TTL |

## 开发顺序和依赖

| 阶段 | 文档 | 必须先完成 | 可并行内容 | Done 条件 |

| M1 身份与项目 | 03、04、05 | 01、02 | 后台和项目可部分并行 | 空间上下文、项目权限、归档规则、用户端和后台对应 API 可供 Agent 和前端调用 |

## 跨域契约优先级

1. 通用 Thrift：`AuthContext`、`RequestMeta`、`AuditContext`、`Pagination`、`BusinessError`、`SafetyEvidenceDTO`。
2. Agent 必需 RPC：`ResolveCurrentSpaceContext`、`CheckProjectAccess`、`ListRoutableSkills`、`CheckToolExecutionPolicy`、`ListAvailableGenerationModels`、`EstimateGenerationCredits`、`EstimateToolCredits`、`FreezeCredits`、`ChargeToolUsageCredits`、`ReleaseFrozenCredits`、`CommitGeneratedAssetAndCharge`、`BatchCheckAssetAccess`。
3. 前端 API 适配层：账户、企业、项目、后台、模型、Tool、Skill、积分、资产、作品、通知必须写回对应领域文档，不能另建脱离领域状态机的“对接设计”。
4. 公开读 RPC：公开作品、精选作品、公开 Skill 摘要。
5. 后台运营 RPC：管理员、用户状态、模型配置、Tool 策略、Skill 审核、积分发放、下架、审计查询。

## Agent 交付参数总表

| 【Agent开发】能力 | 业务服务方法 | Agent 必传参数 | 业务服务返回 | 错误处理 |
| --- | --- | --- | --- | --- |
| 解析当前空间 | `ResolveCurrentSpaceContext` | `auth_context`、`trace_id` | `space_id`、`space_type`、`credit_account_id`、`enterprise_role`、`permission_summary` | `UNAUTHENTICATED` 终止 run；`PERMISSION_DENIED` 发可理解错误 |
| 校验项目访问 | `CheckProjectAccess` | `project_id`、`access_purpose`、`auth_context` | `allowed`、`project_status`、`allowed_actions` | `PROJECT_ARCHIVED` 停止新 Tool，必要时释放冻结 |
| 加载 Skill 池 | `ListRoutableSkills` | `space_id`、`actor_user_id`、`page_size=10`、`resource_type` 可选 | `skills[]`、`next_page_token` | 无可用 Skill 时文本兜底 |
| 获取 Skill runtime spec | `GetPublishedSkillSpec` | `skill_id`、`version_id` 可选、`space_id` | `skill_spec`、`output_element_schema[]`、`tool_bindings[]` | Skill 不可用则重新路由或失败 |
| Tool 执行策略 | `CheckToolExecutionPolicy` | `tool_name`、`tool_type`、`project_id`、`risk_context`、`auth_context` | `allowed`、`risk_level`、`timeout_ms`、`retry_policy`、`requires_confirmation` | 停用/无权限不执行 Tool |
| 模型选择 | `ListAvailableGenerationModels` / `ResolveDefaultModel` | `resource_type`、`space_id` | `models[]`、`default_model`、`pricing_snapshot_id` | 无可用模型阻断生成 |
| 积分预估 | `EstimateGenerationCredits` | `project_id`、`resource_type`、`model_id`、`quantity`、`duration_seconds`、`pricing_snapshot_id`、`tool_usage_items[]` 非必填、`safety_evidence` | `estimate_id`、`line_items[]`、`estimate_points`、`available_points`、`expires_soon_points`、`insufficient` | 积分不足不进入确认 |
| Tool 积分预估 | `EstimateToolCredits` | `project_id`、`tool_usage_items[].tool_name`、`tool_type`、`billing_unit`、`quantity`、`metadata_summary` | `estimate_id`、`line_items[].estimate_item_id`、`pricing_policy_id`、`estimate_points` | Agent 不传单价，只展示业务返回 |
| 冻结积分 | `FreezeCredits` | `estimate_id`、`points`、`run_id`、`idempotency_key`、`auth_context`；`confirmation_id` 可选 | `freeze_id`、`frozen_points`、`expires_at` | 冻结失败不执行生成 |
| 独立 Tool 成功扣费 | `ChargeToolUsageCredits` | `estimate_id`、`freeze_id`、`session_id`、`run_id`、`charge_items[].estimate_item_id`、`tool_call_id`、`actual_quantity`、`idempotency_key` | `charged_line_items[]`、`charged_points`、`freeze_status` | 只对不生成资产且独立扣费 Tool 调用 |
| 释放冻结 | `ReleaseFrozenCredits` | `freeze_id`、`release_points`、`reason`、`run_id`、`idempotency_key` | `released_points`、`release_status` | 可幂等重试 |
| 保存资产并扣费 | `CommitGeneratedAssetAndCharge` | `freeze_id`、`project_id`、`session_id`、`run_id`、`artifacts[]`、`final_elements[]`、`safety_evidence`、`idempotency_key`；`estimate_id`、`artifacts[].estimate_item_id` 新 Agent 必传 | `asset_refs[]`、`charged_line_items[]`、`charged_points`、`released_points` | 保存失败不扣对应积分；已结算 `estimate_item_id` 不重复扣 |
| 批量校验资产引用 | `BatchCheckAssetAccess` | `asset_ids[]`、`project_id`、`access_purpose` | `results[]`、`denied_asset_ids[]` | 有拒绝则阻止引用或提示移除 |

## 验收标准

- [ ] 所有产品文档 `product_status` 已确认为 Done。
- [ ] 每个产品业务规则都映射到业务文档、RPC、数据模型或明确非业务服务职责。
- [ ] 每个业务领域都给出用户端 / 管理端 / 公开端适用的 HTTP API 清单、DTO、权限、幂等和页面状态映射。
- [ ] 所有需要 Agent 提供的输入已用【Agent开发】标注字段和错误处理。
- [ ] 功能点顺序满足先基础、再配置、再资金资产、最后公开触达和测试。
- [ ] 没有把 Agent Runtime 数据纳入业务数据库设计。
- [ ] 没有要求 Agent 直接访问业务数据库。
- [ ] 生产级闭眼门禁 15 已覆盖 Agent 对齐、功能切片、配置、发布、回滚和 smoke test。
